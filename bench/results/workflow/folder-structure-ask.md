## 권고 (측정 해석 — 적용함)

- **always-ask 룰 적용** — iter-1로 검증. `foundation.md:20` 교체 완료.
- **read 클로즈는 `[CRITICAL]` 강제 대신 경로 명시 일반 지시로.** iter-2에서 강제의 측정 가능한 이득 0 + 소프트 언급으로도 100% 준수 → 과한 강제 불필요.

## 추정 (직접 측정 안 됨)

- 원래 실패(잘못된 DDD 트리: `src/domains/` 래퍼 + 컴포넌트별 폴더 + index.ts 배럴)는 **sonnet-4-6에서 재현 안 됨**. 모델 내재화로 추정하나, 시나리오 부족 가능성도 배제 불가 (rule-ablation 「단일 측정 확정 불가」). D 판정 아님 — read 클로즈는 회귀·약한 모델·FSD-priming 대비 **유지**.
- **재측정 트리거**: 모델 메이저 업데이트 시 iter-2 spot check. FSD를 우대에 둔 DDD-지정 시나리오는 미측정 — 추후 보강 후보.
