# instructions/ 하위 README.md 작성 가이드

이 문서는 `instructions/` 폴더 하위에 README.md를 언제, 어떻게 작성해야 하는지 설명합니다.

---

## README.md 생성 조건

다음 중 **하나 이상** 해당하면 README.md를 만드세요:

### ✅ 만들어야 하는 경우

1. **파일 간 순서/관계가 있을 때**
   - 예: step-1 → step-2 → step-3 순서대로 읽어야 함
   - 예: A.md를 먼저 읽어야 B.md 이해 가능

2. **하위 폴더가 여러 개일 때 (카테고리 분류)**
   - 예: `coding/`, `testing/`, `review/` 등 여러 카테고리

3. **폴더 목적이 파일명만으로 불명확할 때**
   - 예: `self-help/` ← 이게 뭐 하는 폴더인지 설명 필요

4. **파일이 5개 이상일 때**
   - 네비게이션 도움 필요

5. **상황별로 다른 파일을 읽어야 할 때**
   - 예: React 작업 시 A.md, Vue 작업 시 B.md

### ❌ 불필요한 경우

1. **파일이 1-2개이고 독립적일 때**
2. **모든 파일이 독립적이고 순서 없을 때**
3. **파일명만 봐도 용도가 명확할 때**

---

## README.md의 역할

### 1. 폴더 개요
- 이 폴더가 무엇을 위한 것인가
- 언제/왜 이 폴더의 문서를 읽는가

### 2. 파일 간 관계 설명
- 순서: step-1 → step-2 → step-3
- 의존성: A를 먼저 읽어야 B 이해 가능
- 카테고리: coding/, testing/, review/ 각각 뭔지

### 3. 상황별 사용 가이드 (핵심!)
각 폴더/파일이 **"언제 필요한지"**를 명확히 안내ㅇ하세요.

---

## 작성 가이드

### 기본 구조

```markdown
# 개요
[이 폴더의 목적과 용도를 2-3문장으로 설명]

---

## 📖 폴더 구조

### [하위 폴더 1]/
[폴더 역할 설명]

**언제 읽는지 명시**

### [하위 폴더 2]/
[폴더 역할 설명]

**언제 읽는지 명시**

---

## 상황별 가이드
[작업 유형별로 어떤 폴더를 읽어야 하는지 안내]
```

**중요**: 개별 파일 목록은 나열하지 마세요. 파일이 추가/삭제될 때마다 README 수정이 필요해집니다.

## 핵심 원칙

### ✅ DO (해야 할 것)

1. **"언제 읽는지" 명시**
   - 각 폴더가 어떤 상황에 필요한지 명시
   - 예: "**기능 구현, 버그 수정, 리팩토링 시 참고하세요.**"

2. **개요를 간결하게 (2-3문장)**
   - 폴더 목적
   - 언제 사용하는지

3. **상황별 가이드 제공**
   - 작업 유형별로 어떤 폴더를 읽어야 하는지 안내
   - 예: "기능 구현 시 → essentials/, coding/ 순서로"

### ❌ DON'T (하지 말 것)

1. **개별 파일 목록 나열하지 말기**
   - 파일 추가/삭제될 때마다 README 수정 필요
   - 폴더 수준 설명만 제공

2. **AI 전용 섹션 지양**
   - ❌ "## AI를 위한 가이드" "## Instructions for AI"
   - ✅ 사람도 함께 이해할 수 있는 제목과 표현 사용
   - 예: "사용 가이드", "주의사항", "진행 방법"

3. **내용 중복하지 말기**
   - README는 목차/개요만
   - 구체적 내용은 각 파일에

4. **너무 길게 쓰지 말기**
   - README는 빠른 네비게이션용
   - 한 페이지에 다 보여야 함
