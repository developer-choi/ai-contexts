# 마크다운 작성 가이드

이 문서는 프로젝트 내에서 마크다운(Markdown, `.md`) 문서를 작성할 때 따라야 할 스타일 및 서식 규칙을 정의합니다.

---

## 1. 이모지 사용 규칙

### 기본 원칙: 사용 금지
마크다운 문서 내에서 이모지 사용은 금지됩니다.

### 예외: 허용되는 경우
다음과 같이 **Before & After** 비교나 **Bad & Good** 대조를 명확히 보여주기 위한 목적으로는 제한적으로 허용됩니다.

- ✅ (Check Mark): Good Case / 권장
- ❌ (Cross Mark): Bad Case / 금지

**사용 예시:**
```markdown
**❌ Bad**
const LINE_1_START = 10;

**✅ Good**
const LINE_1 = { START: 10 };
```

---

## 2. 헤더 및 텍스트 작성 규칙

### 한글(영어) 중복 병기 금지
`한국어 (English)` 형태의 중복 표기는 금지됩니다. 이미 한국어로 의미가 전달되는 단어 뒤에 괄호로 영문을 병기하지 마세요.

**❌ Bad (동의어 반복)**
- `## 계획 (Plan)`
- `## 요약 (Summary)`
- `## 참조해야 할 문서 (Context Load)`

**✅ Good (간결하게)**
- `## 계획` 또는 `## Plan`
- `## Summary`
- `## Context Load`

### 예외: 의미 결합
서로 다른 의미를 결합하거나 고유 명사를 함께 써야 하는 경우는 허용됩니다.

**✅ Allowed**
- `## Track 2. 문제 해결 논리 검증` (식별자 + 설명)
- `## AI의 역할 (Quiz Runner)` (구체적 모드 명시)
