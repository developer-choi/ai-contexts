# Doc Router

구글 문서에 쌓아둔 필기를 활용 가능한 형태로 옮겨 담고, 원본 문서를 지워도 될 만큼 남김없이 비워내는 스킬입니다.

## 동기

그동안 공부한 내용을 구글 문서에 필기해왔습니다. 하지만 구글 문서에 쌓아두기만 하면 나중에 활용하기 어렵고, 정리하는 것 자체도 시간이 오래 걸렸습니다.

이 스킬은 구글 문서의 필기를 읽어 실제로 활용할 수 있는 곳으로 옮깁니다. 학습해서 설명할 지식은 [knowledge-archive](https://github.com/developer-choi/knowledge-archive)로 보냅니다.

예제 코드로 발전시킬 구현 재료나 앞으로 할 작업 메모는 별도의 작업 보관소에 임시로 모아두고, 그곳의 정리 절차에 넘깁니다. 이 보관소는 필기 정리와 분리된 별개의 공간입니다.

## 완전 캡처가 목표

이 스킬이 지향하는 끝 지점은 구글 문서를 비우는 것입니다. 필기 내용을 남김없이 옮겨 담아, 원본 구글 문서를 지워도 될 상태로 만드는 것이 목표입니다.

그래서 글자뿐 아니라 그림, 배경색으로 강조한 부분, 도표 같은 시각 정보까지 빠뜨리지 않고 옮깁니다. 이미지도 AI가 직접 추출해 옮기며 — 코드 캡처나 표처럼 글로 풀 수 있는 것은 텍스트로 바꾸고, 화면·도표만 이미지로 남깁니다 — 그래도 자동으로 옮기기 어려운 것만 "지우기 전에 직접 옮겨두세요"라고 원본별로 따로 알려드립니다.

옮기는 도중에는 원본을 바로 지우지 않아, 제가 검토한 뒤 되살릴 수 있습니다. 다만 이것은 정리하는 동안의 임시 안전장치일 뿐, 원본을 오래 보관하려는 것은 아닙니다.

### 옮긴 뒤 연결 고리 정리

한 문서를 옮기고 나면, 그 문서를 구글 문서 주소로 가리키던 다른 글의 링크가 원본을 지운 순간 끊깁니다.

그래서 문서를 옮긴 직후, 그 문서를 가리키던 링크를 새로 옮겨둔 위치로 바꿔둡니다. 나중에 원본을 지워도 링크가 살아 있게 하기 위해서입니다.

## 보류·폐기 분리

행선지가 애매한 항목은 보류로, 필기로서 가치가 없어 보이는 항목은 폐기로 갈라서 제가 다시 판단할 수 있게 넘깁니다. 어느 쪽도 곧바로 지우지 않으므로, 검토한 뒤 되살릴 수 있습니다.
