# Component 상세 명세 템플릿

컴포넌트 개발 작업의 **특화 항목**을 작성하는 템플릿입니다.

> **주의**: 이 템플릿은 컴포넌트 특화 항목만 다룹니다.
> 공통 항목(개요, 예상 영향 범위, 핵심 코드 스니펫, 기본 테스트 케이스)은 [`step-5.md`](../step-5.md)를 참고하세요.

---

## Props Interface
컴포넌트의 Props 타입을 정의합니다. 모든 props의 타입, 선택/필수 여부, 설명을 명확히 작성하세요.

### 예시
```typescript
interface ComponentNameProps {
  title: string;
  isVisible: boolean;
  onClose: () => void;
  description?: string;
}
```

---

## 데이터 표시 및 형식
리스트나 테이블을 다룰 때, **각 항목의 데이터가 어떻게 표시되는지** 구체적으로 기술합니다.

### 예시
- **텍스트 포맷**: 날짜(YYYY.MM.DD), 금액(1,000원), 말줄임 처리 등.
- **상태별 시각 요소**: 상태값에 따른 배지 컬러(NOTICE: Red, INFO: Blue), 아이콘 등.
- **예외 문구**: 데이터가 없을 때 표시할 텍스트 ("데이터가 없습니다." 등).

---

## UX 전략 및 워크플로우
주요 동작의 흐름 및 사용자 인터랙션 방식을 구체적으로 기술합니다.

### 예시
- **트리거**: 엔터 키 입력 시 검색 실행, 클릭 시 모달 오픈 등.
- **동작 방식**: 무한 스크롤(자동 로드), URL 동기화(쿼리 파라미터 업데이트) 등.
- **페이지 이동 전략**: 싱글 페이지 지향 (모달 활용 여부 등).
- **CRUD 워크플로우**: 각 액션(작성, 수정, 삭제)이 어디서 어떤 컴포넌트를 통해 일어나는지.

---

## 상태 피드백
성공/실패 알림, 로딩, 에러 UI 처리 방식을 명시합니다. 각 상태별 UI가 어떻게 달라지는지 정의하세요.

### 예시
- **Normal**: 기본 상태
- **Loading**: 데이터 로딩 중 (스켈레톤 UI 사용 여부, 스피너 노출 위치)
- **Error**: 에러 발생 시 (토스트 알림, 인라인 콜아웃, 또는 다이얼로그 노출 방식)
- **Empty**: 데이터가 없을 때 (Empty 컴포넌트 노출 조건)

---

## 스타일링 & 엣지 케이스
해당 컴포넌트의 디자인 디테일 및 예외 상황에 대한 처리를 정의하세요.

### 예시
- **텍스트 오버플로우**: 텍스트가 길어질 경우 (줄바꿈 vs 말줄임표 `...`)
- **이미지 에러**: 이미지가 로드되지 않았을 때 (Fallback 이미지 등)
- **반응형 동작**: 모바일/데스크탑에서의 레이아웃 변화
