# Step 1: 프로젝트 배경 파악

이 단계는 프로젝트의 **배경**과 **목적**을 파악하고, 관련 자료를 정리하는 단계입니다.

모든 프로젝트는 Step 1부터 시작합니다.

---

## [CRITICAL] 필수 제공 자료 확인

작업을 시작하기 위해 아래 목록 중 **최소 1개 이상**의 자료가 반드시 필요합니다.

### 검토 자료 목록
- [기획서 링크]
- [피그마 링크]
- [회의 음성 텍스트 요약본]
- [관련 필기 / 티켓]
- [간단한 개발설계]

### 대응 수칙
1. **자료가 하나도 없는 경우**:
   - 즉시 작업을 멈추고 **"작업을 위한 기획서, 피그마, 또는 티켓 링크를 제공해 주세요."**라고 요청하세요.
   - 임의로 추측하여 문서를 작성하지 마십시오.

2. **자료가 하나라도 있는 경우**:
   - **"제공해주신 자료 외에 추가로 참고해야 할 문서가 있나요? 없다면 바로 작성을 시작하겠습니다."**라고 확인 후 진행하세요.

## 작성 양식 (`/plan/background.md`)

`/plan/background.md` 파일을 생성하고 다음 내용을 작성하세요:

```markdown
# 프로젝트 배경

## 1. 비즈니스 배경
- **문제**: 어떤 문제를 해결하려고 하는가?
- **목표**: 이 프로젝트로 달성하려는 목표는?
- **대상 사용자**: 누구를 위한 기능인가?

## 2. 참고 자료
- [기획서 링크]
- [피그마 링크]
- [회의록 링크]
- [티켓 링크]
```

**작성 원칙:**
- 간결하게 작성 (3-5문장)
- "왜"에 집중 (구체적인 "무엇"과 "어떻게"는 다음 단계에서)
- 링크만 나열해도 OK

**예시:**
```markdown
# 프로젝트 배경

## 1. 비즈니스 배경
- **문제**: 마케팅 수신거부 유저가 78%로 너무 높음
- **목표**: 인앱에서 수신 동의를 독려하여 마케팅 도달률 개선
- **대상 사용자**: 마케팅 수신 OFF 상태인 유저

## 2. 참고 자료
- [피그마 - 마케팅 독려](https://figma.com/...)
- [피그마 - 알림 OFF 안내](https://figma.com/...)
```

## Step 1 완료 후

background.md 작성이 완료되면, 다음 단계를 판단하세요:

### 복잡한 프로젝트인가요?
다음 중 하나라도 해당되면 **[Step 2](./step-2.md)로 이동**하세요:
- ☐ 여러 페이지/컴포넌트에 걸친 작업 (2개 이상)
- ☐ 공통 모듈 개발 필요
- ☐ 외부 협업 필요 (앱팀, 백엔드팀)
- ☐ 단계적 배포 필요

### 단순한 작업인가요?
위 항목이 하나도 해당되지 않으면 **[Step 3](./step-3.md)으로 이동**하세요.

**흐름:**
```
복잡한 프로젝트: Step 1 → Step 2 (PR 분할) → Step 3 (각 PR별)
단순한 작업: Step 1 → Step 3
```
