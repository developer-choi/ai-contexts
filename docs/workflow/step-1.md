# Step 1: Motivation 문서 작성 요청
이 단계는 기능 구현의 목적과 방향성을 잡는 단계입니다.

# 요청 내용
**[DOMAIN_NAME]** 도메인의 **[FEATURE]** 기능을 구현하려고 합니다.

전체 작업은 [계획 > 개발 > 리뷰] 순으로 진행되며, 지금은 첫 번째 **계획** 단계입니다.

사용자가 아래의 정보를 제공했는지 확인해주세요.

- 사용자가 제공해야하는 자료
  - `docs/workflow/background.md`
  - [기획서 링크]
  - [피그마 링크]
  - [회의 음성 텍스트 요약본]
  - [관련 필기]
  - [간단한 개발설계 ]

사용자가 자료를 모두 제공했다면,

아래 양식에 맞춰 `/plan/motivation.md` 파일을 작성해주세요.

## 작성 양식 (`/plan/motivation.md`)
### 1. 목적 (Why)
- 현재 문제점 또는 개선하고 싶은 부분
- 이 기능을 왜 만드는가?

### 2. 내용 (What & How)
- 구현(해결) 방법의 개요

### 3. 참고 자료
- 기획서, 디자인, 회의록 링크 등
