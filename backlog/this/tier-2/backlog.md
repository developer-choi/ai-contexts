# backlog

## 동기

백로그를 작업할 때 서로 간의 의존성이나 실행 순서가 꼬이지 않고, 순서대로 올바르게 처리할 수 있도록 돕기 위함입니다.

## [ideation] 백로그 순차 처리 및 의존성 관리 방안

### 동기
백로그 간에 선행 관계가 있는 경우, 선행 작업이 끝나지 않은 백로그를 먼저 착수하거나 중복으로 검토하는 비효율을 방지하고 순서대로 처리하기 위함입니다.

### 핵심 아이디어
- **방법 1 (의존성 명시)**: 백로그 파일의 frontmatter에 `dependencies: [tier-2/some-task.md, tier-1/another.md]` 와 같이 선행 파일 경로를 지정하여, 실행 모드(`exec`) 진입 시 선행 백로그가 해결(삭제/완료)되었는지 자동 검증하는 규칙 도입.
- **방법 2 (네이밍 및 폴더 구조)**: 의존성 번호나 실행 단계를 나타내는 접두사(예: `01-`, `02-`)를 활용하거나 하위 디렉토리를 분리해 순서를 시각화하는 방안.
- **방법 3 (리뷰/실행 모드 오케스트레이션)**: 실행 모드(`exec`) 실행 시 의존성이 꼬인 경우 사용자에게 경고하거나 차단하는 장치 마련.

### 기대상황
- 백로그 처리기(exec/review 모드)가 백로그를 순회할 때 선행 백로그 완료 여부를 판단하여 올바른 순서로 작업을 안내받음.
