# scw

## [ideation] assertion 검증 구조: 팀 vs writer/critic

- 현재 scw가 assertion을 검증할 때 팀 구조(여러 에이전트 역할 분담)를 쓰고 있는데, writer/critic 쌍방이 더 나을 수도 있다는 가설
- writer가 초안 작성 → critic이 반박 → writer가 재반영하는 루프
- 팀 구조 대비 장점: 역할이 명확하고 루프가 단순, 설계 부담 낮음
- 파고들기 전 현재 assertion 검증 흐름을 먼저 파악할 것
