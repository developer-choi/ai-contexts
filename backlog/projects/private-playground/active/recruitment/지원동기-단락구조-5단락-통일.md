# 지원동기: 4단락(합본/customer 공감) 케이스 제거 → 5단락 단일 구조 통일

> PP = `~/WebstormProjects/main/private-playground/`. 아래 경로는 PP 레포 상대경로.

## 동기

지원동기 구조가 "5단락 / 4단락 합본" 두 경우의 수로 병존해, AI도 사용자도 매번 어느 구조인지 혼란한다. 경우의 수를 5단락 하나로 없애 단순화한다.

## [ready] 4단락 합본 경로 제거, 5단락으로 일원화

### 기대상황 (필수)

- 지원동기 구조는 **항상 5단락**. 4단락·합본·"customer 공감" 개념/경로가 스킬 본문과 프리셋에서 **전부 제거**되고, 남은 문서끼리 정합.
- 단일 규칙:
  - **단락1** = 도메인(또는 가치관) 공감. `…그 점이 공감이 됐습니다`로 닫는다. (지원 착지 없음)
  - **단락2** = 본인 사례 + `…지원하게 됐습니다`로 닫는다. 사례는 **도메인 일치면 도메인 사례**, **비일치면 역량 사례**(UX·성능·에러 등) 무엇이든 "나 이거 해봤어요"면 성립.
  - 단락3(역량)·4(입사후)·5(목표)는 그대로.
- 사용자 논거(2026-07-02): "내 사례기반 일치가 있어도 '공감했고 지원했다'면서 '나 여기서 해봤어요' 어필 충분히 할 수 있다. 굳이 단락 경우의 수 2개 만들고 싶지 않다."

### 현재상태 (필수 — 위반 위치, 재현 가능)

**혼란 발생 세션**: 2026-07-02, 앤톡(antock) 프론트엔드 서류지원 중. AI가 도메인 비일치라 판단해 **합본 4단락**으로 단락1 시안을 냈고, 단락1이 "…지원하게 됐습니다"로 끝나자 사용자가 "단락1인데 왜 지원이 붙지? 단락1=공감, 단락2=지원 아니냐"며 혼란. 구조 경우의 수 2개가 원인.

### 현재 생각중인 방법 (필수 — 작성 세션 의견, 맹목 적용 금지)

- **본문 스킬(motivation-criteria.md)**: 위 L8·26·46·55·99-103·305·322에서 "4단락/합본/customer 공감" 분기를 제거하고 5단락 단일 규칙으로 교체. 노하우 1-2는 "도메인 비일치여도 단락2를 **역량 사례**로 채워 5단락으로 간다"만 남긴다(합본 옵션 삭제). L46(합본 안내 괄호) 통삭제.
- **프리셋 이관(단순 삭제 금지)**: 합본 3블록을 5단락 분리형으로 재작성한다. 각 합본은 공감+지원이 한 문단에 뭉쳐 있으니, **문단1 = 공감(…공감이 됐습니다로 끝)**으로 자르고, **문단2 = 본인 역량 사례 + 지원**을 붙인다. 문단2 역량 사례는 이미 대응 competency 블록이 있으니 그걸 참조:
  - legalcare → `competency.md` ai 변형 A(출처 `plang·legalcare·fitt 공통`)
  - bside → `competency.md` error-handling 캐넌(출처 `bside·rowilab 공통`)
  - in4u → `competency.md` ai 변형 B(출처 `morulive·in4u 공통`)
- **분해 참조 모델**: 이미 5단락 분리형인 프리셋 2개를 골격으로 미러링 — `empathy.md`의 **ux(fitt·krm)**(문단1=UX 가치관 / 문단2=꼼꼼함 사례+지원)와 **revenue-impact(umgtech)**(문단1=관점 / 문단2=본인 사례+지원).
