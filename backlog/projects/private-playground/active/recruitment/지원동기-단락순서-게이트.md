# 지원동기 작성: 단락 1·2 확정 전 단락 3 진입 방지 게이트

## 동기

지원동기 시안 작성에서 AI가 단락 순서(공감 1·2 → 역량 3 → 입사후 4 → 목표 5)를 건너뛰어, 사용자가 매번 순서를 교정하게 만든다. motivation-criteria.md에 순서 절차는 이미 있으나 가드가 약해 실제로 위반이 발생했다.

## 현재상태 (위반 사례 — 재현 가능)

- **발생 세션**: 2026-07-01, 마이일타(일타조교) 프론트엔드 서류지원 진행 중.
- **위반 흐름**: AI가 공감 축 전략(edutech ①급, "강사 의존이 콘텐츠 병목" + 본인 "강사 비의존 콘텐츠 라인" 경험)을 **텍스트로 길게 제안**한 뒤, 단락 1·2 시안을 내서 사용자가 고르게 하지 않고, 곧장 **단락 3(역량 ai vs error-handling) 택1**을 `AskUserQuestion`으로 물었다.
- **사용자 교정**: "왜 단락1,2부터 안정하고 단락 3부터 정하는거야?"
- **근본 혼동 지점**: AI가 "공감 뿌리를 전략으로 제안한 것"을 "단락 1·2가 확정된 것"과 동일시했다. 뿌리 제안 ≠ 시안 선택 확정. 이 혼동 때문에 단락 1·2 시안 선택 단계를 통째로 건너뛰고 역량 축 결정을 먼저 물었다.
- **기존 룰 위치**: `motivation-criteria.md`
  - "시안 여러 개 작성 → 확정 후 정리" 절: "단락을 순서대로 한 그룹씩 확정한다. … 2. 공감(단락 1·2) 시안 → 사용자가 1개 선택 / 3. 경험(단락 3) 시안 → 선택" 으로 순서가 이미 명시돼 있음.
  - 노하우 14: "공감/동기 단락(1·2)은 시안을 여러 개 뽑기 전에 뿌리를 한 문답으로 먼저 확정" — 뿌리 확정을 요구하지만, "뿌리 텍스트 제안 = 단락 확정"으로 오독될 여지를 막는 문장은 없음.

### 현재상태 (위반 사례 2 — 더 심한 형태, 재현 가능)

- **발생 세션**: 2026-07-01, 헤리트(HERIT) 에너지 데이터 시각화 플랫폼 프론트엔드 서류지원 진행 중.
- **위반 흐름**: 위반 사례 1(뿌리 제안 후 단락 3 먼저 물음)보다 더 심하게, AI가 **단락별 확정·뿌리 문답을 전부 건너뛰고 공감 1·2 + 역량 3 + 입사후 4 + 목표 5의 풀 5단락 지원동기를 한 번에 완성해 투척**했다.
- **사용자 교정**: "뭐지? 스킬 잘못 읽었어? 단락 1,2부터 나랑 논의하면서 정하는 거 아냐?"
- **근본 원인**: `steps/initial-recruitment.md` Step 6가 "5단락 양식으로 작성해 사용자에게 전달"이라고만 적혀 있어, `motivation-criteria.md`의 "시안 여러 개 작성 → 확정 후 정리"(단락별 한 그룹씩 확정) 절차로 진입하라는 강제력이 없음. 포인터만 있고 게이트가 없는 게 사례 1·2 공통 뿌리.
- **해결방향 (내 의견)**: 위 14-2 가드에 더해, `steps/initial-recruitment.md` Step 6에 `[CRITICAL] motivation-criteria.md를 Read한 뒤 단락별 확정 절차로 진행` 마커를 박아 풀세트 일괄 작성을 원천 차단. (자세한 메타 진단은 같은 폴더 `지원동기-공감단락-프레이밍.md` 참조.)

## 채점 기준 (규칙이 작동했다면 나왔어야 할 모습)

- 공감 뿌리 전략을 제안한 직후, **단락 1·2 시안(보통 각도 다른 2개)을 제시하고 사용자 선택을 받는다.**
- 단락 1·2가 사용자 선택으로 확정되기 전에는 단락 3(역량 축) 선택을 묻지 않는다.
- 위반의 반대(단순히 "순서를 지킨다")로는 부족 — "뿌리 전략 제안"과 "단락 1·2 시안 확정"을 별개 단계로 분리해 인지하는 게 채점 포인트.

## 현재 생각중인 방법 (해결방향)

- `motivation-criteria.md` 노하우 14에 하위 항목 **14-2** 신설:
  > **14-2. "공감 뿌리 전략 제안"과 "단락 1·2 시안 확정"은 다른 단계다.** 뿌리(왜 이 회사·도메인)를 전략으로 제안하는 것만으로 단락 1·2가 확정된 게 아니다. 반드시 단락 1·2 시안을 내고 사용자가 고른 뒤에 단락 3(역량)로 넘어간다. 뿌리를 텍스트로 제안만 한 채 단락 1·2 시안 선택을 건너뛰고 역량 축(단락 3) 결정을 먼저 묻지 말 것.
  > - ❌ before: [공감 축 전략 설명] → 곧장 "단락 3을 ai로 갈까요, error-handling으로 갈까요?" (AskUserQuestion)
  > - ✓ after: [공감 시안 A·B 제시] → 사용자 선택 → 그다음 단락 3 시안
- 대안 검토: 정적분석·hook으로는 강제 불가(대화 흐름 판단이라 텍스트 룰이 맞음). 규칙 보강이 적합.
- 효과 측정: 규칙 추가 후 위 위반 사례(공감 전략 제안 직후 역량 축 먼저 묻기)를 입력으로 ablation 벤치 가능 — 채점 기준은 위 「채점 기준」 사용.

## 종료 조건

`motivation-criteria.md`에 14-2(또는 동등 가드)가 반영되고, before/after 사례가 함께 박히면 이 항목 삭제.
