# 태그 기반 프리셋 discovery를 스킬 전체 컨텍스트 자산으로 확대할지

> PP = `~/WebstormProjects/main/private-playground/`. 경로는 PP 레포 상대.

## recruitment 부분은 반영 완료 (2026-07-02)

축-분리 오판(`ux`를 역량 축으로 찾다 empathy.md `## ux` 블록 놓침, 앤톡 세션)은 `presets/README.md`에 **겸용 컨벤션**으로 해결:

- "한 태그가 두 축을 겸할 수 있다 — 블록은 한 파일에만 두고 다른 축 표에서 크로스 참조" 규칙 신설.
- 첫 사례로 `ux`를 공감·역량 양쪽 표에 등재(블록은 empathy.md, 역량 표는 위치 크로스 참조).

**B안(개념당 파일 1개로 `empathy.md`·`competency.md` 전량 재편)은 보류.** 실패 1건 근거로 전량 해체할 실익 없음. 겸용 컨벤션이 재발을 막으면 파일 구조 변경 불필요.

## 남은 미결 (이 항목이 답할 것)

태그 기반 discovery를 **recruitment 프리셋을 넘어 다른 스킬 컨텍스트 자산(KA·qna 등)에도 일반 패턴으로 도입할지.**

- 현재 근거는 recruitment 실패 1건뿐 → 컨벤션이 recruitment 안에서 먼저 검증돼야 확대 판단 데이터가 쌓인다.
- 검증 안 된 패턴을 스킬 전역에 먼저 이식하는 건 비용 대비 위험. **recruitment에서 굴려보고, 두 번째 겸용 사례가 실제로 나오면 그때 확대 검토** 권장.

## 종료 조건

태그 기반 discovery의 스킬 전체 도입 여부가 결정되면 이 항목 삭제.
