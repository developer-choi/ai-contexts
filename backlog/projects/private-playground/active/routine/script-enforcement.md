# 데이터 파일 직접수정 대신 스크립트 강제

## 배경

2026-06-25 루틴 세션 중, `removeActionFromRoutines` 스크립트가 존재함에도 AI가 `data/users/1/routines.json`을 직접 Edit 도구로 수정함. 사용자가 "스크립트가 없어?" 라고 지적해 스크립트를 사용하게 됨.

## 탐구할 것

AI가 `data/` 하위 파일을 수정하려 할 때, 대응하는 스크립트(`scripts/`)가 있으면 직접 수정 대신 스크립트를 쓰도록 유도·강제하는 방법.

후보 방법들을 토론해서 실효성 있는 안을 고른 뒤, 규칙으로 적거나 hook으로 걸고, 벤치마크(위반 사례 재현 → 적용 후 차단 여부 확인)까지 돌린다.

### 논의 축

- **CLAUDE.md 룰**: "data/ 하위 파일 수정 전 scripts/에 대응 스크립트 있는지 확인, 있으면 스크립트 사용" — 단순하지만 AI가 쉽게 건너뜀
- **PostToolUse Write hook**: Edit/Write 도구가 `data/` 경로를 건드리면 `scripts/`에 해당 파일을 다루는 스크립트가 있는지 체크, 있으면 경고 출력 — hook이 실행을 막진 않지만 경고가 AI 다음 턴에 영향
- **data/ 디렉토리에 AGENTS.md/README**: "이 파일들은 scripts/ 를 통해서만 수정" 문구 삽입 — AI가 파일 읽을 때 맥락으로 잡힘
- **파일명·마커 규칙**: `data/users/` 같은 경로에 `.script-only` 빈 파일을 두고 CLAUDE.md에서 해당 마커가 있는 폴더는 직접수정 금지로 처리

### 벤치마크 계획

재현 시나리오: "routines.json에서 액션 1779485730024 제거해줘" 라고 요청 → AI가 직접 Edit하는지 vs 스크립트로 안내하는지 기록.

측정: 적용 전/후 각 5회, 스크립트 사용 비율.
