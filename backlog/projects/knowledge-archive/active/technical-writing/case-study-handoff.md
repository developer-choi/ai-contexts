# 인수인계 — 기술 케이스 스터디 글 양식 설계

## 현재 상태

기술 블로그 글 양식 두 개(루트/하위)의 시안 작성 완료. 양식 합의됨. 양식의 최종 위치 이관·가이드 정리는 미수행.

## 풀던 문제

사용자가 자기 기술 경험을 글로 정리하는 양식이 필요. 면접에서 쓰던 검증 흐름(문제·목표·성과 → 해결 → trade-off → 대안)을 글에 옮기되, Q&A 톤은 빼고 글다운 톤으로. 한 사례에 목표가 여러 개일 때 어떻게 구조화할지가 핵심.

## 참고 자료

- `deploy/contexts/self-help/roadmap.md` — 5단계 경험 재구성 양식 (카카오엔터테인먼트 블로그 + 대기업 기술면접 가이드 출처). 이론적 골격.
- `https://github.com/developer-choi/monorepo-playground/pull/12` — Sentry 도입 PR. 양식이 실제 글로 작동한 모범 사례.
- `write-init` 스킬의 `blog-post` 타입 — 기존엔 일반 변환용. 케이스 스터디는 별도로 분리하는 게 적합.

## 대화 진화 (요약)

1. 면접 흐름을 글 양식으로 그대로 옮기면 어색함 — Q&A 톤·5-6단계 깊이는 글에 안 맞음. 검증 항목(체크리스트)만 가져오기로.
2. 단일 양식 6단계 (문제 / 목표 / 해결방법 / 선택 / 구현 / 성과)로 시작.
3. 한 사례에 목표가 여러 개면 분리 필요. **글 1개 = 목표 1개**가 인지 부담 단위에 맞음.
4. **모델 A (얕은 인덱스 + 깊은 편)** 확정. 인덱스는 지도, 각 편이 본문.
5. 명칭 확정: **루트 양식 / 하위 양식**.
6. 루트 양식 세부 합의:
   - 표 1개로 합치는 안 → 거부 (나중에 분리하기 힘듦). 섹션 분리.
   - 후보 비교 흔적을 루트에도 노출 (사용자 의도: "이만큼 고민함"을 보여주기).
   - 해결방법과 구현 핵심을 한 블록에 합침 (요약본이라 분리하면 너무 짧음).
   - Before/After 슬롯 X — Before는 1번에 이미 있고 After=성과니까 별도 성과 섹션으로.

## 합의된 양식

### 루트 양식 (인덱스 글)

```
1. 문제 발견 (사례 전체 Before + 근본 원인 + 임팩트)
2. 목표 정의 N개 (각 [→ 하위 글] 링크)
3. 해결 결과 (목표별 한 블록)
   - 선택한 해결방법
   - 검토했던 후보들 (bullet, 한 줄씩 탈락 이유)
   - 구현 핵심
   - [→ 하위 글 비교 섹션 링크]
4. 성과 (목표 순서대로 한 줄씩 + 사례 전체 성과 맨 끝)
```

### 하위 양식 (한 편 글)

```
1. 문제 발견 (이 목표로 좁힘)
2. 목표 정의 (1개)
3. 해결방법 리스트업 + Trade-off (자세히)
4. 선택 및 근거
5. 구현 과정
6. 성과
```

상단에 시리즈 목차(인용구 박스).

### 운영 룰

- 다른 편과 패턴이 겹치면: 한 글이 canonical(원전), 다른 글은 짧게 + 링크.
- 코드 스니펫은 강조점이 다르면 자연스럽게 갈라짐. 핵심 메커니즘 정의 자체는 원전 한 곳.
- trade-off 분류: 성능 / 복잡도 / 유지보수 비용 / 학습 비용 / 운영 부담 / 확장성.
- 루트의 4번 성과 항목 순서는 2번 목표 순서와 동일하게 정렬.

1. `case-study-template.md` 처리 (삭제 / 보존)
2. 양식 시안의 최종 위치 결정

   - 후보 B: `write-init` 스킬의 `templates/`에 등록
   - 둘 다 가능 (가이드 + 템플릿 페어링)
3. `write-init` 스킬에 새 type 추가 결정
   - type 이름 후보: `case-study` / `tech-experience` / `problem-deep-dive`
   - subtype 필요? (root / sub 두 양식이라 subtype으로 분기 가능)
4. 양식 가이드 문서 작성 (각 섹션의 메타 가이드를 별도 가이드 문서로 추출)
5. `self-help/roadmap.md`와의 관계 정리 — 면접 답변 준비용은 그대로 두고, 글 양식만 writing-guide로 분리
