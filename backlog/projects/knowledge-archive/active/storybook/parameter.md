<!-- from PDF p.1 -->
**Storybook 파라미터(Parameters) 한 줄 요약**

"스토리북의 기능과 애드온(addon)의 동작 방식을 설정하는 정적인 데이터"라고 생각하시면 가장 쉽습니다.

**조금 더 자세한 설명** (게임으로 비유)

- `Args` (지난번에 배우셨던 것): 게임 캐릭터의 외형(머리색, 갑옷 등)을 바꾸는 것과 같습니다. 컴포넌트 자체의 `props`를 바꿔서 컴포넌트의 모습을 직접적으로 바꿉니다. 사용자가 UI에서 실시간으로 조작할 수 있죠.
- `Parameters` (파라미터): 게임의 환경 설정(그래픽 품질, 난이도, 배경음악 볼륨 등)과 같습니다. 캐릭터 자체를 바꾸는 게 아니라, 스토리북이라는 환경과 그 도구(애드온)들이 어떻게 동작할지를 설정합니다. 보통 코드에 미리 값을 정해둡니다.

<!-- from PDF p.2 -->
**파라미터로 주로 하는 일들**

- 배경색 바꾸기: `backgrounds` 애드온의 기본 배경색이나 선택 가능한 색상 목록을 설정합니다.

```javascript
parameters: {
  backgrounds: { default: 'dark' },
}
```

- 화면 크기 조절하기: `viewport` 애드온에서 사용할 모바일, 태블릿 등 기기 목록을 미리 설정합니다.

```javascript
parameters: {
  viewport: { defaultViewport: 'iphone6' },
}
```

- 특정 애드온 비활성화하기: 이 스토리에서는 Controls 애드온을 보여주고 싶지 않을 때 사용합니다.

```javascript
parameters: {
  controls: { disabled: true },
}
```

**`Args` vs. `Parameters` 핵심 차이**

| 구분 | Args (인자) | Parameters (파라미터) |
|------|------------|----------------------|
| 목적 | 컴포넌트의 `props` 제어 | 스토리북 환경/애드온의 동작 방식 설정 |
| 성격 | 동적 (Dynamic) - UI에서 실시간 변경 | 정적 (Static) - 코드에 미리 정의 |
| 영향 | 컴포넌트의 모습과 내용 변경 | 스토리북의 기능과 UI 변경 |

<!-- from PDF p.3 -->
**알아두면 대박, `parameters` 객체 안에 들어갈 수 있는 키(key)와 그 구조는 모두 사전에 정의되어 있습니다.**

조금 더 구체적으로 설명하면:

- 누가 정해주나요?: 파라미터는 Storybook의 핵심 기능(예: Docs 에디션, Controls 애드온의 동작 방식)과 관련한 파라미터가 미래 정의됩니다.
- 각각의 애드온(Addon): 대부분의 파라미터는 특정 애드온에 해당 파라미터 이름으로 묶여 있습니다.
  - `@storybook/addon-backgrounds` 애드온은 `parameters.backgrounds` 객체를 읽어서 배경색을 관리합니다.
  - `@storybook/addon-viewport` 애드온은 `parameters.viewport` 객체를 읽어서 뷰포트를 설정합니다.
  - `@storybook/addon-actions` 애드온은 `parameters.actions` 객체를 읽어서 이벤트 핸들링 등을 동작하게 합니다.

2. 동적 성격: 각 애드온마다 요청한 요소만 "그룹"하고 실시간으로 설정이 됩니다. 그러나 스토리북은 스토리에서 접할 때 `parameters` 객체를 설정된 원하는 파라미터 값을 반영합니다. 그러나 각 애드온은 그 객체를 통해 그 설정들을 가져오고 필요한 설정들을 실시간으로 반영하며 자신이 어떤 파라미터를 수행할지 각 애드온의 라이브러리 자체에서 확인하고 사용됩니다.

사용자가 어떤 애드온을 설정하고 사용하느냐에 따라 `parameters` 에 넣을 수 있는 데이터의 종류 및 범위 및 필드 값이 모두 달라집니다. 따라서 설정한 애드온의 특성 및 사용 가능한 `parameters` 의 종류 및 범위가 커집니다. 그리고 자신만의 한도 규칙이 존재하기도 합니다. 설정된 파라미터에 대한 정보는 각 라이브러리의 공식 API 문서에서 확인하고 사용하는 것이 좋습니다.

그래서 정리하면: "어떤 파라미터를 써야해" 등의 질문에 대한 답은 항상 "어떤 애드온을 위한 것인가?"를 공식 문서에서 확인하고 사용하는 것이 중요합니다.
