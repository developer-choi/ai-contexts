# 인증 폼·플로우 UX 원리

회원·인증 도메인 필기에서 추출한 폼·플로우 UX 원리. KA 본체로 graduate 전 한국어 미검증 정리.

## 다단계 인증·찾기 폼은 순차 게이트로 만든다

이전 단계를 통과해야 다음 폼이 노출되고, 이전으로 돌아가려면 처음부터 다시 하게 한다. 모든 폼을 처음부터 동시에 노출하면:

- 입력 도중 이전/이후 단계를 오가며 진행하는 경우의 예외처리가 어렵다 (예: 아이디를 입력하고 이메일 인증번호까지 발송해놓고 아이디를 바꾸면?)
- 선행 검증(아이디 존재 확인 등)을 통과하지 않은 상태에서도 다음 단계(이메일 인증번호 발송 등)를 진행할 수 있는 구멍이 생긴다

## 로그인 유도 시 return url을 물고 간다

로그인이 필요한 동작(좋아요 등)에서 로그인 페이지로 보낼 때 return url을 지정하고, 회원가입으로 이어지더라도 return url을 계속 물고 가서 가입·로그인 성공 후 원래 페이지로 복귀시킨다.

- 복귀는 `replace`로 처리한다. `push`면 복귀 후 뒤로가기 시 인증·비번재설정 페이지로 되돌아가 불편하다.
