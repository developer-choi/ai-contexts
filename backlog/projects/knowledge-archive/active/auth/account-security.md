# 계정 인증 보안 원리

회원·인증 도메인 필기에서 추출한 일반 보안 원리. KA 본체로 graduate 전 한국어 미검증 정리.

## 개인정보 핵심값은 입력받지 말고 외부 인증으로 받는다

생년월일을 `<input>`으로 받아 그 값으로 성인 여부를 판단하는 게 의미가 있나? 성인 여부 판단 같은 핵심 정보는 본인인증 등 외부에서 제공받아야 한다.

## 비밀번호 변경 경과시간 체크값은 서버에 저장한다

비밀번호를 변경한 지 얼마나 됐는지 체크하는 값은 보안 목적이므로, 어느 기기로 로그인하든 일관되게 체크돼야 한다. 클라이언트(브라우저)에 저장하면 기기마다 값이 달라지므로 서버에 저장한다.

## 회원 재확인 여부는 memberId가 아닌 accessToken과 비교한다

로그아웃 후 같은 계정으로 재로그인하면 memberId는 동일해서 "재확인 안 함" 상태로 리셋되지 않는다. accessToken은 로그인할 때마다 달라지므로 재확인 상태가 올바르게 리셋된다.

부수효과: 이렇게 하면 로그아웃 시 재확인 쿠키 삭제를 빠뜨려도 기능 자체엔 문제가 없다.

## SNS 재인증은 로그인 성공 + 계정 식별자 일치까지 확인한다

SNS 로그인 성공만으로 재인증을 통과시키면, 브라우저에 타인(지인·부모 등)의 SNS 계정이 로그인돼 있을 때 그 계정으로 재인증이 통과되는 구멍이 생긴다. 로그인 성공한 계정의 식별자와 기존 로그인된 계정의 식별자가 일치해야 통과시킨다.
