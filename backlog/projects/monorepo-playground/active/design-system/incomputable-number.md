# IncomputableNumber

연산 불가능한 숫자 인풋 위젯. 전화번호·계좌번호·숫자 비밀번호처럼 계산에 쓰이지 않는 숫자값을 입력받는 케이스.

## Usage

1. 연산할 수 없는 숫자값을 입력받고 싶은 경우 사용 (휴대폰입력, 계좌번호입력, 숫자로된 2차비밀번호 입력)
2. 사용자가 잘못 입력한 경우 (숫자가 아닌값), 인터렉션(입력)을 막는 방식으로 구현하고 싶은 경우 사용 (a 누르면 아무런반응없음)
   - 인터렉션을 허용하고싶다면, 적절한 (submit, change, blur) 시점에 유효성검증과 에러메시지로 대체하기

## Features

숫자 관련 모든값 입력 불가능

1. 하이픈 (마이너스), 플러스, dot (소수점표기), e, E (큰숫자표기법),
2. 콤마, 스페이스도 입력 불가능

마스킹 처리 지원해야. (통장비밀번호 입력하는 케이스)

paste를 했을 때 버그만 안생기면됨. 아예 복붙을 막던가 (네이버 페이는 복붙 자체를 막았음)  
복붙하더라도 직접 입력시킨것처럼 동작시키던가. (123asd456asd789 복붙하면 123456789 되게 하던가)

모바일 키패드는 적절한 종류의 숫자로 올라와야함.

- 전화번호면 tel, 그 외 나머지는 numeric이어야함.
- **마스킹 처리로 되었다 하더라도 키패드는 numeric으로 올라와야함.**
- 안그러면 모바일 키패드에 비밀번호 추천되어 올라옴
- inputMode에 password는 없지만, type이 password면 휴대폰 키패드 위에 비밀번호 관련된게 올라옴.
