# Autocomplete

## 공통 동작

### 자동완성 목록을 띄울 수 있는 방법

1. 입력박스 마우스클릭
2. 입력박스에 (탭으로) 포커스대고 방향키 위아래 누르기
3. 입력박스에 (탭으로) 포커스대고 뭔가 입력

### 첫 항목 자동 탐색 기능

사용자가 가장 빠르게 의도하는 항목을 선택할 수 있어야 하기 때문에 필요.

동작 흐름:
1. "Ama"를 입력하면
2. 자동으로 제일 위에 Amadeus가 필터링되어 있고
3. 엔터만 누르면 선택

핵심: 위아래 방향키 탐색, 스크롤 탐색 없이 원하는 항목을 선택.

#### 검색할 때

입력하는 경우 제일 먼저 매칭되는 목록을 자동으로 선택. 이 상태에서도 아래방향키로 다른 항목 탐색 후 엔터로 선택 가능.

#### 최초 데이터 로딩 시점

Loading… 이 끝날 때까지 포커스를 유지하고 있으면, 로딩 완료 후 목록이 노출되면서 첫 번째 항목 자동선택.

#### 처음에 띄울 때

클릭하거나 포커스 대고 방향키 눌러서 목록 띄우는 경우, 자동으로 첫 번째 항목 선택.

## 요구사항별 구현

### Select 폭

요구사항: 선택 가능한 option 중 가장 폭이 넓은 것에 맞춰져야 함 — option 선택 시 잘린 채로 표시되어서는 안 됨.

판단: 정말로 가장 긴 항목에 맞춰 가로길이를 설정하면, Select 박스 2개를 가로로 나란히 펼쳤을 때 목록이 우측영역을 침범하는 문제가 생김. MUI와 동일하게 줄내림 처리로 구현.

### hover / focus 스타일

- 짙은 회색 = 위아래 방향키로 **탐색** 중인 항목
- 하늘색 = **선택**한 항목
- 옅은 회색 = 마우스 hover

### 목록 활성화 방법

클릭 / 방향키 / 입력 3가지 방법으로 목록 띄우기 가능.

#### 클릭하거나, 탭으로 포커스 대고 방향키 누른 경우

데이터 패칭 상태에 따라 두 가지 중 하나 발생:
- 목록 즉시 노출
- Loading… 표시 후 노출 (options props에 function 전달한 경우에만)

#### 탭으로 포커스 대고 입력한 경우

안 떠있는 상태에서 입력 시 즉시 목록 생성. 데이터 상태에 따라 세 가지 중 하나 발생.

#### options는 함수도 가능해야 한다

```ts
// 타입: options props에 function 타입 허용
type Options = OptionItem[] | (() => Promise<OptionItem[]>)
```

props로 함수를 전달하면 컴포넌트 안에서 함수로 데이터를 불러와 옵션 목록을 그림. 데이터 가져오는 동안 로딩 중 표시.

#### 검색 (실시간 필터링)

MUI와 동일하게 한 글자 한 글자 바꿀 때마다 목록 최신화.

- 아무것도 입력 안 된 상태에서 입력 시작할 때도 동작
- 이미 목록이 떠있는 상태에서 마저 입력하는 경우에도 지속 동작

### 포커스 이탈 시 검색어 삭제

항목을 선택하지 않고 바깥영역 클릭 시 검색어 삭제.

반대로 항목을 선택해 놓고 포커스를 빼면 값 유지.

예외: 항목을 선택하지 않았어도 자동완성 목록에 있는 값을 완전히 일치하게 직접 입력한 경우에는 검색어 삭제하지 않음. "option을 선택하지 않고"라는 요구사항에는 어긋나지만, 사용자 입장에서 굳이 삭제할 이유가 없기 때문.

### 상하 방향키 탐색 및 스크롤 이동

동작:
1. 아래로 가기
2. 위로 가기
3. 첫 번째 항목에서 위로 가면 마지막으로
4. 마지막 항목에서 아래로 가면 처음으로

방향키 이동 중 스크롤이 따라감.

### 클릭 또는 엔터로 option 선택

마우스 클릭 또는 화살표 위아래로 항목에 맞춰 놓고 엔터 → 선택.

### 선택된 option 복원

목록을 다시 열 때:
1. 선택한 항목이 시야에 노출될 수 있는 위치까지 스크롤
2. 방향키 탐색 시 선택된 항목이 시작 지점

### 공간 부족 시 위쪽 노출

우선순위:
1. 하단 공간이 넉넉한지 체크 → 하단에 노출
2. 상단 공간이 넉넉한지 체크 → 상단에 노출
3. 둘 다 안 넉넉하면 하단에 노출 + 높이 줄여서 맞추기 (= MUI와 동일)

"넉넉하다"의 기준: 자동완성 목록의 높이값을 JS로 계산. CSS로 높이가 변경돼도 지장 없음. 기본 높이 스타일은 MUI Autocomplete와 동일하게 CSS 540px.

상하단 둘 다 안 넉넉한 경우, 하단 영역을 뷰포트 가장 밑에까지만 높이를 잡음.

## 그 외 상세 로직

### 방향키 탐색 시 목록 스크롤 따라가게 하는 방법

#### 아래 방향 (bottom align)

현재 선택 항목이 시야 아래로 벗어났는지 체크:

```
item의 clientHeight + item의 offsetTop > list의 clientHeight
```

스크롤을 아래로 얼마나 내릴지:

```
item의 clientHeight + item의 offsetTop - list의 clientHeight
```

#### 위 방향 (top align)

현재 선택 항목이 시야 위로 벗어났는지 체크:

```
list의 scrollTop > item의 offsetTop
```

스크롤을 위로 얼마나 올릴지:

```
item의 offsetTop
```
