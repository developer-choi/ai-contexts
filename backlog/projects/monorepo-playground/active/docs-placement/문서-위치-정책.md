# 문서 위치 정책 수립 + 기존 문서 일괄 재배치

## 동기

MP 안에 문서 디렉터리가 세 곳인데 "무엇이 어디로 가는가" 정책이 없어 문서가 섞여 들어가고 있다.

- 최상위 `docs/`
- `apps/examples/docs/`
- `packages/design-system/docs/`

실제로 어긋난 사례:

- design-system 전용 결정문(`docs/decisions/inputbase-responsibility.md`, `inputbase-focus-delegation.md`)을 최상위 `docs/decisions/`에 넣었다. design-system 전용이면 `packages/design-system/docs/`가 맞는지 검토 필요.
- design-system 전용인 `docs/convention.md`도 최상위 `docs/`에 있다(프로젝트 CLAUDE.md가 "design-system 작업 시 docs/convention.md를 따른다"고 가리키는 그 파일).
- 반대로 `apps/examples/`로 가야 할 예제 직결 문서가 최상위 `docs/`에 섞여 있는 것으로 보인다(전수 점검 대상).

## 할 일

### 1. 세 곳의 역할·배치 정책 확정

각 디렉터리가 무엇을 담는지 경계를 정한다.

- 최상위 `docs/` — 레포 전역 공통(정적분석·포매터·전역 컨벤션·인덱스 등)
- `apps/examples/docs/` — 예제 앱에 직결된 문서
- `packages/design-system/docs/` — design-system 패키지 전용(컴포넌트 설계 결정문·convention 등)

위는 초안 가설일 뿐, 확정 시 재검토.

### 2. 기존 문서 일괄 재배치

정책 확정 후 현재 흩어진 문서를 한 번에 올바른 위치로 옮긴다. 후보:

- `docs/decisions/inputbase-*.md` → `packages/design-system/docs/` 계열로?
- `docs/convention.md` → `packages/design-system/docs/`로?
- 최상위 `docs/`에 섞인 예제 직결 문서 → `apps/examples/docs/`로?

## 주의 — 이동 시 참조 경로 전수 업데이트

문서를 옮기면 그 문서를 가리키는 모든 참조가 깨진다. **`WebstormProjects/*` 하위 전체를 뒤져** 경로 참조를 갱신해야 한다.

갱신 대상 예:

- `docs/best-practices-map.md`의 링크
- 각 `CLAUDE.md`의 `[CRITICAL]` 경로·문서 진입점 링크
- 스토리북에 박은 GitHub blob URL — 예: `InputBase.stories.tsx`의 `blob/master/docs/decisions/inputbase-*.md` 링크(문서 이동 시 함께 갱신)
- 다른 레포(AC backlog 등)에서 MP 문서를 가리키는 링크
