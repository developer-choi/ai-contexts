## Timer Required

1. 원하는 포맷으로 일시분초를 노출 (100% 커스텀가능)

2. 현재시간 기준 잔여 시간을 1초마다 까면서 노출

3. 종료됐을 때 실행될 콜백함수를 제공한다.

4. 종료날짜가 미래인경우
   1) 시간초 까여야함.
   2) 다까이면 종료콜백함수 호출되야함.

5. **중간에 종료날짜 바뀌면 리셋되야함**
   - Period Timer는 실제 사례까지는 생각이 안남
   - Diff Timer는 이미 인증번호 타이머 까이고 있는데 (아직 종료안된상태) 재전송 누른 케이스

## Period Timer Required

1. 종료날짜가 과거인경우
   - **바로 종료콜백함수 호출되야함. (즉시)**
   - 시작날짜가 미래인 경우에는 당연히 종료콜백이 실행되면 안됨.

2. **시작날짜가 미래인 경우, 시작을 안해야함.**

## Diff Timer Required

1. start() / initialize() 실행되는 과정에서 불필요한 리렌더링이 발생하면 안됨.
   - start() 했더니 막 따닥 하면서 리렌더링 순식간에 두세번씩 일어난다거나…

2. proceeding 상태에서 또다시 start() 하면 재시작되야함. (인증번호폼에서 재전송하는 케이스)
