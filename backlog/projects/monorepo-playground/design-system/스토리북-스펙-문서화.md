# 테스트로 표현 못 하는 스펙은 스토리북 설명으로

테스트코드로 검증·표현할 수 없는 컴포넌트 스펙(브라우저 네이티브 동작, 설계 의도·금지 사항 등)은 별도 패턴 문서가 아니라 **스토리북 설명**(`tags: ['autodocs']` + `parameters.docs.description.component/story`)에 넣는다.

- 선례 (2026-06-12): InputBase 슬롯 포커스 동작(WHATWG label 위임, preventDefault 금지) 명세를 `packages/design-system/docs/patterns/InputBase.md`로 만들었다가 삭제하고 InputBase.stories.tsx의 docs.description으로 이동. autodocs는 컴포넌트별 opt-in.
- 적용 후보: 같은 성격의 스펙이 생기면 스토리북으로 — 예: Select placeholder의 disabled+hidden 빈 옵션 구현 방식, Radio의 context 대비 개별 prop 우선 규칙.
