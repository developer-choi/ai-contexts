## 공통 요구사항 (어느 페이지에서 인증폼이 노출되더라도)

### 휴대폰 번호 입력란

#### 편의성

휴대폰번호 입력 인풋박스에서 엔터 누르면 폼 제출이 기본값인데, 이를 가로채서 인증번호 전송 버튼이 눌리도록 처리 (+ 유효성검증).

#### 실패 케이스

- Client Side 유효성검증에서 걸린 경우
- 이미 가입된 휴대폰 번호인 경우

→ 모두 단순 에러메시지 표기

#### 예외처리: 10으로 시작하는 번호

해외에서 사는 유저가 주 유저층이 아닌 경우와 그 반대 케이스로 처리 방향이 달라짐.

- 010이 주 유저층인 경우: "10" 입력 시 입력값 자체를 "010"으로 바꾸는 것도 직관적임
- 10이 주 유저층인 경우: 입력값 자체를 바꾸는 건 별로

→ (폼 컨트롤) 10으로 유효성검증 다 돌리고, API 제출할 때에만 010으로 변환하는 방식으로 처리.

#### 성공

1. 인증번호 전송 버튼 텍스트 변경
2. 인증번호 입력폼 노출 + 포커스 이동
3. 타이머 시작

### 인증번호 제출란

#### 편의성

인증번호 입력 인풋박스에서 엔터 누르면 폼 제출이 기본값인데, 이를 가로채서 인증확인 버튼이 눌리도록 처리 (+ 유효성검증).

#### 실패 케이스

단순 에러메시지 표기:
- Client Side 유효성검증에서 걸린 경우
- 인증번호 틀린 경우

예외:
- 타이머 시간이 끝난 경우: 버튼 비활성화, 인증번호 재전송 유도

#### 성공

- 휴대폰번호, 버튼 disabled 처리
- 인증번호 폼은 disabled 처리 또는 미노출 처리

## 회원가입에서의 인증폼 요구사항

### 휴대폰번호 + 인증번호

#### 에러: 이미 등록된 휴대폰 번호

처음 상태에서 휴대폰번호만 입력하고 제출했을 때 [이미 등록한…] 뜬 상태와 동일하게 재현해야 함.

만약 인증번호까지 통과되어 인증번호폼이 사라지고, 휴대폰번호 입력란에 disabled가 걸려있는 상태라면, 다시 전부 풀어야 함.
