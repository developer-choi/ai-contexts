# 이벤트 상세페이지 — 응모 테스트케이스

## 접근 실패 시나리오

- 없는 이벤트 번호 (예: 999999번)
- 유효하지 않은 이벤트 번호 (예: a번)
- 노출설정을 하지 않은 이벤트

## 응모하기 버튼 클릭 → 응모폼 레이어 노출 조건

- 로그인 되어 있고
- 이미 종료된 상태가 아니고
- 이미 참여한 상태가 아니어야 한다

## 응모하기 실패 케이스

에러메시지를 보여줘야 하는 경우:

- 이미 종료된 이벤트 번호로 시도
- 이미 참여한 이벤트 번호로 시도
- 노출설정을 하지 않은 이벤트
- 이벤트 기간 > 요일설정에 해당되지 않는 경우

기타:

- 연속 클릭(광클) 시 처음 제출한 것만 제출

## 응모하기 성공 시나리오

- 모달이 뜬 경우 메모란으로 자동 포커스 이동
- 응모 시 관리자에 집계되는 것까지 확인
- 메모를 입력하지 않고 제출 / 메모를 입력하고 제출
