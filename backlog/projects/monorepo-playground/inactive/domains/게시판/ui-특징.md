# 게시판 UI 특징

본문페이지, 목록페이지, 본문페이지 > 댓글목록 모두 포함.

## 게시판 분류 & 특징

1. 특정 사람만(관리자) 쓸 수 있고, 모든 회원이 볼 수 있는 경우 — 공지사항, 이벤트
   1. 모든 사용자는 볼 수만 있음 (수정·삭제 UI 없음)
2. 특정 사람(관리자)과 특정 회원 한 명끼리만 읽고 쓸 수 있는 경우 — 1대1 문의
   1. 작성자에 한해서만 볼 수 있음
   2. 작성자는 생성자의 권한을 가짐 (수정·삭제 가능 ⇒ 관련 UI도 같이 구현되어야 함)
3. 모든 사용자, 혹은 인증된 사용자들이 읽고 쓸 수 있는 경우 — 게시판, 네이버 카페, 각종 커뮤니티
   1. 작성자인 경우 2의 (2)와 동일하게 구현
   2. 작성자가 아닌 경우 1의 (1)과 동일하게 구현

## 특징을 체득해야 하는 이유

[크리마 리뷰서비스](https://www.cre.ma/)를 비슷하게 구현하려고 분석하고 있었는데, 상품 후기는 누구나 작성할 수 있으므로 3번 게시판에 해당하는데, 내가 분석하는 과정에서 작성자일 때에 대한 UI를 확인하지 않았음.

또 다른 예시로, 기획서를 받았는데 게시글 상세페이지 하단 댓글목록을 보여달라고 기획서에 나와 있으나, 댓글 작성자와 로그인한 사용자가 같으면 해당 댓글을 삭제하거나 수정할 수 있는 UI가 반드시 기획서에 명시되어 있어야 하는데 없으면…. 검토가 필요함.

이런 실수를 반복하지 않기 위해 필기함.
