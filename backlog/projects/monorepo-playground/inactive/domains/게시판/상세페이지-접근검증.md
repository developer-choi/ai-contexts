# 상세페이지 접근 — PK 유효성검증·상태검증·에러처리

## PK값 유효성검증

애초에 **유효성검증에서 실패하면 API를 호출하면 안 된다는 것이 핵심**이다.

검증 케이스:

- 숫자가 아닌 경우 — 예: `/board/abc`
- 숫자이긴 하나 정상 형식이 아닌 경우
  - `/board/+9999`
  - `/board/-999`
  - `/board/09999`
  - `/board/1e+20` 같은 이상한 숫자 표현식
- 숫자이고 형식도 정상이지만, 해당 번호로 조회했을 때 게시글이 없는 경우

## 기타 게시글 상태 유효성검증

해당 PK의 게시글·상품이 **존재하지만** 미진열이거나 접근권한이 없는 상태 (유튜브 Visibility 같은 개념).

## 에러처리 방법

보통 기획은 두 가지다:

- 404 페이지를 바로 보여준다
- 빈 페이지에 시스템 alert 메시지(삭제되었거나 없는 게시물)를 띄우고 리스트 페이지로 보낸다
