# 회원 재확인 페이지

내정보 수정·회원탈퇴 등 민감 페이지 접근 시 유저 본인이 맞는지 재확인하는 페이지.

## 접근 프로세스

### (일반) 로그인됨 + 재확인 안 한 상태로 탈퇴페이지 접근

1. 탈퇴페이지 접근 시도
2. 재확인 페이지로 리다이렉트 (returnUrl = 탈퇴페이지)
3. 재확인 성공
4. 탈퇴페이지로 자동 이동
5. 이후 다른 페이지 갔다가 다시 탈퇴페이지 이동 시 재확인 없이 통과
6. 로그아웃 후 다시 로그인하면 재확인 프로세스 재반복

### (예외) 로그인 안 한 상태로 탈퇴페이지 접근

1. 탈퇴페이지 접근 시도
2. 재확인 페이지로 리다이렉트 (너무 빨라 재확인 페이지는 화면에 안 보여야 함, returnUrl = 탈퇴페이지)
3. 다시 로그인 페이지로 리다이렉트
4. 로그인 성공
5. 재확인 페이지로 리다이렉트
6. 재확인 성공 후 탈퇴페이지로 이동

### (예외) 로그인 + 재확인 모두 된 상태로 재확인 페이지 접근

- redirectUrl 쿼리스트링대로 원래 가려던 페이지로 바로 리다이렉트

## 재인증 수단 노출 규칙

일반(본인인증) 가입과 SNS 가입을 모두 지원하면 회원 데이터에 유저 타입이 있으므로:

- 일반회원 계정: [비밀번호 입력폼] + [SNS 로그인 버튼]
- SNS회원 계정: [SNS 로그인 버튼]만

## 재인증 동작

- 일반계정: 비밀번호 입력 후 통과 확인
- SNS계정: 해당 SNS 채널로 로그인 시 통과 확인
- 미연결 SNS로 로그인 시: "해당 채널로 가입된 정보가 없습니다" 노출
- 연결돼 있으나 기존 로그인 계정과 다른 계정인 경우: "기존에 로그인된 계정과 다른 계정입니다" 노출

## 구현방법

### 비밀번호 (일반회원)

비밀번호 입력 → 비밀번호 체크 API 호출 → 성공 여부로 판단.

### SNS 로그인 (일반회원+채널연결 또는 SNS계정)

1. 재확인 페이지에서 SNS 로그인 버튼 클릭
2. SNS 로그인 팝업에서 로그인 성공
3. 콜백 페이지로 리다이렉트
4. 전달받은 인가코드·상태토큰으로 로그인 API 호출
5. 로그인 실패 시 에러메시지 노출 후 팝업 닫음
6. 로그인 성공해도, 성공한 계정 식별자와 기존 로그인된 계정 식별자가 일치해야만 통과 (보안 근거는 KA `auth/account-security.md` 참조)
7. 일치하면 팝업 닫고 원래 가려던 페이지로 이동

## 재확인 여부 판단 기준

- 로그인 이후 한 번도 재확인 안 했으면 재확인 안 한 것으로 판단
- 로그아웃 후 같은 계정으로 다시 로그인해도 재확인이 필요해야 함
- 구현 시 재확인 여부는 memberId가 아닌 accessToken과 비교한다 (근거는 KA `auth/account-security.md` 참조)
