# 회원가입 페이지

## 폼 기본값 채워넣기

1. 나이스 본인인증 또는 SNS 로그인으로 유저정보를 받아 Session Storage에 임시저장
2. 회원가입 페이지로 이동 완료 후 폼 기본값 설정
3. 회원가입 페이지에서 Session Storage에 값이 없으면(중간 인증 페이지를 거치지 않고 직접 접근한 경우) 잘못된 접근 얼럿을 띄우고 메인페이지로 리다이렉트
   - 이 케이스는 임시 데이터 누락 전용 에러(예: `MissingTemporaryDataError`)로 잡아 에러처리한다

## 데이터 정확성 (SNS·본인인증)

회원가입의 기본은 데이터 정확성. 잘못된 데이터가 쌓이면 답이 없으므로 SNS 로그인·본인인증을 특히 조심한다.

- SNS마다 누락값이 다름 (정보제공 동의 항목에 따라, 아예 안 주는 SNS도 있음)
- 값 형태가 다름
  - 네이버는 birthday가 연월일로 들어옴
  - 페이코는 월일만 있음
  - 특히 휴대폰번호 형식이 다른 게 가장 치명적

## return url

로그인/회원가입으로 유도된 경우 return url을 물고 와, 가입 성공 후 해당 url로 복귀시킨다. (복귀 방식 등 일반 원리는 KA `auth/auth-form-flow.md` 참조)

## 폼 제출 실패 케이스

- 나이 제한: API에서 체크, 에러메시지 노출로 처리 (단, 재현이 어려워 테스트 방법 고민 필요)
- 기타 (에러메시지만 노출): 관리자 가입 승인 필요, 이미 가입된 계정 존재
