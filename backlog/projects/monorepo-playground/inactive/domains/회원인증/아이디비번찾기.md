# 아이디찾기 / 비밀번호찾기 페이지

## 개요

- 아이디찾기: 인증수단(SMS·이메일·본인인증 등)으로 아이디를 찾음
- 비밀번호찾기: 기존 아이디 + 인증수단으로 비밀번호를 찾음

## 아이디찾기 프로세스

1. 인증수단 진행 및 통과
2. 통과 후 찾기 API 호출
3. 성공하면 간단한 유저정보를 Session Storage에 저장
4. 아이디찾기 결과 페이지로 이동해 아이디 노출

## 비밀번호찾기 프로세스

1. 기존 아이디 입력란 노출
2. 아이디 존재 여부 체크 API 호출 (회원가입의 아이디 중복 체크 API와 기능이 유사)
   - 탈퇴한 아이디는 이때 조회되면 안 됨
3. 아이디 존재가 확인되면 인증수단 노출·진행 후 찾기 API 호출
4. 성공하면 중간 유저정보를 Session Storage에 저장하고 비밀번호 변경 페이지로 이동
   - 서버에서 보내준 토큰값을 비밀번호 변경 시 다시 보내줘야 연속성을 유지할 수 있음
5. 비밀번호 변경 페이지에서 변경할 비밀번호 / 확인을 입력받음
6. 유효성검증 후 "비회원 비밀번호 변경" API 호출
7. 성공하면 비밀번호 변경 결과 페이지로 이동

## 페이지 구성 (총 5개)

- 아이디찾기 → 아이디찾기 결과
- 비밀번호찾기 → 비밀번호 변경 → 결과
