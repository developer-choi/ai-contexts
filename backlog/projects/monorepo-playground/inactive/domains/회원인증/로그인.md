# 로그인 페이지

## 아이디 저장 기능

- 로컬스토리지에 저장 (아이디값 1개만 저장해도 됨 — 저장된 아이디가 있으면 "저장됨"으로 판단 가능)
- 공용 PC 경고 얼럿을 띄우는 서비스도 있으나 없는 곳도 있어 기획에 따름

저장 시점 고민:

- 사이트 접근 직후 아이디 입력도 안 한 상태에서 체크만으로 저장하는 건 이상함
- 로그인이 실제 성공한 시점에 그 아이디를 저장하는 게 맞아 보임 (실제 로그인 가능한 아이디를 저장해야 하므로)
- 단, 사용자는 "분명 체크했는데" 로그인 성공 전 다른 데 갔다 오면 풀려 있어 직관적이지 않을 수 있음 → 추후 재고민

## 로그인 프로세스

### SNS 로그인 — API 시도 과정

1. SNS 로그인 성공 후 콜백 페이지로 리다이렉트되며 인가코드·상태토큰을 받음
2. 이 값으로 백엔드 로그인 API 호출
3. 백엔드는 각 SNS 프로필 API를 호출·응답받아 가공해서 클라이언트로 돌려줌
   - 리프레시토큰/액세스토큰/유저정보를 받음
   - 리프레시·액세스토큰은 서버 세션에 저장
   - 유저정보로 우리 사이트에 해당 SNS 계정으로 가입한 계정이 있는지 확인
   - SNS 유저정보 + 로그인 상태값(가입 여부 등) + 해당 계정의 우리 사이트 정보(주문정보 등)를 합쳐 응답
4. 프론트에서 성공/실패 처리
   - 로그인 API가 오래 걸리면 콜백 페이지에 흰 화면이 길어지므로 로딩 UI 표시 권장

### 일반 로그인 — API 시도 과정

아이디/비밀번호 입력 후 위 SNS 항목의 4번부터 동일. 단, 로그인 API 호출~응답 전까지 로그인 버튼을 mutating 상태로 비활성화한다.

### 에러 응답 처리

보통 error response body의 code로 실패 사유와 에러메시지가 온다.

- 가입된 계정 없음
  - 일반 로그인: 에러메시지 노출("아이디가 존재하지 않아요"), 서비스에 따라 회원가입 유도 얼럿
  - SNS 로그인 (로그인 페이지): "가입되지 않은 회원정보입니다. 회원가입을 진행하시겠습니까?" → 회원가입 유도
    - (아니오) 윈도우 팝업만 닫음
    - (예) SNS 유저정보를 가공해 회원가입 API 호출
  - SNS 로그인 (회원가입 소개 페이지): 얼럿 없이 바로 회원가입 페이지로 이동
- 기타
  - 관리자가 차단한 회원, 탈퇴한 회원(재가입 불가 메시지 포함되는 경우도) → 대체로 메시지만 노출
  - SNS 로그인은 얼럿 후 마지막에 윈도우 팝업을 닫음

### 성공 응답 처리

1. 회원가입 소개 페이지에서 SNS 로그인 성공 시: "이미 가입된 계정으로 로그인하겠다" 얼럿 후 진행하거나, 얼럿 없이 바로 진행 (서비스마다 다름)
2. 아이디 저장 옵션 설정 시 로컬스토리지에 저장 (일반 로그인)
3. 자동 로그인 고려해 쿠키 등에 유저정보 저장 (공통)
4. 윈도우 팝업 닫음 (SNS 로그인)
5. 서비스가 요구하는 별도 대상 회원 처리 (공통)
   - 이관 대상: 이관 팝업/페이지 노출
   - 통합 회원 대상 처리
   - 휴면 회원: 휴면 페이지 이동 또는 팝업 노출
   - 비밀번호 재설정 안내 대상 처리
6. 위에 해당 안 되는 회원만 성공 후처리 (공통)
7. 뒤로가기 시 로그인 페이지로 다시 오면 안 됨 (공통)

## API 호출 요약

### 로그인 API

- Request: (일반) id/password, (SNS) 인가코드·상태토큰
- Response: 로그인 상태값(가입 여부·성공 등), 우리 사이트 유저정보(주문정보 등), SNS 유저정보

### SNS 로그인 과정

1. 윈도우 팝업 띄우고 SNS 로그인 성공 후 인가코드 등 받아 로그인 API 호출
2. 성공하면 로그인 성공 처리
3. 실패가 단순 미가입이면 "가입 안 한 계정인데 가입할래?" 컨펌
4. 아니오 → 종료
5. 예 → 회원가입 API 호출

## 로그인 / 회원가입 소개 페이지 관계

- 로그인 페이지: 로그인 성공 처리 + 미가입자 가입 유도(컨펌 후)
- 회원가입 소개 페이지: 로그인 성공 처리(이미 계정 있으면 바로 로그인) + 미가입자 가입(컨펌 없이 바로 가입)

SNS 로그인에 한해 두 페이지 기능이 사실상 동일할 수 있으나, 기획에 따라 달라질 수 있음.
