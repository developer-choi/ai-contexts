# 상품리스트 필터 노출 조건

## 필터 / 데이터 유무 4경우

- 적용된 필터 X + 데이터 X = **필터 미노출 + 데이터 없습니다 노출**
  - 대체로 서버에러 또는 집계할 상품이 없는 경우 (주문 개발이 안 되어서 베스트상품 목록이 아예 없는 경우)
- 적용된 필터 O + 데이터 X = **필터 노출 + 데이터 없습니다 노출**
  - 필터를 적용했더니 상품이 없으니까
  - 적용된 필터를 보여주고 이 필터 해제하면 상품이 보일 거라는 걸 암시함
- 적용된 필터 X + 데이터 O = **필터 노출, 데이터 노출**
- 적용된 필터 O + 데이터 O = **필터 노출, 데이터 노출**

## 필터 UI

- 필터 적용 또는 적용취소 하면 상품 전체갯수와 상품목록이 최신화되어야 함.
- 2페이지 이상에서 필터를 적용하면, 1페이지로 이동하면서 동시에 목록 제일 윗부분으로 스크롤 이동해야 함.

## 미노출 경우의 수

- 처음으로 입장했는데 목록이 없는 경우 (대체로 서버에러) > NEW IN 눌러서 갔는데 상품이 없음 > 필터 미노출 + 리스트 미노출
- 검색 결과 상품이 없는 경우 > 필터 미노출 + 리스트 미노출
- 처음 입장했을 때는 상품이 있었으나, 필터 적용하니 상품이 없는 경우 > 필터 노출 + 리스트 미노출
  - 존재하지 않다는 부분 마크업은 프론트가 임의로 진행
- 메인페이지 (별도): 처음 진입했는데 상품 없으면 그냥 전부 다 미노출

## 미노출 시나리오

- 상품 없을 것 같은 검색어 입력 > 필터 미노출, 리스트 미노출된 페이지 보여짐
- 카테고리 리스트 페이지 진입 > 아무 필터나 전부 다 적용함 > 필터 노출, 리스트 미노출된 페이지 보여짐
- (고의로) 상품목록 API 이상하게 호출하도록 구현한 후 상품 리스트 페이지 진입 (= API에서 에러가 나는 상황 가정) > 필터 미노출, 상품목록 미노출된 페이지 보여짐
