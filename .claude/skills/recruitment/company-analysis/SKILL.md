# Company Analysis

채용 진행중인 회사를 분석하여 면접 준비의 공통 재료를 만든다. 사실(Fact)과 해석(Insight)을 분리해 한 번 작성하고 모든 면접 시점(기술/컬쳐핏/임원)에서 재사용한다.

## 호출 맥락

- recruitment 시점 2 ("내일 OOO 기술면접" / "내일 OOO 컬쳐핏/임원") 진입 시 가장 먼저 실행
- 산출물이 이미 있으면 재실행하지 않고 갱신만 한다 (이전 시점의 산출물을 다음 시점이 그대로 받아 이어쓴다)
- 단독 호출 금지 — recruitment 라우터 경유

## 분석 항목

### [Fact] 사실 — 한 번 모으고 모든 시점이 참조

- BM·수익원·수익 배분 구조
- 밸류체인 (회사가 어디서 부가가치를 만들어 어디로 흐르는가)
- 핵심 플로우 (사용자 유형별 진입 → 목적 달성)
- 서비스 종류·주 사용자·주 연령층
- 데이터 플로우 (생성 주체, 흐름)
- 기술 스택 (공고/링크드인/기술 블로그 기반)
- 채용 포지션 정의 (왜 그 회사가 프론트를 뽑는가)

### [Insight] 해석 — 시점별 강조점이 다르므로 태깅하여 보관

- 유저가 받는 가치 (구체 질문으로 검증)
  - 예: 사장님이 "나 오늘 얼마 벌었지? 우리 가게에서 뭘 팔아야 하지?"를 어디서 확인하는가
- 회사가 강조하는 역량 (공고·채용 페이지·블로그·대표 인터뷰)
- 내 역량 ↔ 회사 가치 매핑
- 시점별 태깅: `#tech` (기술면접용) / `#exec` (임원·컬쳐핏용)

## 분석 절차

### 1. 입력 자료 요청

사용자에게 아래를 요청한다 (있는 만큼):

- 채용공고 (URL 또는 본문)
- 회사 채용 페이지 / 채용 브랜딩 페이지
- 서비스 사이트 (B2C면 직접 접속, B2B면 소개 페이지)
- 회사 기술 블로그 / 미디엄
- 대표·CTO 인터뷰

### 2. 공동 정독

자료별로 사용자와 같이 정독한다. **혼자 분석하지 않는다.** 자료 하나가 끝나면 사용자 반응을 확인한 뒤 다음으로 넘어간다.

병행하여 AI는 분석을 수행하되, 결과는 정독이 모두 끝난 뒤 한꺼번에 공유한다 (정독 중에는 자료 이해와 사용자 논의에 집중).

### 3. 분석 결과 공유 + 사실/해석 분리

정독이 끝나면 [Fact] 항목과 [Insight] 항목으로 나누어 한 번에 공유한다. 사용자와 논의하여 확정한다.

### 4. 산출물 작성

`outputs/company-analysis.md`에 [output-template.md](output-template.md) 형식으로 기록한다.

## 산출물 위치

- 브랜치: `interview/{회사명}` (recruitment SKILL.md의 브랜치 전략 참조)
- 파일: `outputs/company-analysis.md`
