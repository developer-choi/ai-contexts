# Company Analysis

채용 진행중인 회사를 분석하여 면접 준비의 공통 재료를 만든다. 사실(Fact)과 해석(Insight)을 분리해 한 번 작성하고 모든 면접 시점(기술/컬쳐핏/임원)에서 재사용한다.

## 호출 맥락

- recruitment 시점 2 ("내일 OOO 기술면접" / "내일 OOO 컬쳐핏/임원") 진입 시 가장 먼저 실행
- 산출물이 이미 있으면 재실행하지 않고 갱신만 한다 (이전 시점의 산출물을 다음 시점이 그대로 받아 이어쓴다)
- 단독 호출 금지 — recruitment 라우터 경유

## 분석 항목

### [Fact] 사실 — 한 번 모으고 모든 시점이 참조

- BM·수익원·수익 배분 구조
- 밸류체인 (회사가 어디서 부가가치를 만들어 어디로 흐르는가)
- 핵심 플로우 (사용자 유형별 진입 → 목적 달성)
- 서비스 종류·주 사용자·주 연령층
- 데이터 플로우 (생성 주체, 흐름)
- 기술 스택 (공고/링크드인/기술 블로그 기반)
- 채용 포지션 정의 (왜 그 회사가 프론트를 뽑는가)
- 필수 공부 키워드 (공고에서 "무조건 나올 것"으로 추리는 키워드 — **추출까지만 수행**. KA 매칭·학습은 사용자 단독 — AI는 개입하지 않는다)

### [Insight] 해석 — 시점별 강조점이 다르므로 태깅하여 보관

- 유저가 받는 가치 (구체 질문으로 검증)
  - 예: 사장님이 "나 오늘 얼마 벌었지? 우리 가게에서 뭘 팔아야 하지?"를 어디서 확인하는가
- 회사가 강조하는 역량 (공고·채용 페이지·블로그·대표 인터뷰)
- 내 역량 ↔ 회사 가치 매핑
- 시점별 태깅: `#tech` (기술면접용) / `#exec` (임원·컬쳐핏용)

## 분석 절차

### 1. 입력 자료 수집

역할 분담:

- **사용자 제공**: 채용공고 (URL 또는 본문)
- **AI 수집**: 아래를 검색해서 URL + 분석을 함께 가져온다 (없을 수도 있음 — 못 찾으면 없음 처리)
  - 회사 홈페이지
  - 서비스 페이지 (B2C/B2B)
  - 회사 기술 블로그 / 미디엄
  - 대표·CTO 인터뷰

### 2. 분석 + 결과 공유

AI는 위 자료를 단독으로 검색·분석하여 [Fact]와 [Insight]를 분리해 한 번에 사용자에게 공유한다. 사용자는 결과를 보고 모르는 부분을 묻거나 사실을 검증·교정한다 — 별도 "공동 정독" 절차는 두지 않는다.

### 3. 산출물 작성

`outputs/company-analysis.md`에 [output-template.md](output-template.md) 형식으로 기록한다.

### 4. 사용자 컨펌 안내

AI는 산출물 작성 완료 후 다음과 같이 안내한다:

> "회사 분석 1차 완료. 산출물에서 **본인이 기억하고 싶은 내용만 남기고 나머지는 지워서** 정리해주세요. 끝나면 알려주세요 — 다음 단계로 진입."

사실 검증·교정도 받을 수 있지만, 1차 안내는 **사용자 압축**이 우선이다. 산출물의 1차 목적이 "사용자가 기억할 자료"이기 때문.

메인 세션 컨텍스트에 풀 버전이 살아있으므로 다음 단계 합성 시 AI가 잘려나간 정보도 인풋으로 사용 가능. 미래 세션 회고는 그 시점에 재분석을 돌리거나 외부 자료로 복원한다.
