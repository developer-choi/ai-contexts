# 면접 준비

> 시점 2 진입 시 호출되는 공통 흐름. 활성 모드(technical / culture-fit)에 따라 분기.
> 이력·포폴 기반 예상질문은 시점 1에서 미리 만들어둔 `personal-questions.md`로 대체된다 — 모드 진입 시 자동 컨텍스트 로드.

## 진입 시 흐름 안내

interview-day.md 진입 직후 AI는 사용자에게 **전체 흐름(Step 1·2·3 + 각 단계 task)을 한 화면으로 짧게 보여준다**. 사용자가 흐름을 모르면 매 단계 답답해한다. 모드 분기·활성 베이스 로딩보다 먼저.

## 모드 진입 분기

interview-day.md 진입 시 활성 모드에 따라 사용 베이스·흐름이 달라진다.

### technical 모드 (기술면접 트리거)
- **베이스**: `interview-baselines/technical/` + `interview-baselines/culture-fit/` 양 폴더 베이스 파일 동시 로드
- **이유**: 기술면접에서도 컬쳐핏 질문이 섞여 나오므로 함께 준비
- **임원면접 간략 동반 제안 (필수)**: AI가 interview-day.md 진입 직후 사용자에게 *"임원면접도 간략히 같이 준비할까요? 같은 날/같은 슬롯에 같이 보는 경우가 많아서 미리 깔아두면 임원면접 시점 부담이 줄어듭니다"* 라고 먼저 제안. 동의 시 임원면접 파트는 **핵심 질문 4종(지원동기·강점/약점·5년 후·역질문)만 간략히 추가** — 컬쳐핏 단독 면접 깊이까지는 가지 않는다.

### culture-fit 모드 (컬쳐핏/임원 단독 트리거)
- **베이스**: `interview-baselines/culture-fit/` 베이스 파일만 로드
- **컬쳐핏 단독 면접에서는 더 깊게 다룬다**
- **컬쳐핏과 임원면접은 같은 모드로 통합 처리**

## Step 1. 면접 전 준비

> 시점 자유 — 당일 2시간 전부터 1일 전까지 가능. 분량이 많으니 가능하면 미리 시작.

AI가 아래 항목을 위에서부터 진행 관리한다. 항목별 사용자 컨펌 후 다음으로 넘어간다. 시간은 충분 — **순서대로 누락 없이** 진행이 목적.

- 회사 분석: `company-analysis/` sub-skill 호출 (산출물 없으면 생성, 있으면 갱신). 산출물의 "필수 공부 키워드" 섹션이 다음 단계의 인풋이 됨.
- **[사용자 단독] 필수 공부 키워드 KA 매핑 인터럽트**: AI는 작업을 멈추고 사용자에게 *"이제 KA에 가서 키워드별 자료를 매핑·학습해 오세요. 마치면 알려주세요"* 라고 안내한 뒤 대기. AI는 매핑·학습·자료 정리에 개입하지 않는다.
- **[협력] 회사 맞춤 예상질문 뽑기**: 활성 모드 베이스(머리말 분기 참조) + `personal-questions.md` + 회사 분석 산출물 + **사용자가 KA에서 가져온 자료**를 함께 합성. AI 1차 리스트업 → 사용자가 가져온 자료로 보강·검증 → 같이 마무리. 취미 같은 질문조차 회사와 연계.
- 역질문 금지 목록 숙지 (`reverse-interview-do-not-ask.md`)
- 회사 맞춤 역질문 리스트업: 활성 모드의 `reverse-interview.md` + 회사 분석 산출물의 raw 후보. **사용자 raw 표현은 베이스 표현으로 갈음하지 말고 후보로 같이 노출**해서 사용자가 선택하도록 한다.
- 1분 자기소개 말하며 연습 (`self-introduction.md`)
- **최종 요약본 생성** → `outputs/{회사명}/그 회사용 준비물.md` (활성 모드의 `templates/final-summary.md` 사용. 1-2페이지로 압축. Step 2·3에서 사용)

## Step 2. 트레이닝 (-20분까지)

문서 정독은 모의면접 + 최종 요약본 1회 정독으로 대체한다 (베이스 QnA 암기 목적이었으므로).

- AI와 모의면접
  - 베이스: 활성 모드의 모든 질문/역질문 베이스 파일 + `personal-questions.md` + Step 1에서 만든 회사 맞춤 예상질문 + `company-analysis/` 산출물
  - 흐름: AI가 random 질문 → 사용자 답변(텍스트) → AI 짧은 피드백 (회사 가치 매핑·말투·논리 + **임팩트 가시성** 관점)
  - **임팩트 가시성 메타 룰 (technical / culture-fit·임원 양 모드 공통)**: 모든 답변에 대해 *"이 답변으로 면접관이 '얘가 우리 회사 오면 기술/비즈니스 임팩트가 그려진다'고 느낄지"* 를 측정한다. 안 그려지면 피드백에 명시.
  - 우선순위: 사용자가 약하다 느끼는 영역부터. Step 1의 자기소개 피드백에서 잡힌 약점이 있으면 그 영역부터.
- 최종 요약본(`outputs/{회사명}/그 회사용 준비물.md`) 1회 정독
- 엘베 시간 때문에 **20분 전 출발**

## Step 3. 면접 입장 직전 (-5분 ~ 입장)

> 면접장 도착 후 입장 직전까지의 흐름. AI가 카드를 만들고 사용자가 정독한 뒤 입장한다.

### AI 액션 (카드 생성)

1. `outputs/{회사명}/그 회사용 준비물.md`의 **"면접 직전 리마인더" 섹션**을 카드로 출력 (preflight-notes + reverse-interview-do-not-ask 핵심)
2. **회사 분석 핵심 1줄** 강조 — 활성 모드별 강조점:
   - **기술면접**: BM + 내 역량 ↔ 회사 가치 매핑
   - **컬쳐핏/임원**: 회사 비전·인재상 + 내 가치 매핑
3. **태도 리마인드 카드**:
   - 시작 직전부터 **말 천천히 나긋나긋**
   - 말이 너무 많음. 프로다운 모습 보여야 하는 연차
   - 회사 뒷담·단점 **절대금지**, 네거티브 금지, 컨셉은 더 큰 장점
   - 재직중 회사 단점 얘기 = 내 얼굴에 침 뱉는 거

### 사용자 액션

1. AI 카드 정독 (1-2분)
2. 입장
