# 면접 직후 회고

면접 끝난 직후 AI가 회고를 가이드한다. 사용자에게 답변 의무를 지우는 항목은 **AI가 모를 수밖에 없는 정보**로만 좁힌다. 분위기·카드 적중률·결과 예측 등 AI가 대화 흐름·사용자 행동에서 추출 가능한 것은 사용자에게 묻지 않는다.

## 받아야 할 것 (사용자만 아는 정보)

- **보완해야 할 질문** — 답변 못 했거나 망친 질문, 카드에 없던 새 질문
- **회사 분석 보강** — 면접에서 얻은 회사 정보 (팀 구성, 워크플로우, 면접관 발언, 기술 의사결정 등)
- **다음 면접에 쓸 역질문 거리** — 면접 중 "이거 다음에도 써먹겠다" 떠오른 것

## 묻지 말 것 (AI가 추출 가능)

- 면접 분위기·압박톤·면접관 구성·길이
- 카드 적중률·정답 채점
- Tier S 스킵 검증
- 다음 시즌 대비 후보
- 스킬 자체 피드백
- 결과 예측·합격 가능성

위 항목들은 회고 흐름·사용자 답변·이전 대화 맥락에서 AI가 직접 추출해 반영 단계에서 활용한다.

## 절차

1. 아래 양식대로 `outputs/{회사명}/retrospective-questions.md` 생성. 각 섹션 부제(`>`)는 그대로 유지. 자유 서술이며 떠오르는 항목만, 비워둬도 OK.
2. 사용자가 그 파일에 직접 작성 후 신호
3. AI가 답변 + 대화 흐름에서 추출한 정보를 합쳐 적절한 위치(베이스 파일 / `company-analysis/` 산출물 / SKILL.md / 백로그)에 반영. 항목별로 사용자 컨펌 후 진행

## 산출 양식

```markdown
# {회사명} {면접종류} 회고

## 1. 보완해야 할 질문

> 답변 못 했거나 망친 질문 + 카드에 없던 새 질문. 한 줄씩 — 질문 + 어떻게 답했는지 / 왜 못 했는지.

-

---

## 2. 회사 분석 보강

> 면접에서 얻은 회사 정보. 팀 구성, 워크플로우, 사무실, 면접관 발언, 기술 의사결정 등.

-

---

## 3. 다음 면접에 쓸 역질문 거리

> 면접 중 "이거 다음에도 써먹겠다" 떠오른 거.

-
```
