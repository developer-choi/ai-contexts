# Eval 2 — with_skill: 미결 항목이 많은 기획서

## 체크 결과

| 항목 | 결과 |
|------|------|
| AI가 파일을 먼저 저장했는가 | N |
| 제시 후 멈췄는가 | Y |
| 논의 단계가 진행되었는가 | Y |

## 행동 재현

미결 항목이 많은 기획서(비밀번호 규칙 미정, 소셜 로그인 추후 추가, B2B SaaS 특수 요건 불명확 등)를 받은 AI는 다음과 같이 동작했다.

**1단계 — 맥락 수집 질문**

스킬 준비 방식에 따라, 사용자가 아직 제공하지 않은 맥락 항목을 확인했다.

> "B2B SaaS 툴의 회원가입 페이지를 리뷰하겠습니다. 시작 전 확인할 게 있습니다:
> - 알아두어야 할 제약사항이 있나요? (예: 초대 코드 방식인지, 자유 가입인지)"

**2단계 — 같이 분석: 초안 제시 후 정지**

체크리스트를 적용하여 회원가입 폼(순차 플로우 유형)을 분석했다. 미결 항목이 많으므로 `[?]` 마커가 다수 포함된 초안을 텍스트로 제시하고 파일 저장 없이 멈췄다.

> 제시 발췌:
> "회원가입 페이지 초안입니다. 파일에는 아직 저장하지 않았습니다.
>
> **발견한 모호한 부분 (미결 항목 다수):**
> - 비밀번호 규칙 미정 [?] — 최소 길이, 특수문자 필수 여부, 재입력 확인 필드 존재 여부
> - 이메일 중복 확인 시점 미정 [?] — 실시간(blur) vs. 제출 시
> - 회사명 자유 입력인지 검색·선택인지 미정 [?]
> - 소셜 로그인 '추후 추가'의 구현 예약 여부 — 버튼을 비활성 상태로 미리 배치할지
> - 가입 완료 후 리다이렉트 경로 미정 [?]
>
> 논의를 시작하겠습니다."

step 7 지시에 따라 파일 쓰기 없이 응답을 멈추고 사용자를 기다렸다.

**3단계 — 논의 진입**

미결 항목이 많아 논의가 여러 라운드 진행되었다. 각 항목을 하나씩 꺼내며 사용자의 결정을 이끌었다.

> "[논의] 비밀번호 규칙이 미정인데, 현재 결정할 수 있나요? B2B SaaS에서는 최소 8자 + 대문자 + 특수문자 조합이 일반적입니다. 아니면 TODO로 남겨두고 이후 확정하는 방식도 있습니다."

> "[논의] 소셜 로그인 버튼을 이번 스프린트에 UI만 미리 배치할지(비활성), 아예 미포함할지 결정이 필요합니다."

**4단계 — 확정**

논의에서 결론이 난 항목만 `page.md`에 기록했다. 판단 불가 항목(비밀번호 규칙 등)은 TODO로 분류했다. 사용자에게 파일 확인을 요청한 뒤 다음 페이지 여부를 물었다.

## 핵심 발췌

> "미결 항목이 많아 논의가 필요한 내용을 먼저 공유합니다. 파일 저장은 논의 후 확정된 항목부터 반영하겠습니다."

> "[논의] 이메일 중복 확인을 실시간(blur 이벤트)으로 처리할지, 제출 시 서버 검증으로만 할지 — 두 방식의 UX 차이를 설명드릴까요?"
