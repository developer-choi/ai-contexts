# Eval 2 — without_skill: 미결 항목이 많은 기획서

## 체크 결과

| 항목 | 결과 |
|------|------|
| AI가 파일을 먼저 저장했는가 | Y |
| 제시 후 멈췄는가 | N |
| 논의 단계가 진행되었는가 | N |

## 행동 재현

스킬 없이 미결 항목이 많은 회원가입 기획서를 받은 AI는, 불확실한 항목을 스스로 가정하거나 "미정"으로 표기한 채 파일을 완성하고 세션을 종료했다.

**1단계 — 맥락 확인 없이 직접 분석 착수**

사용자에게 배경 질문을 하지 않고 바로 분석을 시작했다. B2B SaaS 특수성(초대 방식, 팀 계정 등)에 대한 확인 없이 일반적인 가입 플로우로 가정했다.

**2단계 — 분석 및 파일 저장 (한 번에 처리)**

다음 파일들을 즉시 생성하고 내용을 채웠다:

- `/plan/background/global.md`
- `/plan/pr1/page.md`

미결 항목(비밀번호 규칙 등)은 파일 내에 "(미정)" 또는 "(추후 확인 필요)"로 표기한 채 저장했다.

> 행동 발췌:
> "회원가입 페이지 리뷰를 완료했습니다. `/plan/pr1/page.md`에 분석 결과를 저장했습니다. 미정 항목은 파일 내에 표시해 두었습니다."

**3단계 — 사용자 논의 없이 요약 전달**

발견 사항과 질문 목록을 한 번에 제시하고 끝냈다. 각 미결 항목에 대해 사용자와 결론을 내는 과정이 없었다.

> 요약 발췌:
> "주요 미결 항목:
> 1. 비밀번호 규칙 — 기획자 확인 필요
> 2. 이메일 중복 확인 방식 — 기획자 확인 필요
> 3. 소셜 로그인 배치 시점 — 추후 결정
> 4. 가입 완료 후 리다이렉트 경로 — 기획자 확인 필요
>
> 위 사항 확인 후 문서를 업데이트하세요."

**논의 단계 누락**

미결 항목에 대해 AI가 사용자와 함께 결론을 내는 대화가 전혀 없었다. 이미 파일이 저장된 상태라 "논의 → 반영" 흐름이 단절되었다. 사용자는 혼자 파일을 수정해야 하는 상황이 됐다.

## 핵심 발췌

> "분석 결과를 저장했습니다. 미정 항목은 파일 내 '(미정)'으로 표기했으니 확인 후 직접 채워주세요."

(논의 단계 없음 — 파일 완성 후 단방향 전달로 종료)
