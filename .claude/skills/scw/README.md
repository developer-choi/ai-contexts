# SCW (Skill Creator Wrapper)

현재 런타임에서 제공되는 skill-creator에 자체 원칙과 품질 검증을 추가한 스킬입니다.

## 왜 감쌌나요?

skill-creator에 제 원칙을 얹기 위해서입니다.

**how 최소화** — 스킬에는 "무엇을 하라(what)"만 남기고 "어떻게 하라(how)"는 최소화합니다. how를 뺐을 때 AI가 못 하는지를 반드시 비교 테스트로 검증합니다.

**산출물 양식 자유** — 마크다운 예시로 양식을 고정하지 않습니다. 필수 필드만 명시하고, 구체적인 형태는 AI에게 맡깁니다. 예시로 고정하면 더 나은 구조를 만들 기회를 막기 때문입니다.

**gotchas 축적** — 스킬을 처음 만들 때는 비워두고, 실사용 중 AI가 반복 실패하는 패턴이 발견되면 그때 추가합니다.

**프롬프트 감사** — 완성된 프롬프트 문서를 명확성, 정합성, 구조, 간결성 관점에서 검증합니다.

**책임 분리 (SRP)** — 규칙을 추가할 때, 그 규칙이 본인 행위인지 다른 곳의 행위인지 점검합니다. 다른 곳 행위면 그 단일 출처에 두고 본인 본문에는 한 줄 위임만 둡니다. 같은 규칙이 두 곳에 적혀 동기화 부담이 생기는 패턴을 막기 위함입니다.

**0건 수렴까지 재벤치** — 벤치 라운드에서 잔류 이슈가 1건이라도 남으면 자체 판단으로 종료하지 않고 다음 라운드를 돌립니다. "97% 통과면 충분하다"가 다음 사용 때 같은 누설로 재발하면서 가이드 신뢰도를 무너뜨리기 때문입니다.
