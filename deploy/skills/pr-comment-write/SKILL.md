# PR Comment Write

남의 PR에 코멘트를 달 때, 사용자가 틀린 말을 하거나 불필요한 질문을 하지 않도록 검증하고, 말투를 다듬어주는 스킬.

## 사전 확인

진행 전에 [STATUS.md](STATUS.md)를 읽고 현재 상태를 파악한다.

## 왜 이 스킬이 필요한가

PR 코멘트는 공개적이고 기록에 남는다. 잘못된 지적은 신뢰를 떨어뜨리고, 날카로운 말투는 관계를 해친다. 이 스킬은 코멘트를 올리기 전에 "내가 틀린 건 아닌지", "이미 PR 본문에 설명된 건 아닌지", "말투가 괜찮은지"를 체크한다.

## 진행 절차

### 1. PR 맥락 파악

사용자가 PR URL이나 브랜치를 제공하면:

- PR 본문(description)을 읽는다
- 변경된 파일의 diff를 읽는다
- 이미 달린 코멘트가 있으면 읽는다

### 2. 사용자 의견 수집

사용자가 코멘트로 남기고 싶은 내용을 받는다. 형태는 자유 — 키워드, 메모, 완성된 문장 다 가능.

### 3. 검증

코멘트를 작성하기 전에 아래를 확인한다:

- **PR 본문 대조**: 사용자가 지적하려는 내용이 PR 본문에 이미 설명돼 있지 않은지. 설명돼 있으면 사용자에게 알린다.
- **변경 범위 확인**: 지적하려는 코드가 이 PR에서 실제로 수정된 건지, 기존부터 있던 건지. 기존 코드면 사용자에게 알린다.
- **기술적 정확성**: 사용자의 지적이 기술적으로 맞는지. 틀렸거나 근거가 약하면 사용자에게 알린다.

검증에서 문제가 발견되면 코멘트를 작성하지 않고 사용자에게 먼저 알린다. 사용자가 그래도 진행하겠다고 하면 진행한다.

### 4. 코멘트 작성

검증을 통과한 내용으로 코멘트를 작성한다.

말투 원칙 ([writing-guide/tone.md](../../contexts/writing-guide/tone.md) 참조):
- 담백하되 부드럽게. "이것도 같이 수정이 가능할까요?" 느낌
- 상대 잘못이 아닐 수 있으므로 단정하지 않고 제안 톤으로
- 기존 코드 문제를 짚을 때: "이 부분은 기존 코드인 것 같은데, 이번에 같이 개선하면 어떨까요?" 톤

### 5. 사용자 확인

완성된 코멘트를 사용자에게 보여준다. 사용자가 확인하면 제출한다.
