# Writer

문서 작성이 필요할 때, writer 팀 에이전트를 통해 톤 교정, 구조 설계, 분량 조절을 수행한다.

## 에이전트 관리

- writer 팀 에이전트(`deploy/agents/writer.md`)가 현재 팀에 없으면 스폰한다
- 이미 떠 있으면 기존 에이전트에 SendMessage로 전달한다
- 세션 종료 전까지 에이전트를 유지한다 (교정 사례 축적을 위해)

## 모드

### create (기본)

텍스트를 writer에게 보낼 때, 아래 정보를 최대한 풍성하게 채워서 함께 전달한다:

- **독자**: 이 문서를 읽는 사람 (채용담당자, 팀원, 일반 사용자 등)
- **목적**: 이 문서의 목적 (포트폴리오, PR 설명, 코드리뷰 코멘트 등)
- **맥락**: 이 텍스트가 나온 배경 (어떤 PR인지, 어떤 과제인지 등)
- **초안**: 다듬을 텍스트. 무에서 유를 만드는 건 어렵지만 풍성한 초안을 추리는 건 쉬우므로, 초안은 과하다 싶을 만큼 많이 담는다

독자·목적·맥락을 사용자가 제공하지 않으면, 현재 작업 맥락(파일, git log, PR 등)에서 추론하여 채운다. 추론할 수 없는 항목은 비워두고 writer에게 전달한다 — writer의 "첫 질문" 로직이 필요 시 사용자에게 직접 확인한다.

writer의 교정 결과를 사용자에게 전달한다. 사용자가 반려하면 반려 사유와 함께 writer에게 다시 보낸다.

### report

writer의 사례 조회 흐름을 호출한다. writer가 세션 중 쌓아둔 (원본, 수정, 사유) 목록을 반환한다.
