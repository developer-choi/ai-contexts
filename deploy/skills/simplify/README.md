# Simplify

라이브러리 소스코드를 학습 목적으로 **간소화**하는 워크플로우입니다.

- **목표**: 복잡한 오픈소스 코드에서 핵심 개념만 남기고 나머지를 제거하여, 읽고 배우기 쉬운 코드로 만들기
- **접근**: 기능을 하나씩 점진적으로 제거하며, 매 단계마다 문서화
- **사용법**: `/simplify [init | 작업 단계]`

## 무슨 문제를 해결하려고 했나요?

오픈소스 라이브러리의 소스코드를 읽고 싶었습니다.

하지만 실제 프로덕션 코드에는 학습과 무관한 것들이 많습니다:
- 브라우저 호환성 처리, SSR 안전장치, deprecated prop 지원
- 복잡한 ref 전달, 접근성 세부사항, 엣지 케이스 방어 코드

이런 코드를 걷어내고 핵심 구현만 남기면, 라이브러리의 설계 의도와 패턴을 훨씬 빠르게 파악할 수 있습니다.

## 작업 흐름

```
1. 대상 라이브러리 clone & 프로젝트 초기화
2. 코드베이스 구조 분석 & 분석 큐 작성
3. 대상 선택 → 계획서 작성 → 사용자 승인
4. 기능을 하나씩 제거하며 커밋 (원본 문서 → 단순화 → 단순화 문서)
5. 패턴 문서 업데이트 → 다음 대상으로 반복
```

## 주요 산출물

| 산출물 | 설명 |
|--------|------|
| 간소화된 소스코드 | 핵심 구현만 남긴 코드 (커밋 단위로 추적 가능) |
| `*-original.md` | 원본 코드 요약 (간소화 전 스냅샷) |
| `*-simplified.md` | 단순화된 코드의 상세 학습 가이드 |
| 패턴 문서 | 공통/라이브러리별 유지-삭제 패턴 카탈로그 |
