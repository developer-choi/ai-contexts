# 원본 분석 문서 양식 (component)

> 간소화 작업 **전에** 원본 코드를 빠르게 이해하기 위한 요약 문서. 상세 내부 구현은 simplified.md에서 다루므로 여기선 요약만 적는다.

마크다운 구조·표·트리 표현은 자유. 아래 **필수 필드**를 모두 포함한다.

## 필수 필드

- **무슨 기능을 하는가** — 한 줄 정의(핵심 역할·책임)와 핵심 기능 목록
- **주요 코드 구조**
  - 파일 위치 및 크기(줄 수)
  - 렌더링 구조(컴포넌트 합성 트리)
  - 주요 Props (이름·타입·기본값·설명)
  - 핵심 로직 발췌 — 가장 중요한 부분만
- **복잡도의 이유** — 이 원본이 복잡한 이유
- **간소화 방향** — 제거 고려 대상과 각 이유

> 상세 간소화 결과는 `[ComponentName]-simplified.md`로 연결한다.
