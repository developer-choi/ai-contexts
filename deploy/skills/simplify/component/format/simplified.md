# 단순화 학습 문서 양식 (component)

> 단순화된 코드의 **상세 학습 가이드**. 코드만 봐선 이해하기 어려운 동작 원리·핵심 패턴·설계 의도를 설명한다. 원본 빠른 파악은 `[ComponentName]-original.md` 참고.

마크다운 구조·표·플로우차트 표현은 자유. 아래 **필수 필드**를 모두 포함하고, 각 `<ai-only>` 지시를 지킨다.

## 필수 필드

### 무슨 기능을 하는가 (남은 것)

한 줄 정의와 남은 핵심 기능 목록.

<ai-only>
해당 컴포넌트 자체가 하는 일만 작성한다. 하위 컴포넌트의 기능까지 포함하지 않는다.
</ai-only>

### 핵심 학습 포인트

이 컴포넌트에서 배울 핵심 개념·패턴을 코드 발췌와 함께 설명한다. 각 포인트마다 **학습 가치**(왜 중요한지, 어떤 상황에 쓰는지)를 포함한다.

### 내부 구조

- **렌더링 구조** — 합성 트리(각 노드 역할), 현재 줄 수와 원본 줄 수
- **핵심 상태** — ref·state·변수의 용도
- **함수 역할** — 함수마다 역할·호출 시점·핵심 로직·**왜 이렇게 구현했는지**
- **동작 흐름** — 핵심 로직을 플로우차트와 시나리오 예시로 시각화
- **핵심 패턴/플래그** — 특수 목적 변수·패턴을 **비유**와 함께, "없으면 생기는 문제"까지
- **주요 변경 사항(원본 대비)** — 제거·변경된 부분과 이유

### Props

이름·타입·기본값·설명. **제거된 Props**는 제거 이유와 함께 별도 표기한다.

### 커밋 히스토리로 보는 단순화 과정

<ai-only>
- 계획서의 커밋 번호/단계와 반드시 일치
- 각 커밋의 해시(Hash) 포함
- 삭제된 코드와 그 이유(학습 목적·복잡도)를 함께 설명
</ai-only>

### 원본과의 차이점

코드 라인(감소율), Props 개수, 기능별 유지/제거 대비.

### 학습 후 다음 단계

관련 컴포넌트(학습 목적)와 실전 응용 예시.
