# write-doc

메인 에이전트가 초안을 작성하고, editor 팀 에이전트가 톤·구조·분량을 리뷰한다.

## 에이전트 관리

- editor 팀 에이전트(`deploy/agents/editor.md`)가 현재 팀에 없으면 스폰한다
- 이미 떠 있으면 기존 에이전트에 SendMessage로 전달한다

## 모드

### create (기본)

**메인 에이전트가 초안을 작성한다.** 컨텍스트(파일, git log, PR 등)를 직접 읽어 사실에 기반한 초안을 생성한다. 초안은 과하다 싶을 만큼 풍성하게 작성한다 — 추리는 건 쉬우므로.

초안을 editor에게 보낼 때, 아래 정보를 함께 전달한다:

- **독자**: 이 문서를 읽는 사람
- **목적**: 이 문서의 목적
- **맥락**: 이 텍스트가 나온 배경
- **초안**: 메인이 작성한 초안 전문

사용자가 독자·목적·맥락을 제공하지 않으면 현재 작업 맥락에서 추론하여 채운다.

editor의 리뷰 결과를 받으면:
- 이슈가 있으면 메인이 사용자에게 이슈 목록과 현재 초안을 함께 제시한다. 사용자가 반영/무시를 결정한다. 반영이면 메인이 수정 후 editor에게 재검토를 요청한다. 무시이면 현재 초안을 최종본으로 제시한다.
- "OK"이면 사용자에게 최종본을 제시한다.

사용자가 반려하면 메인이 초안을 수정하고 editor에게 재검토를 요청한다.

### report

editor에게 "세션 중 쌓아둔 사례 목록을 반환해달라"고 SendMessage로 요청하고, 반환된 목록을 사용자에게 그대로 전달한다.
