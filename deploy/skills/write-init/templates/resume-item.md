## 구조 규칙

- 한 항목은 **제목 + 문제 + 해결 + 성과** 4줄 블록으로 구성한다.
- **링크는 본문에 넣지 않는다.** PR URL·문서 URL은 `refs` 필드에만 보관한다.
- 수치·디테일(페이지 수, 버튼 수, 코드 라인 수 등)은 `성과`에 몰아 넣는다. `문제`·`해결`은 담백하게.
- 방법(how)과 성과(outcome)를 섞지 않는다 — "AI 하네스 엔지니어링 도입" 같은 방법은 `해결`, 그 결과 수치는 `성과`.
- **rendering_env**: 이력서 플랫폼의 입력 필드가 보통 plain-text이므로 `plain-text`를 기본으로 둔다. 백틱·헤딩 마크·**볼드** 등 마크다운 문법은 본문에 사용하지 않는다.
- **톤**: write-refine에서 `resume-guide.md`의 톤 예외(명사형·한다체)를 적용한다. write-init 단계에서는 재료만 정리하고 톤은 신경 쓰지 않아도 된다.

## 시안 다양화 연결

write-refine이 이 패키지를 받으면 자동으로 "시안 다양화 모드"에 진입해 서로 대비되는 3축 시안을 뽑는다. 따라서 write-init에서 한 가지 표현으로 고정하지 말고, 재료(수치·방법·결과)를 열거하는 식으로 남겨도 된다.

## 시리즈 작업 주의

여러 항목(여러 경력기술서 주제)을 순차로 만들 때는 `refs`에 이전 확정 항목들의 경로를 누적한다. write-refine이 시리즈 작업 모드에서 "재사용 금지 재료 목록"을 만들 때 참조한다.
