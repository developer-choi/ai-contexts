# 시안·기획 자료 수집 가이드

디자이너·기획자에게서 시안·기획서를 받을 때 자료 수집 방식 자체가 잘못되면, 시안 정독 단계에서 추측·오해가 누적된다. 받기 전 또는 받은 직후에 이 가이드대로 자료가 준비되어 있는지 점검한다.

## 권장 매체 매핑

| 정보 종류 | 매체 |
|---|---|
| validation 규칙·정확한 텍스트 | Figma 텍스트 복사 (이미지 X) |
| description 박스 | Figma 텍스트 복사 |
| 컬럼명·라벨·enum 한글 매핑 | Figma 텍스트 복사 |
| 레이아웃·컴포넌트 형태 | Figma export 2x PNG |
| 모달·에러·별도 화면 | Figma export 2x PNG (의미 단위 별도) |

핵심: **텍스트 정보는 텍스트로, 레이아웃은 이미지로**. 텍스트가 든 화면을 PNG로만 받으면 흐림·오타 위험.

## 받기 전 안내 문안

사용자가 시안을 아직 받지 않은 상태면 모델이 능동적으로 안내한다:

> 시안 받기 전에 자료 수집 방식 미리 알려드릴게요. 이 방식이 시안 정독 시 추측·오해를 가장 많이 줄입니다.
>
> - **레이아웃·컴포넌트** — Figma `Export` → PNG 2x 권장 (윈도우 화면 캡처 X — 텍스트 흐림 위험)
> - **기획 description / validation 규칙 / 컬럼 명세 / 옵션 라벨** — Figma에서 텍스트 그대로 복사 → md로 붙여넣기
> - 페이지당 캡처 1장 + 텍스트 메모 정도면 충분. 분할 X.
> - 모달·에러 같은 별도 화면은 별도 캡처

## 이미 받은 후 점검

자료가 이미 들어와 있으면, 흐린 영역·식별 어려운 텍스트를 구체적으로 짚어 다시 요청한다:

- "캠페인 테이블 컬럼이 식별 안 됨 → 그 영역만 Figma export 2x로 다시 받을 수 있나요?"
- "validation 규칙은 텍스트로 복사 부탁합니다."

추측으로 진행 금지.

## Anti-pattern

```
❌ 윈도우 화면 캡처(Win+Shift+S)로 페이지 전체를 zoom 안 한 채로 받음
   → 텍스트 흐림 → 컬럼 7개로 추측 (실제 9개)
   → 사용자가 직접 짚어줄 때까지 잘못된 가정으로 진행

✅ Figma export 2x + description 텍스트 별도 복사
   → 컬럼 9개, enum 한글 매핑, 데이터 포맷, 권한별 동작까지 한 번에 식별
```
