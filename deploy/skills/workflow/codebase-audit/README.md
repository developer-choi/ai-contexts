# Codebase Audit

기존 코드의 구조와 영향 범위를 분석하는 스킬입니다. workflow의 PR 분할 단계에서 호출됩니다.

## 왜 만들었나요?

새로운 코드를 짜기 전에 기존 코드를 먼저 파악해야 합니다. 이미 구현된 것이 있는지, 수정해야 할 위치는 어디인지, 재사용할 수 있는 것은 무엇인지를 모르고 시작하면 중복 구현이나 의도치 않은 사이드이펙트가 생깁니다.

이 스킬은 지정한 파일이나 폴더를 탐색하여 모듈 간 의존성과 영향 범위를 분석하고, PR 분할과 구현 전략의 기반을 만듭니다.
