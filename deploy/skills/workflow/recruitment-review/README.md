# Recruitment Review

채용 과제의 요구사항을 분석하는 스킬입니다.

## 왜 만들었나요?

채용 과제를 받으면 바로 코드를 짜기 전에, 공고·메일·과제 요구사항을 교차 분석합니다. 공고에서 강조하는 역량과 과제 구조를 대조하면, 명시되지 않은 평가 기준이 보입니다.

## 어떻게 분석하나요?

AI가 공고, 메일, 과제 요구사항을 동시에 읽으며 교차 분석합니다. 놓치기 쉬운 세부 스펙, 공고의 자격요건에서 역추론한 암묵적 평가 포인트, 대상 서비스의 API와 도메인 구조까지 병렬로 조사합니다.

분석이 끝나면, 공고가 중시하는 역량에 맞춰 어필 포인트를 먼저 제안합니다. 완성도(404 페이지, 파비콘, OG 태그, 로딩/에러 상태), UX(뒤로가기, 빈 상태, 토스트 피드백) 등 사람이 미처 챙기지 못하는 디테일까지 TODO로 쌓아둡니다.

이 과정은 AI 혼자가 아니라 사용자와 함께 진행합니다. 서로 아이디어를 주고받으며 구현 스펙을 만들어갑니다.
