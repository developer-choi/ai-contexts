# Recruitment Review

채용 과제의 요구사항을 분석하여, 이후 PR 분할과 구현의 기반이 되는 분석 문서를 작성합니다.

## 입력

Step 1에서 사용자가 `/plan/background/`에 작성한 원본 자료 3개:

- `공고.md` — 채용공고 원본 (공고가 닫히면 접근 불가하므로 보관 필수)
- `메일.md` — 채용 담당자가 보낸 메일 본문
- `과제요구사항.md` — 과제 요구사항 문서

이후 step(과제 정의, 리뷰 등)에서 원본을 직접 참조하여 누락/개선점을 확인한다.

## 산출물: `/plan/background/`

- `analysis.md` — 분석 결과

## 절차

### 1. 요구사항 파악

입력 파일 3개(`공고.md`, `메일.md`, `과제요구사항.md`)를 읽고 다음을 정리합니다:

- 필수 구현 범위와 선택 구현 범위
- 명시된 기술 스택 제약 (있는 경우)
- 제출 기한 및 제출 방식
- 과제에서 명시적으로 요구하는 산출물 (README, 배포 링크 등)

### 2. 평가 기준 추론

과제 설명에서 명시적/암묵적 평가 기준을 추론합니다:

- 명시된 평가 기준이 있으면 그대로 정리
- 없으면 과제 성격에서 추론 (코드 품질, 완성도, UX, 기술 선택 근거 등)

### 3. 기술 스택 결정 포인트 정리

기술 선택이 필요한 지점을 나열하고, 각각의 선택지와 판단 기준을 제시합니다.
사용자와 논의하여 확정합니다.

### 4. 모호한 부분 및 질문 목록

과제 설명에서 불명확한 부분을 짚고, 채용 담당자에게 질문할지 / 본인 판단으로 결정할지 사용자와 논의합니다.
