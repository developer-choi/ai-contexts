# PR Comment Respond

내 PR에 달린 코멘트를 분석하고, 유형별로 대응 전략을 세우는 스킬. 답글 작성은 pr-comment-write가 담당하고, 이 스킬은 "무엇을 해야 하는가"를 정리한다.

## 사전 확인

진행 전에 [STATUS.md](STATUS.md)를 읽고 현재 상태를 파악한다.

## 왜 이 스킬이 필요한가

코멘트를 받으면 단순히 고치는 것보다 중요한 게 있다. 내가 몰라서 틀린 건지, 실수한 건지, 의견 차이인지에 따라 대응이 달라진다. 모르는 걸 받으면 학습해야 하고, 실수를 반복했으면 재발 방지 조치가 필요하다.

## 진행 절차

### 1. 코멘트 수집

사용자가 코멘트 내용을 알려준다 (복붙, 스크린샷, 구두 설명 등).

### 2. 분류

각 코멘트를 아래 유형으로 분류한다:

- **모르는 것**: 내가 몰랐던 지식이나 패턴. 공식문서 등으로 학습이 필요한 것.
- **실수/오류**: 알고 있었지만 놓친 것. 재발 방지가 필요한 것.
- **의견 차이**: 틀린 게 아니라 접근법이 다른 것. 논의가 필요한 것.
- **단순 수정**: 오타, 네이밍 등 바로 고치면 되는 것.

분류 결과를 사용자에게 보여주고 확인받는다.

### 3. 유형별 대응

#### 모르는 것
- 해당 주제를 조사한다 (공식문서, 웹 검색 등)
- 학습한 내용을 사용자에게 설명한다
- 사용자가 어디에 기록할지 결정한다 (KA, 메모리 등)

#### 실수/오류
- 왜 발생했는지 원인을 분석한다
- 재발 방지를 위해 백로그에 메모한다 (어떤 스킬/규칙을 보완하면 되는지)
- 코드 수정이 필요하면 수정 방향을 제안한다

#### 의견 차이
- 양쪽 근거를 정리한다
- 사용자가 수용할지 반박할지 판단할 수 있게 정보를 제공한다

#### 단순 수정
- 바로 수정한다

### 4. 답글 준비

대응 전략이 정리되면, 답글이 필요한 코멘트에 대해 pr-comment-write 스킬로 넘겨서 답글을 작성한다.

### 5. 전체 점검

모든 코멘트를 처리한 뒤, 수정한 코드가 다른 곳에 영향을 주지 않는지 전체 점검한다. 한 곳 고쳤더니 다른 데 문제될 수 있으므로.
