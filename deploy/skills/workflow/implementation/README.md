# Implementation

구현 방침에 따라 코드를 작성하고 커밋하는 스킬입니다. workflow의 구현 단계에서 서브에이전트로 호출됩니다.

## 왜 서브에이전트인가요?

메인 에이전트가 직접 코드를 짜면 구현 과정의 세부 맥락(파일 내용, diff, 에러 메시지 등)이 대화에 쌓여 컨텍스트가 오염됩니다. 이후 단계(최종 점검, PR 본문 작성 등)에서 불필요한 정보가 판단을 방해합니다.

서브에이전트에 구현을 위임하면, 메인 에이전트는 구현 결과(커밋)만 받고 깨끗한 컨텍스트로 다음 단계를 진행할 수 있습니다.

> 자세한 내용은 [SKILL.md](SKILL.md)를 참고하세요.
