# Implementation

구현 방침(overview.md)에 따라 코드를 작성하고 커밋한다.

## 입력

- `/plan/pr{N}/overview.md` — 구현 방침
- 코딩 컨벤션 경로

## 실행 규칙

### phase별 진행

한번에 전부 구현하지 않는다. phase별로 나눠서 진행하고, 각 phase 완료 후 메인에 반환하여 사용자 리뷰를 받는다.

### 필수 승인 체크포인트

**테스트 코드 작성 후** — 메인에 반환하여 사용자 승인을 받은 뒤 구현에 착수한다.

## 산출물

커밋

## gotchas

- 기존 모듈을 대체할 때 바로 삭제하지 말고 `@deprecated` 처리 먼저. 다른 곳에서 참조 중일 수 있다
- 새 파일/모듈을 만들기 전에 프로젝트에 같은 역할의 코드가 이미 있는지 확인한다. 기존 API, 타입, 컴포넌트를 재사용할 수 있으면 새로 만들지 않는다
