# Spec Review

기획서나 디자인 시안을 받았을 때, 코드를 작성하기 전에 개발자 관점에서 빠진 부분이 없는지 점검하는 스킬입니다.

## 왜 만들었나요?

기획 단계에서 빠진 내용을 미리 발견하면, 구현 도중 "이 경우는 어떻게 하죠?"라는 질문이 줄어들고 재작업도 줄어듭니다.

## 어떻게 점검하나요?

점검은 두 가지 축으로 진행합니다.

**관심사별 점검** — 어떤 화면이든 공통으로 확인합니다.

- 이 화면의 목적과 전후 화면과의 관계
- 사용자 동선과 상태 변화 (로딩, 완료, 비어 있음 등)
- 에러와 예외 상황 (네트워크 오류, 권한 없음, 잘못된 입력)
- 데이터 출처와 갱신 범위
- URL 설계와 뒤로가기/새로고침 동작
- 디자인 시안에서 놓치기 쉬운 세부 규칙

**페이지 특성별 점검** — 화면의 구조적 특성에 따라 추가합니다.

- 폼: 입력값 검증 시점, 이탈 시 경고, 포커스 이동 순서
- 목록: 빈 상태, 대량 데이터, 검색/필터 조합
- 순차 플로우: 단계 간 이동, 중간 URL 직접 접근, 이탈 후 복귀
- 상세 페이지: 메타데이터/SEO, 존재하지 않는 리소스 처리

> 자세한 내용은 [SKILL.md](SKILL.md)를 참고하세요.
