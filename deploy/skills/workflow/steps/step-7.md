# Step 7: PR 본문 작성

> **이 단계의 목표: PR 본문을 작성한다**

Step 6 완료 후, PR 본문을 작성하는 단계입니다.

---

## Step 7.1. 준비

- **문서 톤 규칙 로드**: [coding-standards/README.md](../../../contexts/coding-standards/README.md)에서 문서 작성 관련 컨벤션을 찾아 로드
- **이전 PR 본문 참고**: 이전 PR의 `pr-body.md`를 읽고, 이미지 배치·섹션 구조·서술 패턴을 맞추기

---

## Step 7.2. PR 본문 작성 (`/plan/pr{N}/pr-body.md`)

### 참고 자료
- `/plan/project.md`의 **공통 참고** + 현재 PR **참고**에 명시된 background 파일
- `/plan/pr{N}/overview.md`
- `/plan/pr{N}/review.md` (있는 경우)
- 커밋 로그

[문제 해결 로드맵](../../../contexts/self-help/roadmap.md)의 5단계를 **사고 프레임워크**로 활용합니다. 고정 양식이 아니라, 각 단계의 관점에서 사용자의 문제 해결 과정을 끌어내고 PR 본문으로 구조화합니다. 세션에서 step 1~6을 함께 진행하며 쌓인 맥락을 활용하되, 사용자가 놓치고 있는 단계가 있으면 질문으로 끌어냅니다.

- **작성 스타일**:
  - 눈에 보이는 현상/동작부터 시작한다.
  - 구체적 코드/컴포넌트 이름으로 설명한다.
  - Before/After 같은 직관적 비교를 사용한다.
- **말투 일관성**: 작성 완료 후 전체를 훑어 음슴체/습니다체 혼용이 없는지 확인

---

## 보고 내용

산출물 작성 후 사용자에게 다음을 요약하여 보고:

- PR 본문의 핵심 구조 (어떤 섹션으로 구성했는지)
- 이전 PR 본문과 톤·구조 일관성 여부
