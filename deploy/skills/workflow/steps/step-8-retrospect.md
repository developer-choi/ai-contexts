# 워크플로우 회고

해당 세션에서 진행한 작업의 컨텍스트를 활용하여 워크플로우를 되돌아본다. PLANNING_SESSION은 계획 완료 시, 마지막 IMPLEMENTATION_SESSION은 구현 완료 시 각각 수행한다.

## 사용자 패턴 분석

세션에서 사용자가 **반복적으로 요청하거나 수정 지시한 패턴**을 찾는다.
- 같은 종류의 수정을 여러 번 요청했다면 → 가이드에 추가 제안
- 매번 수동으로 지시하던 것이 있다면 → 자동화 또는 가이드화 제안

## 베스트프랙티스맵 점검

세션 중 구현한 패턴을 [coding-standards/map.md](../../../contexts/coding-standards/map.md)의 베스트프랙티스 섹션과 대조한다.

- 맵에 없는 새 패턴을 정립했는가 → 엔트리 추가 제안
- 기존 엔트리의 방식을 개선했는가 → 엔트리 수정 제안
- 더 이상 유효하지 않은 패턴을 발견했는가 → 엔트리 제거 제안

## 가이드 업데이트 제안

도출된 개선 사항을 정리하여 어떤 파일에 어떤 내용을 추가/수정해야 하는지 구체적으로 제안한다.
