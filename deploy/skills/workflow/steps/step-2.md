# Step 2: PR 분할 전략 수립

여러 PR로 나뉠 수 있는 큰 작업의 분할 전략을 수립합니다.

---

## 작업 크기 판단

- **작은 작업** (커밋 3개 이하): 이 단계 건너뛰고 다음 단계로
- **큰 작업** (커밋 4개 이상, 또는 여러 도메인 포함): PR 분할 전략 수립

작업이 작다고 판단되면 사용자에게 스킵을 제안하세요.

---

## 채용과제: 고정 PR 구조

채용과제는 PR1~3이 고정이다. 구체적 항목은 Step 3에서 PR별로 정의한다.

| PR | 경계 |
|----|------|
| PR1 (프로젝트 세팅) | 빌드·린트·포맷 등 static checking 도구와 설정. 런타임 코드 없음 |
| PR2 (공통 인프라) | 런타임 의존성 설치 + Provider·API 클라이언트 등 앱 인프라 구성 |
| PR3 (공통 컴포넌트) | 2개 이상 페이지에서 쓰이는 재사용 UI 컴포넌트 |

**PR1~3은 분할 판단 대상이 아니다.** 분할 판단은 PR4 이후에만 적용한다.

### PR1 ≠ FOUNDATION

채용 모드에서 PR1을 표기할 때 "담당: FOUNDATION"·"FOUNDATION이 PR1 수행/완료" 같은 단정 표현 금지. FOUNDATION은 PR1 워크트리에 **베이스 두 커밋(폴더 마이그레이션·코딩 스탠다드 마이그레이션)만** 제공하고, PR1의 본격 작업(빌드·린트·포맷·tsconfig 등 static checking 도구 설정)은 PR_1_PLAN/IMPL/WRITING이 일반 도미노로 수행한다.

기존 `project.md` PR1 섹션에 "FOUNDATION ... 완료" 같은 표현이 있으면 그대로 수용하지 말고 본 규칙에 맞춰 정정한다 (글로벌 「입력 산출물 비판적 검토」 발동). FOUNDATION이 "완료"한 것은 베이스 두 커밋이지 PR1이 아니다.

---

## PR 분할 원칙

- **한 번에 하나의 PR, 하나의 구체적인 기능**만 다룹니다.
- **의존성 순서**: 공통 모듈 → 개별 페이지
- **기술적 과제는 다음 단계에서** — 이 단계는 PR 분할에만 집중
- **PR3+ stub 시점부터 병렬**: `PR_{N}_PLAN.step-4` 종료 시점에 stub 커밋이 외부 시그니처를 확정하면, PR_{N+1}_PLAN과 PR_{N}_IMPL을 동시 spawn 가능
- **PR1·2는 인프라성 → 순차**: 빌드·린트·포맷 설정 / 패키지 설치 등이라 step-4에서 LLM 분석 결과 stub 불필요 → 작업 시작도 순차 (이전 PR 머지 후 다음 PR_{N+1}_PLAN spawn)

## 기획 점검 (TODO 식별)

PR 분할 초안 작성 후, 각 PR별로 사용자에게 사전 확인이 필요한 TODO가 있는지 확인한다.

---

## Step 1 폴더 재정비

Step 1에서 생성된 `/plan/pr{N}/` 폴더는 페이지 플로우 순서 기준이다. PR 경계가 확정되면 번호가 달라질 수 있다.

- 경계 확정 후 Step 1 폴더(`pr{N}`)를 실제 PR 번호에 맞게 rename/merge한다.
- `project.md`의 "참고 background 파일" 경로도 함께 갱신한다.

---

## 산출물: `/plan/background/consumable/project.md`

PR별로 아래 항목을 포함한다:
- PR 제목
- 범위
- 참고 background 파일
- 기획 확인 필요 사항 (TODO)

---

## 보고 내용

- 총 PR 수와 각 PR 범위 한 줄 요약
- PR 간 의존 순서
- 사용자 판단이 필요한 열린 질문 (있는 경우)

---

## Step 2 종료 — 분기점

step-2는 BG 세션의 마지막 step + 후속 세션 spawn 분기점 (SKILL.md 「세션」 표 (4) 컬럼).

「보고 내용」 출력 직후, 「세션 회고」 진입 전에 SKILL.md 「세션 spawn 안내 메커니즘」 절차를 발동하여 후속 spawn 안내 출력.

---

## 세션 회고

이 세션의 마지막 step이다. 보고 후:

- 이 세션에서 사용자가 반복 수정 지시한 패턴이 있었으면 가이드 업데이트를 제안한다.
