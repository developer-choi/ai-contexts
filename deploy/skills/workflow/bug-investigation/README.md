# Bug Investigation

버그나 성능 문제의 근본 원인을 분석하는 스킬입니다. workflow의 과제 정의 단계에서 호출됩니다.

## 왜 만들었나요?

"API가 느려요"는 증상이지 원인이 아닙니다. 증상만 보고 수정하면 근본 원인이 남아 같은 문제가 반복됩니다.

이 스킬은 증상과 근본 원인을 명확히 구분합니다. 무엇이 잘못되고 있는지(증상), 어떻게 발견했는지(발견 경로), 사업적 영향은 무엇인지(비즈니스 임팩트), 왜 발생하는지(Root Cause)를 정리하여 이후 구현 단계의 기반을 만듭니다.
