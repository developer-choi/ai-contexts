# 부정 명시 메아리 자가 점검 — 정규식 패턴

[SKILL.md](../SKILL.md) 「부정 명시 메아리 자가 점검」 절차의 step 1에서 사용한다. 산출물 텍스트에 다음 패턴 중 하나라도 매치되는 라인이 있으면 분류 단계로 넘어간다.

## 패턴 분류

### 부정 동사

사용자가 "X 쓰지 마"라고 했을 때 산출물에 "X 안 쓴다"로 메아리하는 형태.

- `안 만든다`, `만들지 않[는음다]`, `만들지 않을`, `만들지 않음`
- `미적용`, `미포함`, `적용하지 않`, `적용 안`, `적용 안 함`
- `추가하지 않`, `추가 안 함`, `추가 안함`
- `사용하지 않`, `사용 안 함`, `사용 안함`
- `도입하지 않`, `도입 안 함`, `도입 안함`
- `다루지 않`, `포함하지 않`, `포함 안 함`
- `건드리지 않`, `건드림 없`
- `안 쓴`, `안 쓰[는음다]`, `쓰지 마[라]?`, `쓰지 않[는음다]`
- `끄지 마[라]?`, `끄지 않[는음다]`
- `만들지 마[라]?`, `넣지 마[라]?`, `두지 마[라]?`

### 발화 원문 인용 (우회 패턴)

LLM이 위 부정 동사 메아리를 회피하려고 사용자 발화 원문을 인용 블록·"사용자 발화" 섹션 등에 박는 우회 형태. 인용이라도 산출물에 부정 지시가 박히면 사용자 메아리에 해당한다.

- `사용자 발화\s*[:：]` (산출물에 사용자 발화 인용 섹션 등장)
- `^>\s*["“].{0,80}(쓰지 마|끄지 마|만들지 마|안 쓴|안 만|넣지 마|두지 마)` (quote 블록 안 부정 지시)
- `> ".+(쓰지 마|끄지 마|만들지 마|안 쓴|안 만)` (인용 안 부정 지시)

**판정**: 인용이라도 사용자가 적지 말라고 한 어구가 산출물에 등장하면 메아리. 발화 인용 섹션 자체가 노이즈 — 사용자는 자기 발화를 안다. 결정의 채택안과 근거만 적고, 발화 인용은 박지 않는다.

### 거부 대안 흔적 (우회 패턴)

LLM이 위 두 패턴을 회피하려고 "검토했다"·"초안"·"기각" 같은 과거 행위 서술 형식으로 거부 대안을 박는 우회 형태. 부정 동사 없이도 거부 흔적을 산출물에 남기면 메아리에 해당한다.

- `LLM 초안\s*[:：]`, `초안\s*[:：].{0,80}(검토|도입 검토|대안 검토)`
- `검토한 (대안|선택지)\s*[:：]`, `대안 검토\s*[:：]`
- `기각된 대안`, `거부된 대안`, `채택하지 않은`
- `(\S+) (vs|또는) (\S+).{0,30}검토` (예: "axios vs fetch 검토")

**판정**: 사용자가 부정한 라이브러리·패턴·도구를 "검토 흔적"·"초안"·"비교 대상" 등 어떤 형식으로든 산출물에 남기면 메아리. 채택안만 적고, 거부된 것은 적지 않는다. 사용자가 "X 쓰지 마"라고 했으면 X는 산출물 어디에도 등장하지 않아야 한다 — 본문이든, 메타 섹션이든, 인용이든, 검토 흔적이든.

### 변경 부재

"기존을 그대로 두고 새로 만들지 않는다"의 자기현시 형태.

- `변경 없음`, `변경 안 함`, `변경하지 않`
- `보존만`, `그대로 유지`
- `시그니처 그대로`, `시그니처 보존`, `외부 API 보존`

### 스코프 가면

"이번 PR에선 안 한다"·"다음 단계 예정" 같이 스코프 선언으로 메아리를 위장하는 형태.

- `이번 (범위|PR|단계).{0,20}(제외|에선 안|에서 안)`
- `이 (단계|범위).{0,20}(제외|에선 안)`
- `다음 단계.{0,20}예정`
- `후속 (PR|단계).{0,20}예정`
- `별도 PR(에서)?`, `별도 단계`
- `스코프 외`, `범위 (외|밖)`

### 부재 표현

자체 판단이라며 근거 없이 부재만 적는 형태.

- `별도 .{0,30}(없음|안 만|안 함|만들지 않)`
- `폐기` (단독 — 근거 없이)

## 분류 가이드 (step 2 보조)

### 사용자 메아리 판정

사용자 발화의 부정 지시(`negative_directives`로 식별)와 산출물의 부정 표현이 의미적으로 매칭되면 **사용자 메아리**. 매칭 기준:

- 같은 단어·약어·풀네임
- 같은 라이브러리·기능·파일을 다른 표현으로 가리킬 때

### 근거 동반 판정

같은 문장 또는 즉시 인접 라인(±1)에 다음 중 하나가 명시되어 있으면 **근거 동반**:

- 측정값·수치 (예: "100ms 지연")
- 기존 코드베이스·패턴 참조 (예: "MP에 같은 패턴 없음")
- BG 단계 결정 인용 (예: "BG #3 결정")
- 충돌 지점 명시 (예: "starter Geist 폰트와 충돌")

근거가 없거나 일반론적 진술("성능 트레이드오프")만 있으면 **근거 없음**.
