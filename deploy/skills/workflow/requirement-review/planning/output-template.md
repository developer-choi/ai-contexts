# 명세 분석 산출물 템플릿

AI가 초안을 채우고 사용자가 리뷰하는 명세 분석 양식. 산출물 저장 위치는 [conventions/plan-folder.md](../../conventions/plan-folder.md) 참조.

PR 번호는 **확정 순서대로** 매긴다 — 확정할 때마다 다음 번호를 준다. 사용자 플로우 순서도, 의존 순서도 아니다 ([../../conventions/pr-split.md](../../conventions/pr-split.md) 「산출물」). 의존은 번호가 아니라 각 PR 절의 의존 항목이 표현한다. 각 page.md 상단에 페이지명을 H1으로 적어 어떤 페이지인지 식별한다.

---

## global.md 템플릿

```markdown
# [기능명] 명세 분석

## 개요

- 서비스: [서비스 한 줄 설명]
- 목적: [이 기능의 목적]
- 대상 사용자: [대상 사용자]
- 핵심 플로우: [페이지 순서 요약]

## 공통 컴포넌트

### [컴포넌트명]

- props: [prop 목록]
- 상태: [신규/기존]
- 사용 페이지: [pr1, pr2]
- 구현 메모:
  - [구현 메모]

## 공통 로직

- [여러 페이지에 걸쳐 적용되는 로직]

## 공통 레이아웃

- [여러 페이지에 걸쳐 적용되는 레이아웃]

## 반응형 정책

- [모바일 온리 / PC+Mobile / 등]

## 공통 엣지케이스

- [여러 페이지에 걸쳐 적용되는 엣지케이스]

## 기획자/디자이너 질문 목록

- [전체 페이지 종합 질문]
```

## layout.md 템플릿

```markdown
# [레이아웃명]

- 적용 페이지: [pr1, pr2, ...]
- [간단한 역할]

## 레이아웃 구조

- [헤더 / 사이드바 / 푸터 등 고정 영역과 콘텐츠 영역의 배치]

## 공통 UI 요소

- [네비게이션, 브레드크럼, 사이드바 메뉴 등 레이아웃에 포함되는 요소]

## 레이아웃 메타데이터

- 타이틀: [브라우저 탭에 표시될 페이지 제목]
- OG 타이틀: [공유 시 표시될 제목]
- OG 디스크립션: [공유 시 표시될 설명]
- OG 이미지: [공유 시 표시될 이미지]
```

## page.md 템플릿

```markdown
# [페이지명]

- URL: [/path/:param]
- [간단한 역할]

## 페이지 렌더링

- 서버 캐싱: [static / revalidate / dynamic]
- 컴포넌트: [Server / Client]

## 페이지 메타데이터

- 타이틀: [브라우저 탭에 표시될 페이지 제목]
- OG 타이틀: [공유 시 표시될 제목]
- OG 디스크립션: [공유 시 표시될 설명]
- OG 이미지: [공유 시 표시될 이미지]

## API 목록

### [GET/POST/PUT/DELETE] [API 이름]

- parameter: [요청 파라미터]
- success response: [성공 응답 형태]
- error response: [에러 응답 형태]
- 에러처리: [모달 or Error Boundary]
- 로딩처리: [버튼 로딩 or 스켈레톤]
- 성공처리: [특별한 동작이 필요한 경우 작성]

## 동작 명세

이 페이지의 입출력 동작 규칙(WHAT)을 적는다. 구현 방법(함수 시그니처·파라미터/리턴 타입, "TDD 테스트케이스" 형식)은 PLAN/IMPL이 정하므로 여기서는 다루지 않는다.

### 동작 규칙

`입력 상황 → 결과` 형식. 순수 기획(입출력 내용)이라 구현 선택과 무관하다. 함수명·타입·리턴타입은 붙이지 않는다.

```
- [입력 상황] → [결과]
- [엣지케이스 입력] → [결과]
```

<ai-only>
엣지케이스를 반드시 포함합니다. 입력의 극단 케이스(빈 값, 0, null, 경계값 등)를 확인하고, 누락이 있으면 짚습니다. 명세에 없는 칸은 지어내지 말고 `[?] (기획 공백 — BG 에스컬레이션)`으로 표시해 사용자·기획자 확인을 받습니다. 산문으로 압축할 때 규칙이 틀어지지 않게 주의합니다(예: "당시기기가 없으면"을 "기기가 다르면"으로 뭉개지 않기).
</ai-only>

### 컴포넌트 동작

`조건 → 기대 결과` 형식.

```
### [컴포넌트명]
- [props/상태 조건] → [기대 렌더링 결과]
- [사용자 인터랙션] → [기대 동작]
```

### 페이지 동작

`사용자 행동 → 결과` 형식.

```
### [페이지명]
- [URL/진입 조건] → [기대 렌더링]
- [사용자 행동] → [기대 결과]
- [에러 조건] → [기대 에러 처리]
