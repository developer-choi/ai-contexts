# 기획서 리뷰 가이드

## 슬롯

### 준비 방식

#### 맥락 수집

리뷰 시작 전, 사용자에게 맥락을 질문합니다. 사용자가 이미 설명했다면 해당 항목은 생략합니다.

- 이 서비스가 무엇을 하는 서비스인지 (예: 외국어 화상 수업 플랫폼, 이커머스 등)
- 이번에 리뷰할 기능이 무엇인지 (한 줄 요약)
- 신규 서비스/기능인지, 기존 기능 개선인지
- 알아두어야 할 배경이나 제약사항이 있는지

#### 산출물 생성

[output-template.md](./output-template.md)의 global.md 템플릿을 `/plan/background/global.md`에, page.md 템플릿을 `/plan/pr1/page.md`에 생성합니다. 이 시점에서는 빈 템플릿으로 생성하고, 자료를 받은 후 초안을 채웁니다.

### 입력 자료

페이지 단위의 기획서 자료. 예: 캡처 이미지, 피그마 링크, 문서 파일

### 체크리스트

기획 체크리스트를 적용합니다. `checklist/` 하위 파일을 참조합니다.

페이지 유형(폼, 리스트, 상세, 순차 플로우 등)을 식별하고, 해당하는 `../page-type/` 체크리스트를 추가 적용합니다.

### 산출물

- `/plan/background/global.md` — **생성**. 전체 서비스 맥락, 공통 컴포넌트, TODO 목록
- `/plan/pr{N}/page.md` — **생성**. 페이지별 분석 결과 (N은 사용자 플로우 순서)
- `/plan/background/layout.md` — **필요 시 생성**. 여러 페이지가 공유하는 레이아웃이 식별된 경우

### 분석 과정

#### 자료 받기

한 페이지의 리뷰가 끝나면, [output-template.md](./output-template.md)의 page.md 템플릿으로 다음 `/plan/pr{N}/page.md`를 생성합니다. 사용자에게 다음 페이지 자료를 요청합니다.

사용자가 자료를 제출하면, 이미지인 경우 `/plan/pr{N}/` 하위로 이동합니다. 1장이면 `page.png`, 여러 장이면 식별 가능한 이름을 붙입니다 (예: `page-list.png`, `page-detail.png`).

- 이미지가 5개 이상이면 사용자에게 리뷰 대상이 아닌 것은 제거하라고 안내합니다.
- 현재 리뷰 대상과 관련 없어 보이는 자료가 있으면 사용자에게 알립니다.

#### 같이 분석

AI가 자료와 체크리스트를 기반으로 `pr{N}/page.md`의 초안을 채웁니다:

1. 페이지 유형을 식별하고, 해당하는 `../page-type/` 체크리스트를 추가 적용합니다.
2. `checklist/`와 `../page-type/` 하위 체크리스트를 적용합니다.
3. 템플릿의 각 섹션을 자료에서 파악한 내용으로 채웁니다. 확신이 없는 항목은 `[?]`로 표기합니다.
4. 기획서에서 모호하거나 빠진 정의를 짚고, 왜 문제인지 설명합니다.
5. 엣지케이스를 선제시합니다.
6. UX 개선 제안 (해당 시에만): 더 나은 대안이 있을 때 선택지를 제시합니다. 단정하지 않고 장단점을 함께 설명합니다.
7. 초안과 발견한 문제를 사용자에게 제시합니다.

#### 공통화 검토

직전 페이지와 겹치는 컴포넌트나 UI 패턴이 있으면 사용자에게 알리고, global.md 공통 컴포넌트로 올릴지 검토합니다. 여러 페이지가 공유하는 레이아웃이 식별되면 사용자에게 layout.md를 생성할지 확인합니다.

#### 종합

모든 페이지 리뷰를 마친 후, 페이지 단위를 넘어 전체 플로우 관점에서 점검합니다.

1. [cross-page.md](./checklist/cross-page.md)의 체크리스트를 적용합니다.
2. 공통 컴포넌트/로직/레이아웃을 식별합니다.
3. TODO 목록을 최종 정리합니다.

AI가 `global.md`와 필요 시 `layout.md`의 초안을 채웁니다. 사용자가 리뷰합니다.
