# 디자인 시안 리뷰 가이드

기획서 리뷰(planning)의 산출물(page.md, global.md)이 이미 있으면 보완하고, 없으면 디자인 시안을 기반으로 신규 생성한다.

## 슬롯

### 준비 방식

#### 맥락 수집

기획서 리뷰가 선행된 경우 그 맥락을 기반으로, 아닌 경우 처음부터 확인합니다:

- 디자인 시스템이나 스타일 가이드가 있는지
- 디자인 시안의 범위 (기획서와 다른 부분이 있는지)

### 입력 자료

페이지 단위의 디자인 시안 자료. 예: 캡처 이미지, 피그마 링크, 문서 파일

### 체크리스트

`checklist/` 하위의 디자인 체크리스트를 적용합니다.

페이지 유형(폼, 리스트, 상세, 순차 플로우 등)을 식별하고, 해당하는 `../page-type/` 체크리스트를 추가 적용합니다.

### 산출물

기존 산출물이 있으면 보완하고, 없으면 신규 생성합니다.

- `/plan/background/global.md` — 디자인 관련 공통 컴포넌트, TODO
- `/plan/pr{N}/page.md` — 디자인 분석 결과

### 분석 과정

#### 같이 분석

체크리스트 슬롯에서 정의한 체크리스트를 `pr{N}/page.md`에 적용합니다:

1. 디자인 상세(간격, 색상, 타이포, 반응형, 인터랙션 등)를 짚습니다. 기획서 리뷰가 선행된 경우, 기획서에서 다루지 못한 부분을 중점적으로 봅니다.
2. 디자인과 기획서 간 불일치가 있으면 사용자에게 알립니다.
3. 초안과 발견한 문제를 사용자에게 제시합니다.

#### 공통화 검토

직전 페이지와 겹치는 컴포넌트나 UI 패턴이 있으면 사용자에게 알리고, global.md 공통 컴포넌트로 올릴지 검토합니다. 여러 페이지가 공유하는 레이아웃이 식별되면 사용자에게 layout.md를 생성할지 확인합니다.

#### 종합

planning 플래그도 켜져 있으면, planning과 design의 종합을 마지막에 한 번 통합 처리한다.
design 단독(planning 꺼짐)이면 design 관점의 종합만 수행한다: global.md 공통 컴포넌트 정리, 페이지 간 디자인 패턴 일관성 점검.
