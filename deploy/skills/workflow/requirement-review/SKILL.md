# 요구사항 리뷰

## 역할

- **AI**: 요구사항을 비판적으로 분석하고, 발견한 문제를 사용자에게 설명한다. 모호한 부분을 짚고, 엣지케이스를 선제시하며, 개선안을 제시한다. 단정하지 않고 장단점을 함께 설명한다.
- **사용자**: AI가 제시한 내용을 확인/판단한다. 추가 자료를 제공한다.

## 입력 타입

argument로 플래그를 받는다. 각 플래그는 독립적으로 켜거나 끌 수 있으며 자유 조합 가능하다.

| 플래그 | 유형 | 가이드 | 설명 |
|--------|------|--------|------|
| recruitment | 독립 절차 | [recruitment/guide.md](recruitment/guide.md) | 채용과제 요구사항 리뷰 |
| planning | 페이지 단위 루프 | [planning/guide.md](planning/guide.md) | 기획서 리뷰 |
| design | 페이지 단위 루프 | [design/guide.md](design/guide.md) | 디자인 시안 리뷰 |

### 실행 순서

플래그 조합에 관계없이 아래 순서를 따른다:

1. **recruitment** — 켜져 있으면 먼저 실행한다. 가이드가 정의한 자체 절차를 처음부터 끝까지 따른다. 페이지 단위 루프를 사용하지 않는다.
2. **planning + design 루프** — 켜진 플래그에 해당하는 가이드를 동시에 적용하며, 페이지마다 루프를 반복한다. planning만, design만, 또는 둘 다 켤 수 있다. planning만 켜고 세션을 종료한 뒤, 이후 세션에서 design만 단독으로 실행하는 것도 지원된다.

종합 단계는 모든 플래그의 루프가 끝난 후 한 번만 수행한다. planning과 design이 모두 켜져 있으면 통합 종합으로 처리한다.

### 가이드 슬롯

각 가이드가 정의하는 슬롯:

| 슬롯 | 설명 | 페이지 단위 루프에서의 역할 |
|------|------|---------------------------|
| 준비 방식 | 리뷰 시작 전 수행할 것 | 루프 진입 전에 실행 |
| 입력 자료 | 사용자에게 요청할 자료 | 루프 1단계에서 참조 |
| 체크리스트 | 분석 시 적용할 체크리스트 | 루프 2단계에서 적용 |
| 산출물 | 분석 결과 파일 | 루프 3~4단계에서 기록 |
| 분석 과정 | 입력 타입별 분석 절차 | 루프의 페이지별 단계(1~2단계)를 구체화하고, 루프 종료 후 단계(공통화 검토, 종합)를 정의 |

독립 절차(recruitment)는 페이지 단위 루프를 사용하지 않는다. 가이드가 정의한 자체 절차(공동 탐색 → 평가 기준 추론 → 프로젝트 결정 → 모호한 부분)를 처음부터 끝까지 따른다. 슬롯은 가이드 내부에서 자체적으로 참조된다.

## 페이지 단위 루프

planning, design 중 하나라도 켜져 있으면, 플래그별 준비가 끝난 후 아래 사이클을 페이지마다 반복한다. 각 가이드의 슬롯을 참조하여 실행한다.

planning과 design이 모두 켜져 있으면, 동일 페이지에서 planning 확정(3~4단계) 완료 후 design 분석을 시작한다. 루프를 두 번 돌지 않는다.

### 1. 자료 참조 (수집은 step-1.1)

리뷰 대상 자료는 step-1.1이 수집해 `background/`에 저장한 것을 참조한다. **처음부터 다시 요청하지 않는다** — 빠진 페이지·항목이 있으면 그 갭만 콕 집어 요청한다. 갭 요청 방식은 가이드의 준비 방식 슬롯을 따른다.

### 2. 같이 분석

가이드의 체크리스트 슬롯을 기반으로 AI와 사용자가 같이 분석한다. 구체적인 분석 과정은 가이드의 분석 과정 슬롯이 이 단계를 구체화(override)한다.

`/plan/background/` 하위에 기존 산출물이 있으면, 현재 페이지에 해당하는 내용을 page.md 초안에 가져온다.

### 3. 논의

`/discussion`을 사용하여 사용자와 대화하며 미결 항목의 결론을 낸다.

- 확정된 내용 → 산출물에 기록
- 사용자가 판단할 수 없는 항목 → TODO로 분류
- 논의가 끝나지 않은 항목 → "미확인"으로 표기

### 4. 확정

논의 결과를 산출물에 반영한다. 사용자가 수정한 말투나 표현은 다듬지 않는다. 사용자에게 산출물을 확인하라고 안내한다. 수정할 부분이 있으면 반영하고, 없으면 다음 자료로 넘어간다.

2단계에서 기존 산출물로부터 가져온 내용은 원본에서 삭제한다. 공통 항목은 page.md 대신 global.md로 이관한다.
