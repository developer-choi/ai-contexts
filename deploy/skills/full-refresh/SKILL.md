# Full Refresh

최근 커밋이 있는 프로젝트를 탐지하여 프로젝트별 최신화 스킬로 검증하고, 위반이 있으면 인수인계 문서를 작성한다. 기간은 인자로 받는다 (기본값: 7d).

## 프로젝트 레지스트리

| 약어 | 경로 | 최신화 스킬 | 스킬 경로 |
|------|------|-------------|-----------|
| KA | `~/WebstormProjects/main/knowledge-archive` | validate | `.claude/skills/validate/SKILL.md` |

## 실행 흐름

### Phase 1: 커밋 탐색

레지스트리의 각 프로젝트에서 지정된 기간 내 커밋 존재 여부를 확인한다. 커밋이 있는 프로젝트만 필터링하고 결과를 보고한다.

커밋이 있는 프로젝트가 없으면 여기서 종료한다.

### Phase 2: 검증 계획 제출

커밋이 감지된 프로젝트 중 최신화 스킬이 등록된 프로젝트에 대해 검증 계획을 제출한다.

사용자가 조정할 수 있다 ("이건 이미 했으니 빼줘" 등). 승인 후 Phase 3으로 진행한다.

### Phase 3: 검증 실행

승인된 계획에 따라 프로젝트별로 서브에이전트를 위임하여 검증한다.

서브에이전트 지시:
- 대상 프로젝트의 스킬 파일(SKILL.md)을 읽고, 기술된 검증 규칙에 따라 위반을 탐지하라
- 파일을 수정하지 마라. 위반 목록만 정리하여 반환하라

**위반이 없으면** 완료 보고 후 종료한다.

**위반이 있으면** 인수인계 문서를 작성한다.

#### 인수인계 문서

새 세션에서 다시 코드를 뒤지지 않고 바로 해결할 수 있는 수준으로 작성한다.

각 위반 항목에 포함할 내용:
- **위반 내용**: 어떤 규칙을 어떻게 위반했는가
- **위치**: 파일 경로와 해당 라인/섹션
- **해결 방법**: 구체적인 수정 방향

인수인계 문서를 사용자에게 제시하고, "해당 프로젝트에서 위반을 해결하세요"로 안내하며 종료한다.

## 범위

- **포함**: 커밋 탐색, 검증 계획/실행, 인수인계 문서
- **제외**: 위반 수정 (사용자가 새 세션에서 직접), 프로젝트 최신화 (향후 이터레이션)
