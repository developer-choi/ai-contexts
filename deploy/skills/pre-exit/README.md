# Pre-exit

세션이 끝날 때 회고를 수행하는 스킬입니다.

## 동기

같은 실수를 두 번 하고 싶지 않았습니다.

AI와 작업하다 보면 방향을 수정하거나, 잘못된 결과물을 반려하는 일이 생깁니다. 이런 경험을 그냥 넘기면 다음 세션에서 같은 일이 반복됩니다. 세션이 끝날 때마다 무엇이 잘못됐는지, 어디에 규칙으로 남겨야 하는지를 정리하면 다음 세션부터는 같은 지적을 하지 않아도 됩니다.

이 회고 과정 자체를 프롬프트로 만든 것입니다.

## 회고에서 하는 일

- 반복된 실수 패턴을 찾아 규칙화할 위치를 제안합니다
- 사용자가 반려하거나 수정한 것을 규칙 문안과 before/after 사례로 함께 축적합니다
- 세션에서 얻은 교훈은 그것이 활용될 맥락에 직접 심어둡니다. 컨벤션이면 컨벤션에, 워크플로우 개선이면 스킬에. 사람이 기억할 필요 없이 AI가 해당 맥락에서 자동으로 꺼내 씁니다
- 커밋되지 않은 변경사항을 확인합니다

## 보강 회고 분기

세션 종류에 따라 추가로 챙겨야 할 회고 항목이 있으면 별도 보강 파일로 분리해 둡니다. 기본 회고 절차 진입 전에 인자나 자동 감지(예: `plan/pr{N}/**` 변경)로 매칭된 `augmentations/<키워드>.md`를 먼저 실행하고, 발견 항목은 일반 회고 목록에 합류시킵니다.

분리 기준은 하나입니다. **그 스킬을 안 쓴 세션에서도 할 일이 있는 회고면 본 절차에, 그 스킬이 돈 세션에서만 의미 있는 회고면 보강에** 둡니다. 신규 보강 작성 절차는 [SKILL.md](SKILL.md)의 「보강 회고 분기」를 참고하세요.

## 팀 에이전트 통합

세션에서 띄운 팀 에이전트·서브에이전트가 있으면, 본 회고를 시작할 때 각 에이전트에게 회고를 요청해 결과를 회수한 뒤 종료합니다. 에이전트가 자기 컨텍스트에서 본 문제를 메인 회고와 함께 다루기 위함입니다.

> 자세한 내용은 [SKILL.md](SKILL.md)를 참고하세요.
