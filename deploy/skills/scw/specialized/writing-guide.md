# Writing-Guide 특화 체크리스트

`contexts/writing-guide/` 리뷰 시 추가로 점검하는 항목. 탐색 절차는 `map.md` 참조(Glob + 각 파일 frontmatter `tags` 기반, 수기 목록 없음).

## [CRITICAL] 분류 검증

다른 문서가 역할 기준으로 파일을 참조하므로, 분류가 틀리면 규칙 누락이나 잘못된 로드가 발생한다.

- **규칙** (tone.md): "~해라, ~금지" 형태. AI가 텍스트를 대조할 때 참조
- **사례** (example.md): Bad/Good 쌍. 규칙의 구체적 적용 예시
- **가이드**: 특정 문서 유형의 작성 절차/구조 (readme-guide, pr-body-* 등)
- 판단 기준: "이것은 규칙인가 사례인가 가이드인가?" — 혼합된 파일이 있으면 분리 검토

## 규칙-사례 연결성 검증

- 사례(example.md)에 관련 규칙 태그가 있는지
- 규칙(tone.md)에 대응하는 사례가 있는지 (없어도 에러는 아니나, 커버리지 파악)
