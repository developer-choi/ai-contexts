# Backlog

개발하면서 떠오른 작업 아이디어와, 제가 다루는 여러 저장소의 작업·지식·참고 자료를 모으고, 분류하고, 정리하는 기능입니다. 제가 직접 불러낼 때만 동작합니다.

## 동기

개발하다 보면 "이건 스킬로 만들면 좋겠다", "이 스킬에 이 기능을 추가하고 싶다" 같은 아이디어가 떠오릅니다.

메모를 쌓는 건 쉽지만, 관리하는 데 시간이 많이 들었습니다. 새 메모가 기존에 적어둔 것과 겹치는지 확인하고, 겹치면 합치고, 주제별로 분류하고, 우선순위를 매기는 과정이 반복됐습니다.

이 정리 작업 자체를 AI에게 맡기기로 했습니다.

메모를 넘기면 AI가 갈래별로 분류하고, 기존 백로그와 합치고, 기존 스킬·프로젝트와 어떻게 이어지는지 파악합니다. 준비가 다 된 항목은 그 내용이 반영될 곳(설정 규칙, 안내 문서, 다른 저장소의 코드 등)에 실제로 옮겨 적용합니다.

## 두 갈래로 나눠 담기

백로그는 크게 두 갈래로 나뉩니다.

- 저장소별 작업·지식·참고: 제가 다루는 저장소마다 해야 할 일, 알게 된 지식, 나중에 볼 참고 자료를 모아둡니다.
- 기술블로그 발행 재료: 블로그에 올릴 글의 재료를 글 한 편 단위로 모아둡니다.

이 중 일부는 할 일 목록처럼 관리하고, 나머지는 자유롭게 모아두기만 합니다. 제가 스스로 고쳐 나가는 설정 저장소의 할 일들은 준비 정도를 표시해 두고 준비가 끝난 것부터 처리합니다. 그 밖의 자료는 준비 정도 없이 편하게 쌓아둡니다.

## 작업 준비 상태 점검

할 일 항목은 다른 날, 다른 컴퓨터에서 제가 다시 열었을 때 추가 질문 없이 바로 시작할 수 있는 수준이 되면 "준비됨"으로 표시합니다.

이 표시를 붙이기 전에, 검토를 맡은 AI가 사람 대신 몇 가지를 확인합니다. 인용한 외부 자료의 핵심이 항목 안에 함께 담겨 있는지, "어제"·"방금"처럼 그때만 통하는 표현 대신 정확한 시점이 적혀 있는지, 왜 필요한지·언제 끝나는지·무엇부터 시작할지가 빠짐없이 담겨 있는지를 확인해 표시를 갱신합니다.

## 항목에 담는 내용

할 일 항목 하나는 파일 하나로 관리합니다. 각 항목에는 제목과 준비 정도를 적어두고, 본문에는 다음 세 가지를 담습니다.

- 기대하는 최종 모습: 어떤 상태가 되면 이 일이 끝나는지
- 지금의 모습: 현재 어떻게 되어 있어서 기대와 어긋나는지. 이 일이 왜 필요한지의 근거입니다.
- 지금 떠올린 방법: 어떻게 풀지에 대한 후보입니다. 적어둔 시점의 생각일 뿐이라, 나중에 실제로 작업할 때는 더 나은 방법이 보이면 그쪽을 따릅니다.

## 직접 부를 때만 동작
