# Bug Investigation

버그·성능 문제의 증상과 근본 원인(Root Cause)을 구분해 분석하는 스킬입니다.

## 왜 만들었나요?

"API가 느려요"는 증상이지 원인이 아닙니다. 증상만 보고 수정하면 근본 원인이 남아 같은 문제가 반복됩니다.

이 스킬은 증상과 근본 원인을 명확히 구분합니다. 무엇이 잘못되고 있는지(증상), 어떻게 발견했는지(발견 경로), 사업적 영향은 무엇인지(비즈니스 임팩트), 왜 발생하는지(Root Cause)를 정리하여 이후 수정·구현의 기반을 만듭니다.

같은 코드를 여러 번 고쳐도 버그가 안 잡히는 **수정 루프**도 이 스킬이 감지합니다 — 무작정 더 고치는 대신 멈추고, 증상·시도한 접근·의심 원인을 보고한 뒤 개입 방식을 사용자가 정하게 합니다.
