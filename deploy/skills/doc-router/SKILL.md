# Doc Router

구글 문서의 혼합 콘텐츠를 KA와 DP로 분류하여 분리한다. 분리 후 변환은 각 프로젝트에서 별도로 수행한다.

## 입력

사용자에게 구글 문서를 **PDF와 MD 두 형식으로 export**하도록 요청한다. 이미 제공된 경우 건너뛴다.

- PDF: 이미지, 레이아웃, 서식 확인용
- MD: 텍스트 추출, 하이퍼링크 URL 확보용

## 분류 기준

| | KA (knowledge-archive) | DP (dsa-playground) |
|---|---|---|
| 성격 | 개념/정의/원리 | 코딩 팁/패턴/API 시간복잡도 |
| 판단 질문 | "이게 뭐야?" | "이렇게 하면 안 돼" |
| 예시 | Array 정의, Random Access가 O(1)인 이유 | push vs concat 성능 차이, slice() O(n) |

## 분류 단위

문서마다 구조가 다르다. 고정된 단위(Q&A, 섹션 등)를 가정하지 않는다. 문서의 실제 구조를 파악한 뒤 내용의 성격이 바뀌는 경계를 찾아 분리한다.

## 버리는 내용

KA/DP 어디에도 속하지 않는 내용은 산출물에서 제외한다. 제외한 내용 목록은 사용자에게 알려준다. 구체적인 버림 기준은 gotchas 참고.

## 애매한 내용

KA/DP 경계가 불분명한 내용은 가장 적합한 행선지를 제안하고 사용자 확인을 받는다. 사용자가 판단한 결과는 즉시 아래 gotchas 섹션에 기록한다.

## 산출물

분류 결과를 KA 부분과 DP 부분으로 명확히 분리하여 출력한다. 사용자가 각 프로젝트에서 변환 작업을 할 때 바로 입력으로 사용할 수 있어야 한다.

## 후속 변환 수행

분류 결과를 사용자에게 확인받은 후, 서브에이전트를 사용하여 각 프로젝트에서 변환을 직접 수행한다.

- KA 내용 → 서브에이전트가 KA 프로젝트(`~/WebstormProjects/main/knowledge-archive/`)의 convert 스킬 파일을 찾아 읽고, 그 절차대로 변환 수행
- DP 내용 → 서브에이전트가 DP 프로젝트(`~/WebstormProjects/main/dsa-playground/`)의 `docs/주제.md`에 추가

스킬 간 하드코딩 의존 없이, 대상 프로젝트의 스킬 파일을 런타임에 읽어서 수행한다.
