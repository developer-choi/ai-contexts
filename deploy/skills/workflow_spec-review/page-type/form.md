# 폼 페이지

사용자 입력이 주요 인터랙션인 페이지에 추가 적용한다.

## 포커스

- 페이지 첫 접근 또는 윈도우 포커스 시, 메인 폼 요소에 autoFocus가 필요한가?

## 입력 유효성 검증

- 필수/선택 필드 구분이 되어 있는가?
- 각 필드의 타입과 포맷이 정의되어 있는가? (문자열 길이, 숫자 범위, 날짜 형식 등)
- 클라이언트 검증과 서버 검증의 범위가 구분되어 있는가?
- 검증 실패 시 에러 메시지가 정의되어 있는가?

## 이탈 보호

- "뒤로가기" 시 이전 입력값이 유지되는가, 초기화되는가?
- "취소" 시 확인 다이얼로그가 필요한가?

## 제출

- 폼 제출 실패 시 입력값이 유지되는가?
- 버튼 더블클릭 방지가 필요한가?

## 환경

- 모바일 키보드가 올라올 때 레이아웃 영향
