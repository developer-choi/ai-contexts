# 리스트 페이지

데이터 목록을 나열하는 페이지에 추가 적용한다.

## 빈 상태

- 데이터가 0건일 때 화면이 어떻게 보이는가?
- 데이터 변환/필터링 결과가 0건일 때 처리가 정의되어 있는가? (API 응답은 있지만 프론트 로직이 걸러낸 결과 빈 값이 되는 경우)
- 검색 결과가 없을 때 안내 문구/액션이 정의되어 있는가?
- 최초 사용자(첫 진입)와 데이터를 삭제한 사용자를 구분하는가?

## 대량 데이터

- 목록 아이템이 매우 많을 때 (페이지네이션? 무한스크롤? 제한?)

## 검색/필터

- 쿼리스트링으로 관리할 상태가 있는가? (필터, 탭, 페이지네이션 등)
- 검색 결과 0건 시 안내 및 대안 제시
- 빠른 연속 입력 시 디바운스 처리
- 필터 조합이 불가능한 경우 안내
