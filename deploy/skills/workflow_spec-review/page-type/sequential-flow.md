# 순차 플로우

여러 페이지를 순서대로 거쳐야 하는 플로우에 추가 적용한다. (온보딩, 회원가입 단계 등)

## 네비게이션

- A에서 B로 보낼 때 push vs replace 중 무엇이 맞는가? (뒤로가기로 이전 단계에 돌아가야 하면 push, 돌아가면 안 되면 replace)
- 중간 단계 URL 직접 접근 시 이전 데이터 없음 → 리다이렉트

## 이탈/복귀

- 중간에 이탈했다가 다시 돌아왔을 때 이어서 진행 가능한가?
- 뒤로가기 시 이전 입력값 유지
- 브라우저 새로고침 시 현재 단계 데이터 유실 여부
- 앱/탭 종료 후 재진입 시 이어서 진행 가능 여부
