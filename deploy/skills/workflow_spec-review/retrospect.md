# 자유 검토 및 체크리스트 회고

정해진 점검 항목을 실행하는 것만으로는 모든 문제를 잡을 수 없다. 체크리스트는 "최소한 이건 해라"일 뿐이다. 점검 항목 실행이 끝난 뒤 반드시 아래 절차를 수행한다.

## Step 1. 자유 검토

체크리스트와 무관하게 기획서와 디자인 시안을 처음부터 다시 보고, 체크리스트가 잡지 못한 문제를 찾는다.

- 체크리스트 항목에 해당하지 않지만 기획/구현상 문제가 있는 부분이 있는지
- 체크리스트가 다루지 않는 관점(사업적 맥락, 기술 부채, 타 기능과의 충돌, 디자인 시스템 일관성, 접근성 등)에서 문제가 없는지

## Step 2. 체크리스트 회고

이번 리뷰에서 체크리스트가 얼마나 유효했는지 되돌아본다.

- 체크리스트의 각 항목을 다시 열어, 실제 리뷰 결과와 1대1로 대조한다
- 지적이 없었던 항목: 기획/디자인이 잘 되어 있어서인지, 아니면 해당 유형의 문제를 체크리스트가 아예 잡지 못하는 건지 판별
- 자유 검토에서 발견한 문제: 체크리스트의 어디에 어떤 항목을 추가하면 잡을 수 있었는지 특정
- 누락한 절차가 있다면: 무엇을 누락했고, 왜 누락했는지(지시가 모호/단순 실수/해당 기획서에서 불필요) 원인 분석

## Step 3. 개선 제안

착안점 예시:

- **반복 패턴의 규칙화**: 매번 같은 유형의 모호함이나 엣지케이스가 나온다면, checklist/에 항목으로 추가
- **암묵적 판단의 명문화**: 판단 기준 없이 감으로 결정한 것이 있다면, 해당 기준을 체크리스트에 문서화
- **산출물 템플릿 개선**: output-template.md에서 빠진 섹션이나 불필요한 섹션이 있었다면 수정 제안
- **리뷰 흐름 개선**: 캡처 순서, 질문 타이밍, 사용자와의 상호작용 방식에서 비효율이 있었다면 SKILL.md 수정 제안

## Step 4. 기록

위 1~3단계의 결과를 AUDIT_RESULT.md의 "회고" 섹션을 만들어 기록한다.
