# 공통 디자인 점검 항목

컴포넌트 리뷰, 페이지 리뷰 모두에서 확인해야 하는 항목.

## 1. Figma 레이어 구조

피그마의 레이어 패널에서 요소의 계층 구조를 파악한다.

- 부모-자식 관계가 HTML 마크업 구조로 자연스럽게 대응되는가?
- 시각적으로는 같은 레벨이지만 레이어에서 중첩된 요소가 없는가?
- Auto Layout이 적용된 프레임과 아닌 프레임을 구분한다.

## 2. 인터랙션 범위

어디까지가 클릭/탭 가능한 영역인지 확인한다.

- 클릭 영역이 시각적 요소보다 넓은가? (터치 타겟 최소 44x44px)
- 아이콘만 보이지만 클릭 영역은 텍스트까지 포함하는 경우가 있는가?
- **신규 컴포넌트**의 hover / active / focus 상태가 정의되어 있는가? (디자인 시스템에 이미 있는 컴포넌트는 제외)

## 3. margin vs padding vs size

여백이 어떤 방식으로 구현되는지 확인한다.

- 요소 간 간격이 부모의 gap/padding인가, 자식의 margin인가?
- Auto Layout의 spacing과 padding을 구분한다.
- 여백이 특정 요소의 크기(width/height)에 포함되는 경우를 확인한다.

## 4. 크기 정책

각 요소의 크기가 어떤 방식으로 결정되는지 확인한다.

- **Fixed**: 고정 크기 (px 단위)
- **Fill**: 부모의 남은 공간을 채움 (CSS flex: 1 또는 width: 100%)
- **Hug**: 내부 콘텐츠에 맞춰 축소 (CSS width: fit-content)
- 가로/세로 각각 다른 정책이 적용될 수 있다.

## 5. 텍스트 줄바꿈

텍스트가 두 줄 이상이 될 가능성과 그때의 처리 방법을 확인한다.

- 한 줄로 고정인가? (말줄임 처리 — text-overflow: ellipsis)
- 여러 줄 허용인가? (최대 줄 수 제한이 있는가?)
- 줄바꿈 시 인접 요소의 레이아웃이 어떻게 변하는가?
