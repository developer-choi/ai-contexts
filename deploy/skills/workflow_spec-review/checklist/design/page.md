# 페이지 전용 디자인 점검 항목

디자이너가 피그마로 전달한 페이지 시안을 검토할 때 추가로 확인하는 항목.

> 빈 상태, 에러 상태, 데이터 극단값의 **존재 여부**는 기획 체크리스트([edge-cases.md](../edge-cases.md))에서 점검한다. 여기서는 해당 상태의 **UI 형태**를 점검한다.

## 1. 로딩 UI

- 전체 페이지가 로딩 상태인가, 특정 섹션만 로딩인가?
- 스켈레톤 UI가 정의되어 있는가? 있다면 어떤 형태인가?
- 로딩 중 사용자가 조작할 수 있는 영역이 있는가?

## 2. 에러 UI

- 전체 페이지 에러 UI가 정의되어 있는가?
- 특정 섹션만 에러일 때의 UI가 정의되어 있는가?
- 재시도 버튼/액션이 포함되어 있는가?

## 3. 메타데이터

- 페이지 타이틀이 정의되어 있는가?
- OG 태그(타이틀, 디스크립션, 이미지)가 필요한 페이지인가?
- 필요하다면 OG 값이 정의되어 있는가?
- 동적 파라미터를 포함하는 페이지에서 메타데이터도 동적으로 바뀌어야 하는가?

## 4. 반응형

프로젝트 레벨의 반응형 정책은 global.md에서 한 번 정의한다. 여기서는 해당 페이지가 프로젝트 정책과 **다른 점**이 있는지만 확인한다.

- 이 페이지만 별도의 반응형 대응이 필요한가?
- 특정 요소가 화면 크기에 따라 숨겨지거나 재배치되는가?
