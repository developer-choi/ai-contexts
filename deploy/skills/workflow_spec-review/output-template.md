# 명세 분석 산출물 템플릿

리뷰 완료 후 AI가 작성하는 명세 분석 산출물의 형식.

산출물은 **디렉토리 단위**로 생성한다.

## 디렉토리 구조

```
[기능명]/
  global.md        # 공통 사항 (목적, API, 공통 컴포넌트/로직)
  layout-1.md      # 레이아웃 (같은 도메인 페이지가 공유)
  page-1.md        # 첫 번째 페이지
  page-2.md        # 두 번째 페이지
  ...
```

<ai-only>
페이지 번호는 사용자 플로우 순서대로 매깁니다.
각 page 파일 상단에 페이지명을 H1으로 적어 어떤 페이지인지 식별합니다.
</ai-only>

---

## global.md 템플릿

```markdown
# [기능명] 명세 분석

## 개요

- 서비스: [서비스 한 줄 설명]
- 목적: [이 기능의 목적]
- 대상 사용자: [대상 사용자]
- 핵심 플로우: [페이지 순서 요약]

## 공통 컴포넌트

### [컴포넌트명]
- props: [prop 목록]
- 상태: [신규/기존]
- 사용 페이지: [page-1, page-2]
- 구현 메모:
  - [구현 메모]

## 공통 로직
- [여러 페이지에 걸쳐 적용되는 로직]

## 공통 레이아웃
- [여러 페이지에 걸쳐 적용되는 레이아웃]

## 공통 엣지케이스
- [여러 페이지에 걸쳐 적용되는 엣지케이스]

## 기획자/디자이너 질문 목록
- [전체 페이지 종합 질문]
```

## layout-N.md 템플릿

```markdown
# [레이아웃명]
- 적용 페이지: [page-1, page-2, ...]
- [간단한 역할]

## 레이아웃 구조
- [헤더 / 사이드바 / 푸터 등 고정 영역과 콘텐츠 영역의 배치]

## 공통 UI 요소
- [네비게이션, 브레드크럼, 사이드바 메뉴 등 레이아웃에 포함되는 요소]

## 레이아웃 메타데이터
- 타이틀: [브라우저 탭에 표시될 페이지 제목]
- OG 타이틀: [공유 시 표시될 제목]
- OG 디스크립션: [공유 시 표시될 설명]
- OG 이미지: [공유 시 표시될 이미지]
```

---

## page-N.md 템플릿

```markdown
# [페이지명]
- URL: [/path/:param?key=value]
- 렌더링 방식: [SSG / Streaming / CSR]
- [간단한 역할]

## 페이지 메타데이터
- 타이틀: [브라우저 탭에 표시될 페이지 제목]
- OG 타이틀: [공유 시 표시될 제목]
- OG 디스크립션: [공유 시 표시될 설명]
- OG 이미지: [공유 시 표시될 이미지]

## API 목록

### [API 이름]
- method: [GET/POST/PUT/DELETE]
- parameter: [요청 파라미터]
- success response: [성공 응답 형태]
- error response: [에러 응답 형태]
- 에러처리: [모달 or Error Boundary]
- 로딩처리: [버튼 로딩 or 스켈레톤]
- 성공처리: [특별한 동작이 필요한 경우 작성]

## 함수 / Hook

### [함수/hook명]
- parameter: [param: type]
- return: [ReturnType]
- 엣지케이스: [throw 조건]

<ai-only>
사용자가 엣지케이스를 "없음"으로 작성한 경우, 실제로 없는지 검증합니다. 입력값의 극단 케이스(빈 값, null, 변환/필터링 결과가 0건 등)를 확인하고, 누락이 있으면 짚습니다.
</ai-only>
```
