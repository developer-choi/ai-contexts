# Rules 특화 체크리스트

`deploy/rules/` 리뷰 시 base audit(`audit/SKILL.md`)의 **명령문** 체크리스트를 순회한 뒤, 아래 항목을 추가 순회한다.

## 배경

rules 파일은 `~/.claude/rules/`에 배포되어 **매 세션 자동 로드**된다. skills나 contexts와 달리 명시적 호출 없이 항상 컨텍스트를 소비하므로, 간결성과 자체 완결성이 특히 중요하다.

## 특화 항목

### [CRITICAL] 분량 제한

Claude Code 공식 문서 기준, CLAUDE.md 및 rules 파일은 **200줄 이하**를 권장한다. 초과 시 컨텍스트 소비가 늘고 지시 준수율이 떨어진다.

- 파일이 200줄을 초과하는지
- 초과한다면 분리하거나 축소할 수 있는 내용이 있는지

### [CRITICAL] 자체 완결성

- 다른 파일 경로를 참조하고 있지 않은지 (예: "자세한 내용은 X.md 참고" 같은 문장)
- 참조가 있다면, 해당 내용을 인라인하거나 해당 규칙을 삭제해야 한다

### 소속 적합성

rules는 **항상 로드되는 전역 규칙**이므로, 특정 스킬이나 특정 프로젝트에만 해당하는 내용이 섞여 있으면 안 된다.

- 특정 스킬 실행 시에만 필요한 규칙이 rules에 들어있지 않은지
- 특정 파일 타입에만 적용되는 규칙이라면 `paths` frontmatter로 범위를 제한했는지
