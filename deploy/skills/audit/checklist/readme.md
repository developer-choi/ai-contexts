# README.md 검증

README.md는 **사람만을 위한 소개 문서**다.

> 적용 대상: 검증 대상이 README.md이거나, 검증 대상 폴더에 README.md가 포함된 경우

## 독자 설정

- 이 문서를 읽는 사람이 "이게 뭔지, 왜 만들었는지, 어떻게 쓰는지"를 알 수 있는지
- AI가 아닌 사람(동료, 오픈소스 방문자)이 읽었을 때 자연스러운지

## SKILL.md와의 역할 분리

- README.md: 사람용 소개 (목적, 배경, 사용법, 주요 산출물, FAQ 등)
- SKILL.md: AI용 실행 지시 (frontmatter, 행동 가이드, 체크리스트 참조 등)
- 같은 내용이 README.md와 SKILL.md에 중복되어 있지 않은지
- README.md에서 SKILL.md로의 링크가 있는지 ("자세한 내용은 SKILL.md를 참고하세요")

## 필수 내용

- **제목 + 한 줄 설명**: 이게 뭔지 즉시 파악 가능한지
- **왜 만들었는지**: 어떤 문제를 해결하려 했는지. 배경이 없으면 도구의 존재 이유를 모른다
- **사용법**: 실행 방법이 한눈에 보이는지 (예: `/audit 파일경로`)

## 선택 내용

필수는 아니지만, 문서의 설득력과 이해도를 높이는 섹션. 없다고 지적하지 않되, 있으면 품질을 검증한다.

- **사용 예시**: 입력 → 결과를 구체적으로 보여주는지 (가장 효과적인 소개 방법)
- **주요 산출물/구성 요소**: 이 도구가 무엇을 만들어내는지
- **FAQ**: 예상되는 의문에 선제 답변

## 양식

```markdown
# [도구/스킬 이름]

[한 줄 설명]

- **사용법**: `/[스킬명] [인자]`

## 무슨 문제를 해결하려고 했나요?

[배경과 동기. 왜 이 도구가 필요한지.]

## 사용 예시

[입력 → 결과를 구체적으로 보여주는 예시. 선택사항이지만 가장 효과적인 소개.]

## [도구별 고유 섹션]

[주요 산출물, 작업 흐름, 구성 요소 등 도구 성격에 맞는 내용]

---

## FAQ

### [예상 질문]?

[답변]

> 자세한 내용은 [SKILL.md](SKILL.md)를 참고하세요.
```
