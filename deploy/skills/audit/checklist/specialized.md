# 특화 체크리스트 작성 가이드

새로운 specialized 파일을 작성하거나 기존 파일을 리뷰할 때 참고하는 가이드.

## 필수 구조

모든 specialized 파일은 아래 구조를 따른다.

### 1. 헤더

`# [대상] 특화 체크리스트` 형식으로, 어떤 대상의 리뷰에 추가 적용되는지 명시한다.

### 2. 배경 (해당 시)

대상의 특성상 base audit만으로 부족한 이유를 설명한다. 왜 특화 항목이 필요한지 모르면 항목의 중요도를 판단할 수 없다.

### 3. 진행 방식 (해당 시)

리뷰 순서가 일반적이지 않은 경우에만 명시한다.

- 파일 단위 개별 리뷰 (예: coding-standards — 각 파일이 독립적)
- 전체 파악 후 리뷰 (예: workflow — 파일 간 순서와 흐름이 중요)
- 명시하지 않으면 base audit의 기본 순서를 따른다

### 4. 특화 항목

base audit에 없거나, 이 대상에서 특별히 엄격하게 적용해야 하는 항목만 기술한다.

## 중복 방지 원칙

- base audit checklist에 이미 있는 항목을 specialized에서 반복하지 않는다
- base audit 항목을 "특히 엄격하게" 적용해야 하는 경우, 항목을 복사하지 말고 base audit 파일명과 항목명을 참조한다
  - 예: `base audit checklist/consistency.md의 "참조 완결성" 항목을 특히 엄격하게 적용한다`
- specialized 파일 간에도 동일 항목을 중복하지 않는다. 2개 이상의 specialized에서 필요한 항목이면 base checklist로 승격을 검토한다
