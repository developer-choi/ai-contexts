# 출력 형식

검증 결과는 대화에 직접 출력하지 않고, **마크다운 파일(.md)로 작성**한다.

## 파일 위치

- 검증 대상 프롬프트와 같은 디렉토리에 `AUDIT_RESULT.md`로 저장한다.
- 이미 `AUDIT_RESULT.md`가 존재하면 기존 내용에 추가한다.

## 파일 구조

```markdown
# AUDIT_RESULT — <검증 대상 파일명>

## AUDIT_TARGET

<ai-only>
체크리스트 순회 및 자유 검토에서 발견한 AUDIT_TARGET의 문제점과 개선 제안을 기록한다.
지적(문제점)과 제안(추가하면 좋겠는 내용)을 구분하여 번호를 매긴다.
</ai-only>

### 지적 1 — (제목)

- **위치**: 파일명:행번호
- **문제**: 구체적 설명
- **권장**: 개선안

### 제안 1 — (제목)

- **내용**: 추가하면 좋겠는 내용 설명
- **근거**: 왜 이 내용이 필요하다고 판단했는지

## AUDIT_SOURCE 개선

<ai-only>
자유 검토 및 AUDIT_SOURCE 회고(retrospect.md)에서 발견한 체크리스트 개선 항목을 기록한다.
어떤 AUDIT_SOURCE 파일의 어디에 어떤 항목을 추가/수정해야 하는지 구체적으로 적는다.
</ai-only>

### 제안 1 — (제목)

- **대상**: 체크리스트 파일명
- **내용**: 추가/수정할 항목
- **근거**: 왜 이 항목이 필요하다고 판단했는지

-------------------------------------------------------------------------------------------
-------------------------------------------------------------------------------------------

# ✅ 반영할 항목 (커밋 대상)

> 위 지적 중 반영할 블록을 여기로 옮겨주세요.
> 블록에 코멘트(`==> 내용`)를 달아 수정 방향을 지시할 수 있습니다.

-------------------------------------------------------------------------------------------
-------------------------------------------------------------------------------------------

# ❌ 반려 (반영 안 함)

> 반영하지 않을 지적을 여기로 옮겨주세요.
> 블록에 코멘트(`==> 사유`)를 달아 반려 이유를 남길 수 있습니다.

-------------------------------------------------------------------------------------------
-------------------------------------------------------------------------------------------

# ❓ 질문 (재설명 요청)

> 이해가 안 되는 지적을 여기로 옮겨주세요.
> 블록에 코멘트(`==> 내용`)를 달아 구체적으로 질문할 수 있습니다.
```

## 규칙

- 지적은 **위치 → 문제 → 권장** 3항목을 반드시 포함한다.
- 제안은 **내용 → 근거** 2항목을 포함한다. AUDIT_SOURCE 개선 제안은 추가로 **대상**(체크리스트 파일명)을 포함한다.
- 지적/제안이 없는 섹션은 생략한다.
- 반영 안 할 지적/제안은 **반려** 섹션으로 옮기고, 반려 사유를 코멘트로 남긴다.

## CLI 안내

AUDIT_RESULT.md 작성 후 CLI에서 아래 내용을 안내한다:

- `AUDIT_RESULT.md` 작성 완료했습니다.
- 반영할 지적 → **반영할 항목** 섹션으로
- 반영 안 할 지적 → **반려** 섹션으로 (코멘트로 사유 남겨주세요)
- 이해가 안 되는 지적 → **질문** 섹션으로
- 각 블록에 코멘트(`==> 내용`)를 달면 한번에 요청/답변 받을 수 있습니다.
- 정리가 끝나면 알려주세요.
