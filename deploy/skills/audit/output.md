# 출력 형식

검증 결과는 대화에 직접 출력하지 않고, 마크다운 파일로 작성한다.

## 파일 위치

- 검증 대상과 같은 디렉토리에 `AUDIT_RESULT.md`로 저장한다.
- 이미 존재하면 기존 내용에 추가한다.

## 파일 구조

```markdown
# AUDIT_RESULT — <검증 대상 파일명>

### 1. (제목) [지적]

- **위치**: 파일명:행번호
- **문제**: 구체적 설명
- **권장**: 개선안
- **판정**: [승인 / 반려 / 질문 — 사유]

### 2. (제목) [제안]

- **내용**: 추가하면 좋겠는 내용 설명
- **근거**: 왜 이 내용이 필요하다고 판단했는지
- **판정**: [승인 / 반려 / 질문 — 사유]
```

## 규칙

- 각 항목에 번호를 매기고, `[지적]` 또는 `[제안]`으로 종류를 표시한다
  - 지적: 현재 문서에 존재하는 문제점
  - 제안: 없어도 되지만 추가하면 좋겠는 내용
- 지적은 **위치 + 문제 + 권장** 3항목을 반드시 포함한다
- 제안은 **내용 + 근거** 2항목을 포함한다
- **판정** 필드는 placeholder를 남긴다 — 사용자가 아래 중 하나로 채운다:
  - `승인` — 반영 (코멘트 없이 단독 사용 가능)
  - `반려 — 사유` (예: `반려 — 의도적인 중복임`)
  - `질문 — 내용` (예: `질문 — 이 부분 좀 더 설명해줘`)
- 항목이 없으면 파일을 생성하지 않는다
