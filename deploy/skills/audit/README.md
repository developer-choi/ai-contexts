# Prompt Audit

AI 프롬프트(md 파일)의 품질을 **체크리스트 기반으로 평가**하는 도구입니다.

- **목표**: AI 프롬프트 잘 쓰기
- **접근**: "잘 쓴 프롬프트"의 평가 기준을 정의하고, 그 기준으로 검증
- **사용법**: `/audit 파일경로`

## 사용 예시

### 1) 평가 대상 프롬프트 (못 쓴 예)

```markdown
# CONVENTIONS.md

### 규칙
if문 순서는 중괄호 블록이 작은 것에서 큰 것 순으로 작성해야 합니다.

### AI 행동 가이드
if문 순서가 중괄호 블록이 작은 것에서 큰 것 순으로 작성되어 있는지 확인하세요.
```

### 2) 이때 적용되는 체크리스트 (발췌)

```markdown
# checklist/consistency.md — 중복

- 같은 파일 내에서 여러 섹션이 같은 내용을 반복하고 있지 않은지
```

### 3) `/audit CONVENTIONS.md` 실행 결과

```markdown
# AUDIT_RESULT — CONVENTIONS.md

### 지적 1 — 규칙과 AI 행동 가이드 중복
- **위치**: CONVENTIONS.md:2-5
- **문제**: "규칙"과 "AI 행동 가이드"가 동일한 내용을 반복.
            규칙을 그대로 복사한 검증은 토큰만 낭비한다.
- **권장**: AI 행동 가이드를
            "위 규칙이 모두 반영되었는지 자가 검토하세요" 한 줄로 대체
```

## 무슨 문제를 해결하려고 했나요?

AI 프롬프트를 잘 작성하고 싶었습니다.

그러려면 두 가지가 필요합니다:

1. 사람이 AI 프롬프트 잘 작성하는 법을 이해하고 (공식 문서 등)
2. 본인 또는 타인이 작성한 프롬프트를 검증해야 합니다.

하지만 잘못된 부분을 찾고, 근거를 대고, 고치는 과정에도 시간이 듭니다.

사람이 생각하는 "잘 쓴 AI 프롬프트의 조건"을 체크리스트로 만들어 두면, 이 과정을 AI로 단축할 수 있다고 판단했습니다.

## 체크리스트 목록

| # | 관점 | 검증 내용 |
|---|------|-----------|
| 0 | 내부 정합성 | 자기모순, 중복, 모호한 참조, 암묵적 전제가 없는가 |
| 1 | 명확성 | 행동 동사, 맥락, 출력 형식이 지정되어 있는가 |
| 2 | 예시 | Bad/Good 대비로 기대 출력을 보여주는가 |
| 3 | 구조화 | 긴 자료는 상단, 지시는 하단으로 배치했는가 |
| 4 | 파일 분리 | 지시와 참고자료가 적절히 나뉘어 있는가 |
| 5 | 역할 | AI의 전문성, 행동 패턴, 톤이 구체적으로 정의되어 있는가 |
| 6 | 순서 | 단계별로 설계되고 완료 조건이 있는가 |

> 자세한 내용은 [SKILL.md](SKILL.md)를 참고하세요.

## 체크리스트 자가 개선

체크리스트의 유지보수도 AI가 보조하도록 설계했습니다.

핵심은 **AI 자가검진** — 사람이 시키지 않아도, AI가 먼저 개선점을 제안하는 구조입니다.

### 예시

[checklist/consistency.md](checklist/consistency.md)에는 "모호한 참조가 없는지" 항목이 있습니다.

그런데 검증 중 AI가 "모호한 참조는 아니지만, 섹션 제목과 내용이 불일치하는 문제"를 발견합니다.
기존 체크리스트에 이 항목이 없으므로, AI가 추가를 제안합니다:

```markdown
## AUDIT_SOURCE 개선

### 제안 1 — 섹션 제목과 내용 불일치 항목 추가
- **대상**: checklist/consistency.md
- **내용**: "섹션의 내용이 해당 섹션 제목이 의미하는 범위와 일치하는지" 항목 추가
- **근거**: 이번 검증에서 발견했으나 기존 체크리스트에 해당 항목이 없었음
```

사람이 승인하면 체크리스트에 반영되고, 다음 검증부터 같은 문제를 잡을 수 있습니다.

매 검증이 끝나면 [retrospect.md](retrospect.md)의 자유 검토 & 회고 절차를 통해, 체크리스트 자체의 빈틈을 찾고 개선안을 `AUDIT_RESULT.md`에 함께 출력합니다.

## FAQ

### 체크리스트가 무한히 비대해지면?

1차적으로, **사람이 체크리스트를 잘 관리해야 합니다.** AI가 추가를 제안해도 승인하는 것은 사람의 몫입니다.

2차적으로, 체크리스트가 많아져서 AI가 기존 항목을 제대로 수행하지 못하면 정리하는 것도 사람이 해야 합니다.

그럼에도 이 방법은 유효합니다. 사람 머릿속에만 있는 "잘 만든 프롬프트의 조건"으로 프롬프트를 유지보수하는 것보다는 낫다고 생각합니다.
