# 콘텐츠 배치 정책 (대주제)

KA / MP / AC 어디로 갈지 결정하는 글로벌 분업 정책. doc-router 스킬이 분류 시 이 문서를 참조한다.

## 메타 원칙

**프로젝트 = 대주제의 집합**
한 프로젝트는 하나의 대주제로 묶인다. 한 프로젝트가 여러 대주제를 차지하지 않는다.

**자료-활용 분리**
자료는 같은 성격으로 한 곳에 모이고, 자료를 활용하는 스킬은 여러 개일 수 있다. 자료 위치에 활용 방식을 박지 않는다.

## 대주제 분업

| 프로젝트 | 자료 형태 | 대주제 | 판단 질문 |
|---------|----------|--------|----------|
| KA (knowledge-archive) | 텍스트 (글) — 학습 정리, 코딩 팁 | 기억해야 할 것 | "이게 뭐야? / 왜?" |
| MP (monorepo-playground) | 코드 — 예제코드 또는 코드 직결 가이드 | 작성된 예제코드 | "이걸 어떻게 짜?" |
| AC (ai-contexts) | 짧은 규칙·설정 | 범용 원칙·규약 | "공통 원칙이 뭐야?" |

## MP의 정체

MP는 **예제코드 프로젝트**다. 본문이 텍스트로 보여도 MP 코드와 직결되어야 MP 행이다.

- `apps/example/` — Next.js로 표현된 예제코드
- `packages/design-system/` — 디자인 시스템 예제코드
- `docs/patterns/` — 코드 직결 가이드 (코드 스니펫 동반)

같은 라이브러리도 분류가 갈린다:
- MP patterns: "MP의 카드 컴포넌트 hover를 이렇게 짠다" (MP 코드 직결)
- KA: "useMotionValue는 무엇이고 어떻게 쓴다" (독립 학습 정리)

## 소주제

레포 안에서 어느 디렉토리에 둘지는 각 레포의 폴더 규칙 문서가 책임진다. 대주제 결정 후 해당 레포의 CLAUDE.md를 본다.
