# README 작성 가이드

README는 채용담당자(비개발자일 수 있음)가 읽는 문서입니다. GitHub 프로필에서 핀된 레포를 클릭해서 들어오는 사람이 대상입니다.

톤은 tone.md를 따르되, 1인칭 서술("제가 일하는 방식을", "제가 실무에서")을 사용합니다.

## 금지

- 트리 형식 다이어그램 (폴더 구조, 조직도 등)
- 설치 방법 (npm run, git clone 등)
- AI 지시문 톤 ("로드 시점", "frontmatter 태그" 등)
- 파일 경로 테이블
- 기술 약어 (KA, AC 등 — 풀어서 쓰거나 링크)
- 코드 레벨 용어 (변수명, 함수명, 설정값 등) — 일상 용어로 풀어쓰기
- 커밋 해시, git 명령어 출력, git log 스니펫 — 비개발자는 의미를 알 수 없음
- 개발자용 내부 문서 링크 (코드, 기술 노트 등) — 비개발자 독자가 클릭해도 이해 불가
- 예시·인용 블록 안의 기술 확장 괄호 부연 — 핵심 흐름만 남기고 삭제

## 필수

- **구체적인 예시** — 코드 예시는 최대한 간단하게, 아예 없으면 안 됨

### 스킬 README인 경우

- **"왜 만들었나요?"** — 문제 → 해결 구조
- **SKILL.md 링크** — 하단에 `> 자세한 내용은 [SKILL.md](SKILL.md)를 참고하세요.`
