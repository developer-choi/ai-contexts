# 스킬 자가 개선

스킬 실행 중에 "기록할 가치가 있는데 기존 규칙으로는 안 잡히는 빈틈"을 발견했을 때, 그 자리에서 스킬 자체를 개선하기 위한 메타 규칙이다.

## 적용 대상

분류·검증·라우팅성 스킬에만 적용한다.

- 분류·라우팅: 어떤 입력을 어디로 보낼지 판단하는 스킬 (doc-router 등)
- 검증·리뷰: 산출물에서 패턴 위반을 찾는 스킬 (code-review, scw, write-refine 등)

잘 정의된 단계형 절차 스킬(workflow의 단계 등)은 자가 개선 트리거가 노이즈가 될 가능성이 높으므로 적용하지 않는다.

## 트리거 조건

다음 두 트리거 중 하나가 충족되면 자가 개선을 시도한다.

### 자기 발견 트리거 (3개 모두 충족)

1. **스킬 실행 중 발견**: 사용자가 별도로 지적하지 않았는데 스킬이 자기 절차상 빈틈을 인식한 시점
2. **패턴성**: 동일 케이스가 다음 실행에도 재발할 가능성이 있음 (단발성·우연성 제외)
3. **기존 규칙 미커버**: SKILL.md·연결된 contexts·gotchas에 해당 케이스를 다룰 항목이 없음

### 사용자 행동 패턴 트리거 (3개 모두 충족)

1. **반복 관찰**: 사용자가 같은 카테고리·같은 결정을 N회 반복 (N=3 권장)
2. **결정의 일관성**: 사용자 결정에 흔들림 없음 (한 번이라도 다르게 처리했으면 트리거 안 됨)
3. **기존 규칙 미커버**: 그 패턴이 기존 SKILL.md·연결된 contexts에 명시 안 됨

세션 간 반복은 AI가 직접 기억하지 못하므로 사용자가 "전에도 그랬어" 식으로 알려준 경우에만 감지 가능. 단일 세션 내 반복은 AI가 직접 관찰한다.

각 트리거의 세 조건 중 하나라도 빠지면 자가 개선을 발동하지 않는다.

## 처리 방식

발견 후 스킬은 작업을 중단하지 말고 **현재 산출물은 수동 처리로 마무리**한다. 그 뒤 사용자에게 발견점을 보고하고 다음 둘 중 하나로 역제안한다.

### 즉시 수정

스킬 본체(SKILL.md) 또는 연결된 contexts에 한 줄~몇 줄 추가로 끝나는 변경. 예:

- gotchas에 새 버림 패턴 한 줄 추가
- 기존 규칙에 예시 보강
- 트리거 조건 좁히기

### 백로그 적재

다음 중 하나라도 해당하면 즉시 수정 대신 백로그로 보낸다.

- 새 행선지·새 디렉토리·새 파일 신설
- 다른 프로젝트(MP·KA·DP 등) 구조 변경 동반
- 공개 인터페이스(스킬 description, argument-hint, 외부 참조) 변경
- 같은 스킬 내 여러 섹션을 동시에 손봐야 하는 큰 변경

백로그 적재는 AC `backlog/` 워크트리 절차를 따른다 (`backlog/projects/` 중 적합한 영역 — AC 자체는 `ai-contexts/active/`).

## 역제안 형식

사용자에게 보고할 때 아래 항목을 포함한다.

- 발견 케이스 (어떤 입력에서 어떤 패턴을 만났는지)
- 기존 규칙 어디에 빈틈이 있는지
- 즉시 수정인지 백로그 적재인지 + 그 근거
- 추가/변경할 구체 문안

## 참조 방법

자가 개선 적용 대상 스킬의 SKILL.md 끝에 아래 한 줄을 둔다. 스킬 위치에 따라 경로가 다르다.

- `deploy/skills/<skill>/SKILL.md` (글로벌 배포용): `../../contexts/skill-self-improvement.md`
- `local/skills/<skill>/SKILL.md` (AC 본인용): `../../../deploy/contexts/skill-self-improvement.md`

```markdown
## 자가 개선

본 스킬은 [skill-self-improvement.md](경로) 메타 규칙을 따른다.
```
