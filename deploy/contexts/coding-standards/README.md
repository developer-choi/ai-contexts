# coding-standards

코딩 컨벤션, 품질 원칙, 테스트 규칙을 정의한 폴더입니다.

## 폴더 구조

| 폴더 | 용도 | 로드 시점 |
|------|------|-----------|
| **universal/** | 범용 컨벤션 (회사/개인 공통) | 항상 |
| **personal/** | 개인 프로젝트 전용 컨벤션 | 개인 프로젝트만 |

회사 프로젝트에서는 `universal/`만, 개인 프로젝트에서는 둘 다 로드합니다. 프로젝트 유형을 모르면 사용자에게 먼저 질문합니다.

## 하위 분류

각 폴더 내부는 주제별로 구성되어 있습니다:

| 경로 | 내용 |
|------|------|
| `general.md` | 공통 규칙 (주석, import 등) |
| `react/` | React 컴포넌트 패턴 |
| `typescript/` | TypeScript 타입 패턴 |
| `quality/` | 품질 원칙 (가독성, 유지보수성) |
| `testing/` | 테스트 작성 규칙 |
| `writing/` | 문서 작성 톤 |
| `git/` | Git 작업 규칙 |
| `markup/` | HTML/CSS 마크업 규칙 |

> 모든 주제가 양쪽에 다 있는 건 아닙니다. 하위 파일명/폴더명만 탐색하여 현재 작업에 필요한 파일만 선택적으로 로드합니다.

## 태그

폴더명·파일명만으로 발견하기 어려운 문서에는 frontmatter 태그가 붙어 있습니다. 파일명/폴더명으로 찾은 뒤, 아래 태그에 해당하는 작업이면 추가로 확인합니다.

| 태그 | 의미 |
|------|------|
| `file-folder-structure` | 파일·폴더 분리 기준. 구현 구조를 설계할 때 참조. |
