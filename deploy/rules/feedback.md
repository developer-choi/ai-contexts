## 코딩 컨벤션 경로
- 코딩 컨벤션 문서는 프로젝트 디렉토리가 아닌 `~/.claude/contexts/coding-standards/` 에 위치한다.
- 프로젝트 내부에서 glob 검색하면 찾을 수 없다. 반드시 절대 경로 `C:/Users/forwo/.claude/contexts/` 기준으로 검색할 것.

## 서브에이전트 리서치 결과 검증
- 서브에이전트가 가져온 핵심 주장(특히 "X가 Y로 분류됨" 같은 팩트)은 1차 소스를 직접 확인한 후 사용자에게 전달한다.
- Why: 서브에이전트가 "React가 cloneElement를 legacy로 분류"라고 보고했으나, 실제로는 "uncommon"이었음. 미검증 전달로 신뢰 손상.
- How to apply: 서브에이전트 리서치 결과 중 팩트성 주장은 WebFetch 등으로 원문을 직접 확인한 뒤 전달.

## 출처 제시 시 원문 인용 포함
- 외부 소스를 근거로 제시할 때 URL만 나열하지 않는다. 해당 페이지에서 근거가 되는 원문(영어 등)을 함께 인용한다.
- Why: URL만 주면 사용자가 직접 들어가서 찾아야 하므로 의미가 없음. 같은 질문이 반복됨.
- How to apply: 출처를 언급할 때 반드시 해당 원문 발췌를 함께 제시.

## 수정 지적 시 같은 맥락 일괄 탐색
- 사용자가 산출물(문서, 코드 등)에서 수정 1건을 지적하면, 같은 패턴의 문제를 산출물 전체에서 탐색하여 일괄 수정을 제안한다.
- Why: 사용자가 1개만 지적했을 때 그것만 고치면, 나머지 동일 문제를 사용자가 하나씩 반복 지적해야 함. 특히 /write-doc 말투 교정에서 빈번.
- How to apply: 수정 1건을 받으면 → 같은 맥락의 문제를 전체에서 찾아 → "같은 맥락으로 N건 더 발견했는데 함께 수정할까요?" 확인 후 일괄 수정.
