# 문제 해결 방법론

본 문서는 난관에 봉착했을 때 수단에 매몰되지 않고 목적을 달성하기 위한 사고 체계와 대응 프로세스를 정의합니다.

## 핵심 개념

### XY 문제
https://en.wikipedia.org/wiki/XY_problem

사용자가 원래 해결하려는 문제 **X**가 아니라, X를 해결하기 위한 시도 중 발생한 지엽적인 문제 **Y**에 매몰되어 질문하고 해결하려는 현상입니다.

- **X (본질):** PR 리뷰어 자동 지정
- **Y (수단/장벽):** Draft PR을 Active로 변경하기 위한 권한 획득

### 제1원칙 사고
기존의 관습이나 타인의 제안을 모두 지우고, 가장 근본적인 사실(불변의 진리)만 남겨두고 다시 설계하는 방식입니다. "리뷰어를 지정하는 데 반드시 Draft 상태를 거쳐야 하는가?"라는 질문이 그 시작입니다.

## 사고 전환 트리거 맵

다음과 같은 신호(Trigger)가 감지되면 즉시 손을 멈추고 상황을 객관화해야 합니다.

| 트리거 종류 | 감지 신호 (Signal) | 대응 (Action) |
| :--- | :--- | :--- |
| **복잡도 트리거** | A를 하려는데 B가 막히고, B를 뚫으려다 C를 공부하고 있을 때 | **All Reset:** B, C 없이 A를 할 방법 찾기 |
| **저항감 트리거** | 구현이 비정상적으로 어렵거나 '해킹' 같은 느낌이 들 때 | **Standard Check:** 플랫폼이 권장하는 표준 경로인지 확인 |
| **시간 트리거** | 동일한 지점에서 30분 이상 진척 없이 구글링만 반복할 때 | **Break:** 모니터를 떠나 화장실을 가거나 산책하며 줌아웃 |

## 실전 대응 프로토콜

트리거가 작동했을 때 스스로에게 던지는 2가지 질문입니다.

1. **"내가 지금 풀려는 '진짜 문제'가 뭐였지?"**
   - ❌ Bad: "어떻게 하면 봇에게 Draft 해제 권한을 줄 수 있을까?"
   - ✅ Good: "팀원 중 한 명을 랜덤으로 리뷰어로 지정하고 싶다."

2. **"이 방법이 안 된다면, 가장 단순한 대안은 무엇인가?"**
   - ❌ Bad: 권한 획득을 위해 외부 라이브러리나 복잡한 API 연동
   - ✅ Good: PR이 생성되거나 Ready 상태가 되는 시점에 즉시 지정

## 요약

무언가 비정상적으로 어렵게 느껴진다면 그것은 **실력이 부족해서가 아니라 길을 잘못 들었다는 신호**입니다. 수단은 언제든 바뀔 수 있으며, 중요한 것은 항상 **목적(X)**입니다.
