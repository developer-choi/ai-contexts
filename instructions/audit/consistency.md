# 내부 정합성 검증

다른 단계를 점검하기 전에, 명령문 내부에 자기모순이 없는지 먼저 확인하세요.

## 중복

- 같은 파일 내에서 여러 섹션이 같은 내용을 반복하고 있지 않은지

```
❌ Bad — 지시를 그대로 복사한 검증 (토큰만 낭비)
### 지시
1. 컴포넌트를 분리하세요
2. 에러 처리를 추가하세요

### 검증
1. 컴포넌트를 분리했는지 확인하세요
2. 에러 처리를 추가했는지 확인하세요

✅ Good — 검증은 별도 기준이나 관점을 제시할 때만 분리
### 지시
1. 컴포넌트를 분리하세요
2. 에러 처리를 추가하세요

### 검증
위 지시사항이 모두 반영되었는지 자가 검토하세요.
```

## 모호한 참조

- "위의 내용", "앞서 말한 것", "해당 부분" 같은 불명확한 참조가 없는지
- 참조가 필요하면 태그명이나 섹션명으로 명시했는지

```
❌ Bad — 불명확한 참조
위의 내용을 바탕으로 코드를 작성해줘.

✅ Good — 태그명으로 명시적 참조
<context>에 명시된 기술 스택을 기반으로 코드를 작성해줘.
```

## 섹션 제목과 내용 불일치

- 섹션의 내용이 해당 섹션 제목이 의미하는 범위와 일치하는지

## 암묵적 전제

- AI가 모를 수 있는 내부 지식을 전제하고 있지 않은지
- "우리", "평소처럼", "컨벤션에 맞게" 등의 표현이 있다면 해당 기준이 프롬프트 안에 정의되어 있는지

```
❌ Bad — AI가 모르는 내부 지식을 전제
우리 컨벤션에 맞게 작성해줘.

✅ Good — 기준을 프롬프트 안에 명시
아래 컨벤션에 맞게 작성해줘:
- 함수명: camelCase
- 컴포넌트명: PascalCase
- 파일명: kebab-case
- CSS: Tailwind 유틸리티 클래스 사용
```
