# 서식 검증

프롬프트의 서식이 프로젝트 규칙에 맞는지 점검하세요.

## 이모지

- 불필요한 이모지가 사용되고 있지 않은지
- 예외: ✅/❌는 Good/Bad 대조 또는 상태 표시(완료/미완료) 목적으로 허용
- 예외: ⛔(금지), ⚠️(경고), ❓(질문) 등은 의미 전달 목적으로 허용

## 문장

- 섹션 간 관계를 설명하는 전환 문장이 없는지 (예: "1~2는 ~하는 과정이다. 이 단계에서는~")
- 이전 섹션 내용을 요약·반복하는 문장이 없는지
- 각 문장이 실행 가능한 지시이거나, 실행에 필요한 정보인지

## 헤더 및 텍스트

- 한글(영어) 중복 병기가 없는지

```
❌ Bad — 동의어 반복
## 계획 (Plan)
## 요약 (Summary)

✅ Good — 간결하게
## 계획
## Summary
```

- 예외: 서로 다른 의미를 결합하거나 고유 명사를 함께 써야 하는 경우는 허용

```
✅ Allowed
## Track 2. 문제 해결 논리 검증 (식별자 + 설명)
## AI의 역할 (Quiz Runner) (구체적 모드 명시)
```
