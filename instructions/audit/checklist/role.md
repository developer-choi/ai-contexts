# 역할 검증

AI의 전문성, 관점, 말투를 고정할 필요가 있는 작업인지 확인하고, 필요하다면 아래를 점검하세요.

## 역할 정의

- AI가 어떤 직함/직군인지 명시되어 있는지 (예: 시니어 프론트엔드 엔지니어)
- 전문 분야가 특정되어 있는지 (예: React, 접근성, 성능 최적화)

## 행동 방식

- 역할에 맞는 행동 패턴이 명시되어 있는지
    - 예: "항상 근거를 제시", "오버엔지니어링 경고", "초보자도 이해할 수 있게 설명"
- 단순히 "시니어 개발자야"로 끝나지 않고, 구체적인 동작이 정의되어 있는지
- 잘못된 요청을 제지하고 대안을 제시하는 역제안 패턴이 있는지 (해당 시)

```
❌ Bad — 무조건 수용
사용자: "지난 3개월 커밋 분석해줘"
AI: "네, 분석하겠습니다"

✅ Good — 역제안
사용자: "지난 3개월 커밋 분석해줘"
AI: "3개월은 범위가 너무 넓습니다. 특정 날짜나 최근 1일을 분석할까요?"
```

- AI가 결과만 던지지 않고, 사용자 의견을 구하는 패턴이 있는지 (해당 시)

```
❌ Bad — 결과만 던지고 다음으로 진행
"background.md를 작성했습니다." (바로 다음 단계 진행)

✅ Good — 멈추고 의견 구하기
"제가 파악한 문제와 배경은 이렇습니다. 어떻게 생각하시나요?"
```

## 톤/스타일

- 응답의 어조가 지정되어 있는지 (예: 간결하게, 격식체로, 후배에게 설명하듯)
- 필요한 경우 금지 스타일도 명시되어 있는지 (예: 불필요한 칭찬 금지, 서론 생략)
