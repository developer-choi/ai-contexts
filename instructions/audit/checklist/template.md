# 템플릿 문서 검증

빈칸을 채워 출력물을 만드는 틀이 "채우는 사람이 막히지 않는가"를 확인한다.

## 섹션 설명

- 각 섹션이 무엇을 채워야 하는지 안내(주석, placeholder 등)가 있는지
- 안내가 너무 추상적이지 않은지 (예: "내용을 작성해주세요"만 있는 경우)

## 필수/선택 구분

- 반드시 채워야 하는 섹션과 선택적 섹션이 구분되어 있는지
- 조건부 섹션이 있다면 어떤 조건에서 채우는지 명시되어 있는지

## 죽은 섹션

- 실제 사용 시 거의 항상 비워지거나 삭제되는 섹션이 없는지
- 있다면 해당 섹션이 정말 필요한지, 다른 섹션에 합칠 수 있는지

## AI용 설명 분리 (`<ai-only>` 태그)

템플릿을 AI가 채워서 출력물을 만들 때, 문서에는 두 종류의 내용이 공존한다:

- **템플릿**: 출력에 그대로 나타나는 구조 (헤더, placeholder, 정적 텍스트)
- **AI용 설명**: AI가 채울 때 참고하는 지침 (출력에 포함되지 않음)

AI용 설명은 **템플릿 안에서** `<ai-only>` 태그로 감싼다. 태그 밖의 내용은 템플릿(출력 대상)으로 간주한다. 템플릿 바깥(별도 "규칙" 섹션 등)은 이미 템플릿이 아니므로 태그가 불필요하다.

```xml
## 추가 제안

<ai-only>
기존 지적과 달리, 문서에 문제가 있는 건 아니지만 추가하면 좋겠는 내용을 제안하는 섹션이다.
최대 3개.
</ai-only>

### 제안 1 — (제목)
- **내용**: ...
- **근거**: ...
```

### 검증 항목

- 템플릿과 AI용 설명이 공존하는 문서에서, AI용 설명이 `<ai-only>` 태그로 감싸져 있는지
- 태그 밖에 AI 전용 설명이 누출되어 있지 않은지 (출력에 섞여 들어갈 수 있음)
