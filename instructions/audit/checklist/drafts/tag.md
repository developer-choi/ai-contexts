# 태그 [보류]

## 태그 사용 여부

- 맥락, 지시사항, 예시, 입력 데이터가 각각 XML 태그로 구분되어 있는지

> Claude는 정해진 태그 스펙이 없습니다. 어떤 이름의 XML 태그든 자유롭게 사용할 수 있고, 열기/닫기 태그 사이의 내용을 하나의 블록으로 인식합니다. 아래는 관행적으로 자주 쓰이는 태그입니다.

| 태그 | 용도 |
|------|------|
| `<instructions>` | 지시사항 |
| `<context>` | 배경 정보 |
| `<examples>`, `<example>` | 입출력 예시 |
| `<input>` | 처리할 데이터 |
| `<documents>`, `<document>` | 여러 문서 묶기 |
| `<thinking>`, `<answer>` | 사고/답변 분리 |
| `<constraints>` | 제약 조건 |
| `<formatting>` | 출력 형식 지정 |

## 태그 일관성

- 프롬프트 전체에서 같은 내용에 같은 태그명을 사용하고 있는지
- 태그를 참조할 때 해당 태그명을 언급하고 있는지 (예: "위 `<context>`에 있는 기술 스택을 기반으로...")

## 중첩 구조

- 계층이 필요한 데이터(여러 문서, 여러 파일)는 중첩 태그로 구조화되어 있는지

```xml
<!-- 예시 -->
<documents>
  <document index="1">
    <source>파일명</source>
    <content>내용</content>
  </document>
</documents>
```

## 태그 조합

- 예시를 `<examples>`로, 사고 과정을 `<thinking>`과 `<answer>`로 감싸는 등 다른 기법과 조합했는지
