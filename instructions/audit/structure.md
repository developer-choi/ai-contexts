# 구조화 검증

프롬프트에 여러 종류의 정보(맥락, 지시, 예시, 입력값)가 섞여 있다면 아래를 점검하세요.

## 배치 순서

대량의 코드, 문서, 데이터를 프롬프트에 포함하는 경우 아래를 점검하세요.

- 긴 문서/코드가 프롬프트 **상단**에 배치되어 있는지
- 질문과 지시사항이 프롬프트 **하단**에 배치되어 있는지
- (배치만 바꿔도 응답 품질이 최대 30% 향상될 수 있음)

## 인용 기반 응답 유도

- AI에게 "관련 부분을 먼저 인용한 후 분석해줘"라고 요청했는지
- "파일/코드를 읽은 후에 판단해줘"와 같은 읽기 선행 지시가 있는지
