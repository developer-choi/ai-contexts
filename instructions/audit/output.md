# 출력 형식

검증 결과는 대화에 직접 출력하지 않고, **마크다운 파일(.md)로 작성**한다.

## 파일 위치

- 검증 대상 프롬프트와 같은 디렉토리에 `AUDIT.md`로 저장한다.
- 이미 `AUDIT.md`가 존재하면 덮어쓴다.

## 파일 구조

```markdown
# AUDIT — <검증 대상 파일명>

## 요약

| 단계 | 항목        | 지적 |
| ---- | ----------- | ---- |
| 0    | 내부 정합성 | N건  |
| 3    | 구조화      | N건  |

(지적이 있는 단계만 나열)

## 0단계: 내부 정합성

### 지적 1 — (제목)

- **위치**: 파일명:행번호
- **문제**: 구체적 설명
- **권장**: 개선안

### 지적 2 — (제목)

...

## 3단계: 구조화

(같은 형식 반복)
```

## 규칙

- 각 지적은 **위치 → 문제 → 권장** 3항목을 반드시 포함한다.
- 문제가 없는 단계는 요약 테이블과 본문 모두에서 생략한다.
- 요약 테이블을 파일 상단에 배치하여 전체 결과를 한눈에 파악할 수 있게 한다.
