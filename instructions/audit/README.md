# 명령문 검증 체크리스트

## 사용 방법

1. 검증할 프롬프트를 가져온다. 기본 단위는 파일 1개이지만, 프롬프트끼리 서로 참조하거나 관련성이 있으면 관련 프롬프트를 모두 가져온다.
2. 아래 체크리스트를 0단계부터 순서대로 전부 순회한다. 각 단계 안에 "필요 여부 판단" 항목이 있으므로, 해당하지 않는 단계는 그 시점에서 스킵한다.
3. 특화 체크리스트가 있는 경우(예: audit/workflow), audit 전체 순회가 끝난 후 특화 체크리스트를 추가로 순회한다. 특화 체크리스트는 audit에서 다루지 않는 도메인 고유 관점만 담는다.
4. 검증 결과는 `output.md`에 정의된 형식에 따라 마크다운 파일로 출력한다.

## 체크리스트

| 단계 | 파일               | 검증 내용                                    |
| ---- | ------------------ | -------------------------------------------- |
| 0    | consistency.md     | 내부 정합성 (중복, 모호한 참조, 암묵적 전제) |
| 1    | specific.md        | 명확성 (맥락, 구체적 지시, 출력 형식)        |
| 2    | examples.md        | 예시 (입출력 예시 필요 여부, 품질)           |
| 3    | structure.md       | 구조화 (배치 순서, 인용 유도)                |
| 4    | style.md           | 서식 (이모지, 헤더, 텍스트 규칙)             |
| 5    | file-separation.md | 파일 분리 (SKILL.md vs supporting files)     |
| 6    | role.md            | 역할 (역할 정의, 행동 방식, 톤)              |
| 7    | sequence.md        | 순서 (단계 분할, 검증 루프, 병렬 처리)       |
