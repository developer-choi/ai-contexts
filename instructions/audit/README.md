# 문서 검증 체크리스트

## 문서 종류

검증 대상 문서를 아래 3가지 중 하나로 분류한 뒤, 해당 체크리스트를 적용한다.

| 종류   | 판별 기준                             | 예시                             |
| ------ | ------------------------------------- | -------------------------------- |
| 명령문 | AI에게 행동을 지시하는 문서           | SKILL.md, CLAUDE.md              |
| 가이드 | 사람/AI가 참고하는 규칙·절차·배경지식 | CONVENTIONS.md, TESTING_GUIDE.md |
| 템플릿 | 빈칸을 채워 출력물을 만드는 틀        | PULL_REQUEST_TEMPLATE.md         |

> 문서가 짧거나 종류 간 경계가 모호하면 억지로 나누지 않는다. 하나의 체크리스트로 해당 항목만 순회하면 충분하다. 종류별 체크리스트를 별도 파일로 분리하는 것은 항목이 충분히 늘어난 뒤에 한다.

## AI 행동 가이드

1. 검증할 문서를 가져온다. 기본 단위는 파일 1개이지만, 문서끼리 서로 참조하거나 관련성이 있으면 관련 문서를 모두 가져온다. 검증 도중 참조 문서에서 문제를 발견한 경우, AUDIT.md 자유 검토 섹션에 해당 문제를 기록하고 "이 문서도 별도 audit이 필요하다"고 남긴다. 단, 참조 문서를 전부 검증하는 것은 아니며, 순회 중 자연스럽게 눈에 띈 문제만 기록한다.
2. 문서 종류를 판별하고, 해당 체크리스트를 0단계부터 순서대로 전부 순회한다. 각 단계 안에 "필요 여부 판단" 항목이 있으므로, 해당하지 않는 단계는 그 시점에서 스킵한다. 특화 체크리스트가 있는 경우(예: audit/workflow), 이어서 추가 순회한다.
3. **[CRITICAL] 자유 검토 및 프로세스 회고 — 이 단계가 검증의 핵심이다. 생략하면 검증이 완료된 것으로 보지 않는다.** `retrospect.md`의 범용 절차를 실행한다. audit 맥락에서 기록 형식은 아래와 같다:
   - 자유 검토에서 발견한 문제는 AUDIT.md `## 자유 검토` 섹션에 기록한다.
   - 각 지적에는 기존 형식(위치 → 문제 → 권장)에 **체크리스트 개선 제안**을 함께 적는다. 형식: "`{체크리스트 파일명}` 에 `"{추가할 항목}"` 추가 제안"
4. 2~3의 결과를 합쳐 `output.md` 형식에 따라 AUDIT.md로 출력한다.
5. `output.md`의 **CLI 안내** 섹션에 따라 사용자에게 안내한다.
6. 사용자가 정리 완료를 알리면:
   - **반영할 항목** 섹션의 블록을 코멘트(`==> 내용`)에 따라 수정 실행한다. 지적 1개당 커밋 1개씩 생성한다. 단, 리뷰 대상 파일 수정과 그 외 파일 수정은 별도 커밋으로 분리한다.
   - **커밋 시 수정한 파일만 staging**: AUDIT.md 등 관련 없는 파일이 딸려가지 않도록, 커밋 전 `git status`로 staged 목록을 확인한다.
   - **질문** 섹션의 블록에 대해 답변한다.
   - 반영 완료된 지적은 AUDIT.md에서 삭제한다. 삭제 후 비어버린 섹션도 함께 삭제한다.
   - 모든 지적이 처리되어 AUDIT.md에 내용이 없으면 파일 자체를 삭제한다.
7. 다음 배치로 넘어가기 전에 반드시 묻는다: "아쉬운 점이 있으셨나요? (놓친 지적, 기대했던 피드백 등)"
