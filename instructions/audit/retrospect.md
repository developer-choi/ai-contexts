# 자유 검토 및 프로세스 회고

정해진 프로세스(체크리스트, 워크플로우 등)를 실행하는 것만으로는 모든 문제를 잡을 수 없다. 프로세스는 "최소한 이건 해라"일 뿐이다. 프로세스 실행이 끝난 뒤 반드시 아래 절차를 수행한다.

## 1. 자유 검토

프로세스와 무관하게 대상을 처음부터 다시 보고, 프로세스가 잡지 못한 문제를 찾는다.

- 프로세스 항목에 해당하지 않지만 문제가 있는 부분이 있는지
- 프로세스가 다루지 않는 관점(구조, 분량, 존재 의의 등)에서 문제가 없는지

## 2. 프로세스 회고

프로세스 실행 결과를 되돌아본다.

- 프로세스의 각 단계를 다시 열어, 실제 실행과 1대1로 대조한다
- 지적/발견이 없었던 단계: 프로세스가 충분해서인지, 아니면 해당 유형의 문제를 아예 잡지 못하는 건지 판별
- 자유 검토에서 발견한 문제: 프로세스의 어디에 어떤 항목을 추가하면 잡을 수 있었는지 특정
- 누락한 절차가 있다면: 무엇을 누락했고, 왜 누락했는지(지시가 모호/단순 실수/불필요) 원인 분석

## 3. 개선 기회 제안

1~2는 "뭐가 잘못됐나"를 찾는 과정이다. 이 단계에서는 **"뭘 추가하면 더 좋아지나"**를 생각한다. 세션에서 쌓인 맥락을 활용하여, 현재 프로세스/가이드에 없지만 있으면 좋겠는 것을 제안한다.

착안점 예시:

- **반복 작업의 가이드화**: 세션에서 같은 유형의 작업을 수동으로 반복했다면, 가이드로 정형화할 수 있는지
- **사용자 반복 요청의 규칙화**: 사용자가 매번 같은 지시를 했다면, 해당 규칙을 conventions이나 workflow에 추가
- **AI 반복 실수의 사전 방지**: AI가 같은 유형의 실수를 반복했다면, 체크리스트나 가이드에 주의사항 추가
- **암묵적 판단의 명문화**: 세션에서 판단 기준 없이 감으로 결정한 것이 있다면, 해당 기준을 문서화
- **누락된 검증 단계**: "이걸 미리 체크했으면 나중에 안 고쳤을 텐데" 싶은 것이 있다면, 해당 검증을 프로세스에 추가
- **도구/자동화 기회**: 수작업으로 한 것 중 스크립트나 도구로 대체할 수 있는 것
- **새로운 프로세스 단계**: 기존 step 사이에 빠진 단계가 있다면 추가 제안
- **기존 단계의 세분화**: 하나의 step이 너무 많은 일을 하고 있다면 분리 제안

## 4. 기록

자유 검토, 프로세스 회고, 개선 기회 제안 결과를 기록한다. 각 지적/제안에는 **프로세스 개선 제안**(어떤 문서의 어디에 어떤 항목을 추가/수정해야 하는지)을 함께 적는다.
