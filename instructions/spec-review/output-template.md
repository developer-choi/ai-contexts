# 기획 분석 문서 템플릿

리뷰 완료 후 AI가 작성하는 기획 분석 산출물의 형식.

각 항목은 **확인됨 / 미확인 / 질문 필요** 중 하나로 분류한다.

---

## 템플릿

```markdown
# [기능명] 기획 분석

## 요약

- 목적:
- 대상 사용자:
- 핵심 플로우: (한 문장)

## 사용자 플로우

### 정상 경로 (Happy Path)

1. [진입] →
2. [액션] →
3. [결과]

### 분기

- 조건 A → 경로 A
- 조건 B → 경로 B

### 이탈/복귀

- 뒤로가기: (확인됨 / 미확인)
- 취소: (확인됨 / 미확인)
- 중간 이탈 후 복귀: (확인됨 / 미확인)

## 상태 전이

| 상태 | 전이 조건 | 다음 상태 | 기획서 정의 여부 |
|------|----------|----------|----------------|
| 초기 | 페이지 진입 | 로딩 | 확인됨 |
| 로딩 | API 성공 | 데이터 있음 | 확인됨 |
| 로딩 | API 실패 | 에러 | 미확인 |
| ... | ... | ... | ... |

## 필요 API

| API | 메서드 | 용도 | 존재 여부 |
|-----|--------|------|----------|
| /api/xxx | GET | 목록 조회 | 기존 API |
| /api/yyy | POST | 항목 생성 | 신규 필요 |
| ... | ... | ... | ... |

## 엣지케이스

| 시나리오 | 기획서 정의 | 비고 |
|---------|-----------|------|
| 데이터 0건 | 미확인 | 빈 상태 UI 필요 |
| 텍스트 오버플로우 | 확인됨 | 말줄임 처리 |
| 네트워크 에러 | 미확인 | 재시도 버튼 필요 여부 |
| ... | ... | ... |

## 미확정 사항 / 기획자 질문 목록

> 기획자에게 확인이 필요한 항목. 우선순위 순으로 정렬.

1. [P0] 질문 내용 — 이유: 구현 방향에 직접 영향
2. [P1] 질문 내용 — 이유: 엣지케이스 처리에 필요
3. [P2] 질문 내용 — 이유: 있으면 좋지만 나중에 확인 가능

## 일정 영향도

### 바로 구현 가능

- 항목 1
- 항목 2

### 기획 확인 후 구현 가능

- 항목 3 (질문 #1 답변 필요)
- 항목 4 (질문 #2 답변 필요)

### 일정 리스크

- 항목 5 — 사유: 신규 API 필요, 백엔드 일정 확인 필요
```
