# API/데이터 점검 가이드

기획서의 화면에서 필요한 데이터와 API 요구사항을 추출하고 검증한다.

---

## 점검 항목

### 1. 데이터 식별

화면에 표시되는 모든 데이터를 나열한다:

- 서버에서 가져오는 데이터 (API 응답)
- 사용자가 입력하는 데이터 (폼 필드)
- 클라이언트에서 계산/파생되는 데이터
- 로컬 상태로만 관리되는 데이터

### 2. API 존재 여부

각 데이터에 대해:

- 이미 존재하는 API로 가져올 수 있는가?
- 기존 API를 수정해야 하는가? (필드 추가, 필터 추가 등)
- 완전히 새로운 API가 필요한가?
- 여러 API를 조합해야 하는가?

### 3. 입력 유효성 검증

사용자 입력이 있는 경우:

- 필수/선택 필드 구분이 되어 있는가?
- 각 필드의 타입과 포맷이 정의되어 있는가? (문자열 길이, 숫자 범위, 날짜 형식 등)
- 클라이언트 검증과 서버 검증의 범위가 구분되어 있는가?
- 검증 실패 시 에러 메시지가 정의되어 있는가?

### 4. 데이터 갱신

- 데이터의 갱신 주기가 정의되어 있는가? (실시간 / 폴링 / 사용자 액션 시)
- 수정/삭제 후 목록이 자동 갱신되는가?
- 캐싱 전략이 필요한가?

### 5. 데이터 의존성

- 이 화면의 데이터가 다른 화면/기능에 영향을 주는가?
- 다른 화면에서 변경된 데이터가 이 화면에 반영되어야 하는가?
- 데이터 변경 시 연쇄적으로 변경되어야 하는 데이터가 있는가?
