# 엣지케이스 점검 가이드

기획서에서 정상 경로(happy path) 외의 예외 상황이 정의되어 있는지 점검한다.

---

## 카테고리별 엣지케이스

### 1. 빈 상태 (Empty State)

- 데이터가 0건일 때 화면이 어떻게 보이는가?
- 검색 결과가 없을 때 안내 문구/액션이 정의되어 있는가?
- 최초 사용자(첫 진입)와 데이터를 삭제한 사용자를 구분하는가?

### 2. 에러 상태

- 네트워크 에러 시 UI와 재시도 방법이 정의되어 있는가?
- 서버 에러(5xx) 시 사용자에게 보여줄 메시지가 있는가?
- 클라이언트 에러(4xx) — 특히 404, 403, 409 — 에 대한 처리가 있는가?
- 폼 제출 실패 시 입력값이 유지되는가?
- 타임아웃 시 동작이 정의되어 있는가?

### 3. 권한/인증

- 비로그인 사용자가 접근했을 때 동작이 정의되어 있는가?
- 권한이 없는 사용자가 URL 직접 접근 시 처리가 있는가?
- 세션 만료 시 동작이 정의되어 있는가?
- 권한 등급별(관리자/일반/뷰어 등) 화면 차이가 명시되어 있는가?

### 4. 데이터 극단값

- 텍스트가 매우 길 때 (말줄임? 줄바꿈? 스크롤?)
- 숫자가 매우 클 때 (자릿수 포맷, 오버플로우)
- 목록 아이템이 매우 많을 때 (페이지네이션? 무한스크롤? 제한?)
- 이미지가 없거나 깨졌을 때 대체 처리
- 특수문자, 이모지, 다국어 입력 시 처리

### 5. 타이밍/동시성

- 버튼 더블클릭 방지가 필요한가?
- 빠른 연속 입력(디바운스/스로틀) 처리가 필요한가?
- API 응답이 순서대로 도착하지 않을 경우 (race condition)
- 오래된 탭에서 돌아왔을 때 데이터 갱신 필요성
- 한정된 리소스(예약 슬롯, 재고 등)를 선택 후 확정하기까지의 사이에 다른 사용자가 선점할 수 있는가? (conflict/409 처리)

### 6. 웹 특화: URL 직접 접근 / multi-step form

- multi-step 플로우에서 중간 단계 URL에 직접 접근 시 이전 단계 데이터가 없는 경우 처리 (리다이렉트 등)
- 브라우저 뒤로가기/앞으로가기 시 상태 처리 (이전 입력값 유지 여부)
- 새로고침 시 현재 상태 유지 여부
- 브라우저 탭을 닫았다가 같은 URL로 재진입 시 처리
- URL 공유/북마크 시 의미 있는 페이지가 보이는가

### 7. 브라우저/환경

- 모바일 키보드가 올라올 때 레이아웃 영향
- 복사/붙여넣기 시 데이터 포맷 처리

## 기능 유형별 엣지케이스 매핑

특정 기능을 사용하면 자동으로 따라오는 엣지케이스. 기획서에 해당 기능이 포함되어 있으면 반드시 점검한다.

### 예약/슬롯 선택

- 선택 후 확정까지 사이에 다른 사용자가 선점 (conflict 처리)
- 화면에 표시된 슬롯이 시간 경과로 만료 (stale 슬롯 갱신)
- 예약 확정 후 배정 실패/취소 시 재시도 플로우
- 재시도 횟수 제한 및 초과 시 대안 (고객센터 등)

### multi-step form (퍼널)

- 중간 단계 URL 직접 접근 시 이전 데이터 없음 → 리다이렉트
- 뒤로가기 시 이전 입력값 유지
- 브라우저 새로고침 시 현재 단계 데이터 유실 여부
- 앱/탭 종료 후 재진입 시 이어서 진행 가능 여부

### 실시간/시간 민감 데이터

- 화면 체류 중 데이터가 변경되었을 때 갱신 방식 (폴링 / 웹소켓 / 포커스 시 refetch)
- stale 데이터로 액션을 시도했을 때 서버 거부 처리

### 결제/과금

- 결제 중 네트워크 끊김 → 결제 완료 여부 확인 방법
- 이중 결제 방지
- 결제 성공 후 후속 처리 실패 시 롤백/보상

### 파일 업로드

- 용량 초과 시 안내
- 지원하지 않는 파일 형식 시 안내
- 업로드 중 네트워크 끊김 시 재시도
- 동시 여러 파일 업로드 시 부분 실패 처리

### 검색/필터

- 검색 결과 0건 시 안내 및 대안 제시
- 빠른 연속 입력 시 디바운스 처리
- 필터 조합이 불가능한 경우 안내
