## 3. 코드 작성 규칙

### 타입 정의 및 네이밍
1. **Interface 확장**: 중복 방지를 위해 기존 공통 타입을 `extends`하여 사용
2. **Props 네이밍**:
   - 외국어 원문 데이터: `expression`
   - 한국어 뜻 데이터: `meaning`
