# Step 8: 워크플로우 회고

PR 본문 작성까지 완료된 후, **일하는 방식 자체**를 점검하고 개선하는 단계입니다.

[자유 검토 및 프로세스 회고](../../audit/retrospect.md) 절차를 워크플로우 맥락에서 실행합니다.

이 단계에서는 step 1~7을 함께 진행하며 쌓인 **세션 컨텍스트**를 적극 활용합니다.

---

## 1. 절차 준수 대조

실제 진행한 작업 내역과 `step-1.md` ~ `step-7.md`를 1대1로 대조합니다.

1. 각 step의 md 파일을 다시 열어 지시사항을 하나씩 확인
2. 실제 작업에서 해당 지시사항을 따랐는지 비교
3. 누락한 절차가 있다면:
   - 무엇을 누락했는지 구체적으로 명시
   - 왜 누락했는지 원인 분석 (지시가 모호했는지, 단순 실수인지, 불필요해서 건너뛴 건지)
   - 다음에 누락하지 않으려면 어떻게 해야 하는지 제안 (워크플로우 문서 개선 포함)

---

## 2. 사용자 패턴 분석

세션에서 사용자가 **반복적으로 요청하거나 수정 지시한 패턴**을 찾습니다.

- 같은 종류의 수정을 여러 번 요청했다면 → 해당 규칙을 가이드(workflow, conventions 등)에 추가 제안
- 매번 수동으로 지시하던 것이 있다면 → 자동화 또는 가이드화 제안

---

## 3. AI 자가 검진

세션에서 AI가 **반복 실수하거나 놓친 패턴**을 자가 점검합니다.

- 같은 유형의 실수를 여러 번 했다면 → 해당 주의사항을 가이드에 추가 제안
- 사용자에게 지적받고 나서야 알아챈 것이 있다면 → 사전 방지 규칙 가이드화 제안

---

## 4. 가이드 업데이트 제안

위 1~3에서 도출된 개선 사항을 정리하여 제안합니다.

- 어떤 파일(`instructions/workflow/`, `instructions/conventions/` 등)에 어떤 내용을 추가/수정해야 하는지 구체적으로 명시

---

> "워크플로우 회고 완료. 확인해주세요."
