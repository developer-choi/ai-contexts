# 코드 품질 리뷰 가이드

## 리뷰 원칙
- **명시 규칙 준수**: 하위 문서에 정의된 규칙을 최우선으로 적용합니다.
- **일반 표준 적용**: 문서에 없는 내용이라도 엔지니어링 표준(Clean Code, SOLID 등)에 따라 능동적으로 피드백합니다. "문서에 없다"는 이유로 결함을 방치하지 마세요.

## 피드백 방식
- **Why & How**: 단순 지적이 아닌 구체적인 근거와 개선 방향을 함께 제시합니다.

## 리뷰 예시
- **응집도**: "이 컴포넌트는 UI 렌더링과 데이터 로직이 섞여 있어 응집도가 낮습니다. Hook으로 분리하면 각자의 역할에 집중할 수 있습니다."
- **결합도**: "A 모듈이 B 모듈의 내부 구현을 직접 참조하고 있어 결합도가 높습니다. 인터페이스를 통해 의존성을 낮추는 것이 좋습니다."
- **구조 검증**: "현재 데이터 구조가 UI 형태와 1:1로 매칭되지 않아 불필요한 변환 로직이 발생합니다."
