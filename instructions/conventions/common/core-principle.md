# 핵심 개발 원칙 (Core Principles)

## 1. 개발 및 기술 스택 준수
- **기술 스택 엄수**: `package.json`에 명시된 라이브러리와 버전을 엄격히 준수합니다. 임의로 새로운 라이브러리를 추가하지 마세요.

## 2. Git 전략 & 작업 단위
### add 명령어 금지 사항
- `git add -A` 또는 `git add .` 사용을 **절대 금지**합니다.
- 반드시 `git add [특정파일]` 형태로 변경된 파일만 명시적으로 스테이징하세요. (Untracked 파일 실수 방지)

### 커밋 단위
- 논리적인 작업 단위(기능 구현, 버그 수정 등)가 완료되면 반드시 커밋을 남깁니다.
- 코드를 뭉쳐서 한 번에 커밋하지 말고, 의미 단위로 쪼개세요.

## 3. 답변 및 커뮤니케이션 태도
- **No Small Talk**: "안녕하세요", "최고의 어시스턴트입니다" 등 불필요한 인삿말을 하지 않습니다. 바로 본론으로 들어갑니다.
- **출처 제공 원칙**: 개발 관련 질문에 답변할 때 다음 우선순위로 근거를 제시하세요.
    1. **공식 문서** (MDN, Next.js Docs, React Docs 등)
    2. **신뢰할 수 있는 기술 블로그** (네이버 D2, 카카오, 토스, 우아한형제들 등)
    3. 위 출처가 없다면 일반적인 지식에 기반하여 답변.
