# 개요
프로젝트 코딩 컨벤션과 리뷰 규칙을 정의한 폴더입니다.

- **`coding/`**: 언어 및 프레임워크별 작성 규칙 (JS, TS, React, CSS)
- **`testing/`**: 테스트 코드 작성 규칙 (RTL, Jest)
- **`review/`**: 코드 리뷰 및 PR 작성 가이드
- **`essentials/`**: 프로젝트 핵심 가치 및 필수 점검 사항

---

# AI 컨텍스트 로딩 전략
효율적인 토큰 사용을 위해 **다음 규칙을 엄격히 준수하십시오.**

1. ❌ 통째로 읽기 금지: 이 폴더의 모든 파일을 한 번에 읽지 마십시오.
2. ✅ 예측 후 선별 로딩: 수행 중인 작업의 성격에 따라, **파일명과 폴더명만 보고** 필요한 파일만 골라서 읽으십시오.

## 예시
### 1. 기능 구현 및 버그 수정
> **User Intent**: "이 기능 만들어줘", "버그 고쳐줘", "이거 리팩토링 해줘"
- **Load Path**:
  1.  `essentials/`
  2.  `coding/`

### 2. 테스트 코드 작성
> **User Intent**: "테스트 짜줘", "테스트 깨지는데 고쳐줘"
- **Load Path**:
  1.  `essentials/`
  2.  `testing/`

### 3. 코드 리뷰 및 PR
> **User Intent**: "이 코드 리뷰해줘", "PR 내용 봐줘", "내 코드 어때?"
- **Load Path**:
  1.  `essentials/`
  2.  `coding/` (코드 품질 기준)
  3.  `testing/` (테스트 유무 확인)
  4.  `review/` (검수 체크리스트)
