# 개요
프로젝트 코딩 컨벤션과 리뷰 규칙을 정의한 폴더입니다.

코드 작성, 테스트, 리뷰 등 개발 작업 시 참고하세요.

---

## 폴더 구조

```
conventions/
├─ essentials/    # 프로젝트 핵심 가치 및 필수 점검 사항
├─ coding/        # 언어 및 프레임워크별 작성 규칙 (공통, TypeScript, React, 마크업)
├─ testing/       # 테스트 코드 작성 규칙
└─ review/        # 코드 리뷰 및 PR 작성 가이드
```

---

# AI 행동 가이드

## 기능 구현 및 버그 수정 시

1. **필수 규칙 로드**: `essentials/` 폴더의 모든 파일을 읽으세요.

2. **공통 규칙 로드**: `coding/` 폴더에서 파일명, 하위 폴더명을 보고 언어/프레임워크 무관한 공통 규칙 파일들을 찾아 읽으세요.

3. **기술 스택별 규칙 로드 (조건부)**: 작업 중인 코드의 기술 스택을 파악하고, `coding/` 하위 폴더에서 해당하는 규칙을 찾아 읽으세요.
   - 예: TypeScript 코드 작업 → `coding/typescript/` 읽기
   - 예: React 컴포넌트 작업 → `coding/react/`, `coding/typescript/` 읽기
   - 예: HTML/CSS 작업 → `coding/markup/` 읽기

---

## 테스트 코드 작성 시

1. **필수 규칙 로드**: `essentials/` 폴더의 모든 파일을 읽으세요.

2. **테스트 규칙 로드**: `testing/` 폴더의 모든 파일을 읽으세요.

---

## 코드 리뷰 및 PR 작성 시

1. **필수 규칙**: `essentials/` 폴더의 모든 파일을 읽으세요.

2. **코드 품질 기준**: `coding/` 폴더에서 파일명을 보고 공통 규칙 파일들을 찾아 읽으세요.

3. **테스트 확인**: `testing/` 폴더의 모든 파일을 읽으세요.

4. **리뷰 체크리스트**: `review/` 폴더의 모든 파일을 읽으세요.
